Feature 2 — กดเสร็จ/แก้สถานะ → ย้าย tab อัตโนมัติ (sync status ↔ serviceCompletedAt)

ปัจจุบัน: 2 ตัวแปรแยกกัน — tab "เสร็จแล้ว/กำลังรอ" ดูจาก serviceCompletedAt (timestamp) เท่านั้น; ส่วน status (ยืนยันแล้ว/เสร็จแล้ว ใน dropdown แก้ไขนัด) แยกขาดกัน → กดเสร็จแล้ว status ไม่เปลี่ยน / เปลี่ยน status ใน modal เป็น "เสร็จแล้ว" แล้ว card ก็ยังค้างอยู่ tab กำลังรอ.

ข้อเสนอ: ผูก 2 อย่างเข้าด้วยกัน (serviceCompletedAt = SSOT ของ tab เหมือนเดิม):

การกระทำserviceCompletedAtstatusผล (tab)
กด "✓ ลูกค้ารับบริการเรียบร้อย"= now→ done (ใหม่)→ tab "✓ เสร็จแล้ว"
แก้ไขนัด → status = "เสร็จแล้ว"→ stamp now (ใหม่)done→ tab "✓ เสร็จแล้ว"
กด "↩ กลับไปคิวรอ"→ null→ confirmed (ใหม่)→ tab "กำลังรอ"
แก้ไขนัด → status กลับเป็น "ยืนยันแล้ว" (จาก done)→ null (ใหม่/symmetric)confirmed→ tab "กำลังรอ"
