ตารางนัดหมาย — Density & Mobile Research (2026-05-20)

Root cause + 3 candidate approaches. NOT a final design — direction-finding only (brainstorming + spec come after the user picks).

Root cause

The time-grid hard-couples block height to duration: height = span × SLOT_H − 4 (SLOT_H = 22px per 15-min slot, AppointmentCalendarView.jsx:119,927). A 15-min appointment = 18px tall — physically too short for the customer-name line (text-sm), so it clips + overlaps the grid line. Dense back-to-back 15-min appointments → a column of illegible 18px blocks (the screenshot).

  • The grid already scales content by height: 🎯 purpose@span≥2 (30min), 👨‍⚕️ doctor@span≥3 (45min), + assistant@span≥4 (60min). So span≥2 reads fine — span=1 (15-min) is the failure.
  • Architecture signal: SLOT_H was already bumped 18→22 once (Phase 21.0-quinquies) for the same "ลายตา/ดูยาก" complaint. Bumping again is symptom-fix #2 → don't tune the broken dimension; add layers.
  • Mobile: columns are min-w-160px each (_colMinClass) inside overflow-x-auto → time-col + 3 rooms ≈ 480px+ → horizontal scroll on a 375px phone, on top of 56 rows × 22px = 1232px vertical scroll + the 18px blocks. 2D-scroll + illegible.

คำแนะนำ: ทำทั้ง 3 ชั้น แบบแบ่งเฟส
  1. A (popover) ก่อน — แก้ปัญหาหลัก "เห็นรายละเอียดตอนถี่" ได้ทันที, เสี่ยงต่ำสุด, ใช้ทั้ง desktop + mobile.
  2. B (adaptive cell + rollup) — ให้ตารางเองอ่านง่ายขึ้น (ไม่ล้นเส้น) + รวบ overlap.
  3. C (mobile agenda) — มือถือ default เป็นลิสต์การ์ด; เก็บ grid ไว้ให้ toggle.

เฟส 1 (A) ให้ผลมากสุดต่อความเสี่ยงน้อยสุด. ถ้าอยากเริ่มเล็ก เริ่มที่ A ก่อน.
