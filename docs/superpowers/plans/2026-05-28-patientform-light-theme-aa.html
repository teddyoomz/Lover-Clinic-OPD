PatientForm Light-Theme AA (B-i Selective) Implementation Plan

For agentic workers: REQUIRED SUB-SKILL: superpowers:executing-plans (inline, recommended for this single-file cosmetic stream) or superpowers:subagent-driven-development. Steps use HTML checkboxes for tracking. Spec: docs/superpowers/specs/2026-05-28-patientform-light-theme-aa-design.html.

Goal: Make every text/foreground color in PatientForm.jsx light theme ≥ WCAG-AA (4.5:1), pink/rose-harmonized, keeping per-section semantic zones; dark theme unchanged.

Architecture: Reuse V125's aaAccent(hex,isDark) (src/lib/themeAccent.js — map already has every needed shade). Fix the ~10 INLINE style={{color}} accents that bypass V124's class overrides; derive acLight = aaAccent(ac, isDark) for the dynamic clinic accent; add ONE shared .pf-req asterisk class. Decorative pinks/gradients + the LINE-green button + all V124-handled Tailwind classes are UNTOUCHED.

Tech Stack: React 19, Tailwind, CSS custom-props in src/index.css, Vitest. Rule K (work-first, test-last): source edits Tasks 1–4 → tests Task 5 → verify Task 6. Rule N: targeted tests during, full suite at batch end. No deploy without explicit word (V18).
