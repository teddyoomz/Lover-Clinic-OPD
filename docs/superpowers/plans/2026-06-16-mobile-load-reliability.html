Mobile-Load Reliability Implementation Plan

For agentic workers: execute task-by-task (inline executing-plans). Spec: docs/superpowers/specs/2026-06-16-mobile-load-reliability-design.html.

Goal: Make every mobile entry point (staff app + all customer links) load reliably on the first try — no black-screen / stuck-spinner / empty-skeleton — with an auto-retry → "ลองใหม่" escape that can never stay stuck, plus tests that prove it.

Architecture: (1) global connection fix: experimentalAutoDetectLongPolling; (2) one shared debounced reconnectFirestore(); (3) useResilientLoad hook + <LoadErrorRetry> at the customer links + AdminDashboard queue + App.jsx auth gate; (4) silent auto-heal inside useBranchAwareListener for backend tabs. No offline cache (Q1 = fresh-always).

Tech Stack: React 19, Firebase JS SDK 12, Vite 8, Vitest 4, Playwright 1.59.

Flow

Resilient load lifecycle + shared reconnect
StageWhat happens
1. subscribepage subscribes onSnapshot/fetch; useResilientLoad starts an 8s soft timer (loadStatus='loading')
2a. data arrivessnapshot/fetch callback calls markReady() → loadStatus='ready', timer cleared, attempts reset (doc-found AND doc-not-found both = ready)
2b. 8s, no data / onErrorhandleFail(): attempts<1 → first retry calls shared reconnectFirestore() (debounced network toggle) + bumps retryKey → page re-subscribes, timer restarts (still 'loading', silent to user)
3. retries exhaustedloadStatus='error' → page renders <LoadErrorRetry onRetry={retry}>
4. user taps ลองใหม่retry(): attempts=0, loadStatus='loading', bump retryKey → re-subscribe
App.jsx auth gateanon-auth hangs >8s → re-call signInAnonymously (≤2) → still stuck → LoadErrorRetry (kills black-screen-forever)
backend tabsuseBranchAwareListener: no onChange in 8s → re-subscribe (+reconnectFirestore), capped 2; resets on branch/args change; returns void (backward-compatible)

// src/lib/firestoreReconnect.js
// Shared debounced Firestore network toggle. Heals a half-dead WebSocket
// (looks connected, snapshot stream dead) by forcing a clean reconnect.
// Module-level debounce → many concurrent callers (V17 visibilitychange +
// useResilientLoad + useBranchAwareListener) collapse into ONE toggle (no thrash).
// ponytail: single global debounce; per-listener coordination only if profiling shows contention.
import { disableNetwork, enableNetwork } from 'firebase/firestore';
import { db } from '../firebase.js';

let lastToggleAt = 0;
let toggling = false;
const DEBOUNCE_MS = 1500;

export async function reconnectFirestore() {
  if (toggling) return;
  if (Date.now() - lastToggleAt < DEBOUNCE_MS) return;
  toggling = true;
  lastToggleAt = Date.now();
  try {
    await disableNetwork(db);
    await enableNetwork(db);
  } catch (err) {
    console.warn('[reconnectFirestore] toggle failed:', err?.message || err);
  } finally {
    toggling = false;
  }
}

// test seam — reset module debounce between unit tests
export function __resetReconnectDebounceForTest() { lastToggleAt = 0; toggling = false; }

Step 1: write the hook
// src/hooks/useResilientLoad.js
import { useState, useRef, useCallback, useEffect } from 'react';
import { reconnectFirestore } from '../lib/firestoreReconnect.js';

const DEFAULT_SOFT_TIMEOUT_MS = 8000;
const DEFAULT_MAX_AUTO_RETRIES = 1;

// 3-state resilient load machine for any onSnapshot/fetch load.
//   loadStatus: 'loading' | 'ready' | 'error'
//   retryKey:   put in the load effect's deps; bumping re-runs the load
//   markReady(): call in the SUCCESS callback (doc-found OR not-found both count)
//   markError(): optional — call in onError; funnels to the same retry/error path
//   retry():     bound to the LoadErrorRetry button
// First auto-retry triggers one shared reconnectFirestore() (heal half-dead conn).
export function useResilientLoad(opts = {}) {
  const softTimeoutMs = opts.softTimeoutMs ?? DEFAULT_SOFT_TIMEOUT_MS;
  const maxAutoRetries = opts.maxAutoRetries ?? DEFAULT_MAX_AUTO_RETRIES;

  const [loadStatus, setLoadStatus] = useState('loading');
  const [retryKey, setRetryKey] = useState(0);
  const attemptsRef = useRef(0);
  const statusRef = useRef('loading');
  useEffect(() => { statusRef.current = loadStatus; }, [loadStatus]);

  const markReady = useCallback(() => {
    attemptsRef.current = 0;
    setLoadStatus((prev) => (prev === 'ready' ? prev : 'ready'));
  }, []);

  const handleFail = useCallback(() => {
    if (statusRef.current === 'ready') return; // already succeeded — ignore late failures
    if (attemptsRef.current < maxAutoRetries) {
      const isFirst = attemptsRef.current === 0;
      attemptsRef.current += 1;
      if (isFirst) reconnectFirestore();        // heal half-dead conn before re-subscribe
      setRetryKey((k) => k + 1);                // re-run consumer load effect; stays 'loading'
    } else {
      setLoadStatus('error');
    }
  }, [maxAutoRetries]);

  const markError = useCallback(() => { handleFail(); }, [handleFail]);

  const retry = useCallback(() => {
    attemptsRef.current = 0;
    setLoadStatus('loading');
    setRetryKey((k) => k + 1);
  }, []);

  useEffect(() => {
    if (loadStatus !== 'loading') return undefined;
    const id = setTimeout(() => handleFail(), softTimeoutMs);
    return () => clearTimeout(id);
  }, [loadStatus, retryKey, softTimeoutMs, handleFail]);

  return { loadStatus, retryKey, markReady, markError, retry };
}
Step 2: unit tests (vitest fake timers + @testing-library/react renderHook):
  • initial loadStatus==='loading', retryKey===0
  • markReady() before timeout → 'ready', advancing past timeout stays 'ready'
  • timeout (advance 8s) with maxAutoRetries=1 → retryKey bumps, stays 'loading'; 2nd timeout → 'error'
  • markReady() after a soft-timeout retry → 'ready' (recovered)
  • markError() funnels to auto-retry then 'error' (same as timeout)
  • retry() from 'error' → 'loading' + retryKey bumps
  • first auto-retry calls reconnectFirestore (mock the module, assert called once)
  • markReady is idempotent across many snapshot fires

import { useEffect, useRef, useState } from 'react';
import { useSelectedBranch } from '../lib/BranchContext.jsx';
import { reconnectFirestore } from '../lib/firestoreReconnect.js';

const SOFT_TIMEOUT_MS = 8000;
const MAX_AUTO_RETRIES = 2;

export function useBranchAwareListener(listenerFn, args, onChange, onError) {
  const { branchId } = useSelectedBranch();
  const onChangeRef = useRef(onChange);
  const onErrorRef = useRef(onError);
  useEffect(() => { onChangeRef.current = onChange; }, [onChange]);
  useEffect(() => { onErrorRef.current = onError; }, [onError]);

  const isUniversal = listenerFn?.__universal__ === true;
  const effectiveBranchId = isUniversal ? null : branchId;
  const argsKey = JSON.stringify(args);

  const [retryNonce, setRetryNonce] = useState(0);
  const attemptsRef = useRef(0);
  // reset attempts whenever the SUBSCRIPTION identity changes (not on a retry)
  useEffect(() => { attemptsRef.current = 0; }, [listenerFn, effectiveBranchId, argsKey]);

  useEffect(() => {
    if (!listenerFn) return undefined;
    let enrichedArgs;
    if (isUniversal) enrichedArgs = args;
    else if (typeof args === 'object' && args !== null && !Array.isArray(args)) enrichedArgs = { ...args, branchId };
    else enrichedArgs = args;

    let gotData = false;
    // Silent auto-heal: if no onChange within SOFT_TIMEOUT_MS, reconnect + re-subscribe
    // (capped). Heals a listener that subscribed during a half-dead connection.
    const timer = setTimeout(() => {
      if (gotData) return;
      if (attemptsRef.current < MAX_AUTO_RETRIES) {
        attemptsRef.current += 1;
        reconnectFirestore();
        setRetryNonce((n) => n + 1);
      }
      // else: give up; V17 visibility/online + next branch switch recover.
    }, SOFT_TIMEOUT_MS);

    const unsub = listenerFn(
      enrichedArgs,
      (data) => { gotData = true; clearTimeout(timer); onChangeRef.current?.(data); },
      (err) => { onErrorRef.current?.(err); }
    );
    return () => { clearTimeout(timer); try { unsub?.(); } catch {} };
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, [listenerFn, effectiveBranchId, argsKey, retryNonce]);
}
Step 2: unit test — listener that never calls onChange → after 8s, re-subscribed (listenerFn called 2nd time) + reconnectFirestore called; capped at MAX; onChange fires → no retry; branch change resets attempts; existing behavior (onChange/onError forwarding, universal skip-branch) preserved.
