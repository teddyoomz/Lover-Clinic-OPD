For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use HTML checkboxes for tracking. Spec: docs/superpowers/specs/2026-05-26-appointment-hub-allbutton-opd-tab-lifecycle-design.html.

Goal: Four connected changes on the Frontend "นัดหมาย" (AppointmentHubView) tab + the patient-fill-link (opd_sessions) lifecycle: ① relabel Walk-in button → "เพิ่มนัดหมาย" reusing the existing AppointmentFormModal in create mode; ② new "รอ/ยังไม่ลง OPD" primary pill (states B+C+D, today+future); ③ cron hard-deletes a link once its appointment date passes; ④ hard-delete the session immediately on OPD-save success.

Architecture: ① reuses the SAME AppointmentFormModal AppointmentHubView already renders for edit (add a creatingAppt state mirror of editingAppt) — no new modal. ② adds a 5th TABS entry + an opd-pending case in appointmentHubFilters + an in-view state filter via the existing resolveLinkedSession join (mirror of cardFlowSubPillCounts). ③ adds a date-passed branch to the pure decideCleanupAction + a be_appointments join in the cron's sweepOpdSessionCleanup (sessions don't store the appt date — R5). ④ adds a best-effort hard-delete inside _attachLinkedBookings (the one function all 3 handleOpdClick success paths call), gated on the existing isFromBookingFlow predicate so kiosk walk-in sessions are never deleted.

Tech Stack: React 19 + Vite, Vitest 4 + @testing-library/react, firebase / firebase-admin, Bangkok-TZ midday-UTC date helpers. Pure-JS helpers in src/lib/*; cron in api/cron/*.

/**
 * ② (2026-05-26) — "รอ/ยังไม่ลง OPD" tab membership. True iff the appointment
 * is NOT-yet-in-OPD: state B (booking, no link), C (link sent, not filled), or
 * D (filled, not saved). Excludes A (existing customer), E (saved), and
 * cancelled. Single-source via resolveCardOpdState so any state-machine change
 * propagates to this tab automatically (AV131).
 *
 * @param {object} args
 * @param {object|null} args.appt
 * @param {object|null} args.linkedSession
 * @returns {boolean}
 */
export function isAppointmentOpdPending({ appt, linkedSession }) {
  if (appt?.status === 'cancelled') return false;
  const s = resolveCardOpdState({ appt, linkedSession });
  return s === 'B' || s === 'C' || s === 'D';
}
