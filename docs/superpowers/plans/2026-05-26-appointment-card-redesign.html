Appointment Card Redesign Implementation Plan

For agentic workers: REQUIRED SUB-SKILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use HTML checkboxes. Spec: docs/superpowers/specs/2026-05-26-appointment-card-redesign-design.html (open in browser — approved, dual-theme mockup).

Goal: Cosmetic-shell redesign of the appointment-hub card into a 5-band layout that is beautiful + theme-correct in BOTH Dark and Light, fixing the green-on-green OPD-pill bug — with ZERO changes to any button wiring/flow/logic/data-testid/conditional.

Architecture: 3 files. (1) _apptHubStyles.js — new OPD_PILL token family (light-base + dark:, mirroring STATUS_CHIP_CLS). (2) OpdLifecycleRow.jsx — swap dark-only classes → tokens, remove "⚙ OPD lifecycle" header span, rename save label → "บันทึกเข้าระบบ". (3) AppointmentHubRowCard.jsx — re-lay-out JSX into 5 bands; the round-circle stepper (AppointmentOpdStepperRow / TreatmentLifecycleStepper) is OFF-LIMITS, only re-parented verbatim.

Tech Stack: React 19 · Tailwind 3.4 (theme via data-theme + src/index.css override layer) · Vitest + RTL. Rules: Q-vis (verify pixels in real browser, both themes) · cosmetic-shell · N (targeted tests, full suite at end) · P (class-of-bug). Thai-culture: name never red.

COSMETIC-SHELL INVARIANT (the whole point): every data-testid, every onClick/handler prop, every render conditional (hasTreatmentForDay, rawStatus, isPastDate, showMarkCompleteBtn, showUnmarkBtn, opdLifecycle gates, effectiveStatus, …), and every button label EXCEPT the two sanctioned label changes (Q5 remove "OPD lifecycle" header text, Q6 "บันทึกลง OPD"→"บันทึกเข้าระบบ") stays byte-for-byte. Only wrapping markup + classNames move. The stepper is NOT edited/restyled/recolored.
