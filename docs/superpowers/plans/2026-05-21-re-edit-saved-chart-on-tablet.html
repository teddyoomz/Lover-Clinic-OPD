Goal: Edit (✏️) on a saved chart opens the same PC/tablet choice as add-new; sending an existing chart to the iPad loads its prior annotations (object-level when a re-editable fabricJson exists, raster otherwise) and the result merges back into the SAME slot.

Architecture: Thread a NEW session field editFabricJsonUrl through the existing Firestore-relay (no new collection, no new storage rule). PC sends the chart PNG (templateImageUrl, raster fallback) + the chart fabricJson (editFabricJsonUrl via the existing uploadTransportJson); the tablet resolves json-first → object-level loadFromJSON mirroring PC ChartCanvas:69-79, else raster loadTemplate. TreatmentFormPage untouched (ChartSection owns the relay).

Tech Stack: React, Fabric v7, Firebase Firestore + Storage, Vitest + RTL + Playwright-style real-prod e2e.

Rule K (work-first, test-last): build the 6-file structure (Tasks 1–5), then the test bank (Task 6), then Rule Q/S verify (Task 7). No deploy without explicit "deploy" (V18); object-level tablet path is live-gated on firebase deploy --only storage (#13).
