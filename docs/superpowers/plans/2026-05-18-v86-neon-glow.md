> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace V85's invisible-on-dark-theme black shadows with per-section dual-tone neon glow + 4s breath pulse + hover boost — making backend cards "เปล่งแสง" like cyberpunk signage, while keeping menu system + print views + customer-facing pages untouched.

**Architecture:** Pure CSS feature driven by CSS custom properties (`--neon-c1` / `--neon-c2`) scoped per `[data-section="<id>"]` wrapper. Auto-glow CSS rules consume the vars → 100s of cards light up from ~5 explicit edits + 2 cosmetic data-attr additions. Animation policy: 4s breath steady-state + hover-pause + sharp boost. Reduced-motion fallback to static.

**Tech Stack:** Vanilla CSS (no JS animation libs), CSS custom properties, `@keyframes` + `prefers-reduced-motion` + `prefers-color-scheme` via existing `data-theme` attribute.

**Why**: V86 — per-section glow color flows through CSS cascade via `--neon-c1` / `--neon-c2` custom properties set on `[data-section="<id>"]` wrappers (8 backend sections + 1 admin-frontend zone alias). **Grep**:

- Sanctioned exceptions: `:root` fallback (no section context) MAY hardcode fallback rgba values.
