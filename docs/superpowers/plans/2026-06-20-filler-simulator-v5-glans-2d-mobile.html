Filler Simulator v5 Implementation Plan

For agentic workers: implement task-by-task. Steps use HTML checkboxes for tracking. Spec: docs/superpowers/specs/2026-06-20-filler-simulator-v5-glans-2d-mobile-design.html.

Goal: Add an "initial glans size" slider (ratio of shaft Ø, default 1.0×, scales with diameter), enlarge the 2D illustration (desktop 40/60 + fill the card), and reorder mobile so the illustration sits above the controls.

Architecture: 4 files in the existing pure-client ?play=filler page; estimate() already accepts baseGlansDiameterCm so the math is unchanged — only a range constant + UI wiring + CSS. No new dependency, no data, no API.

Tech Stack: React 19 + Vite, inline styles + a small <style> block, Vitest source-grep + estimate() unit tests, Rule Q L1 via Claude Preview.

Task 1: GLANS_BASE_RATIO constant + estimate ratio test

Files: Modify src/lib/fillerMath.js; Test tests/filler-math.test.js

Step 1: Write the failing test (append inside the v2 glans describe block)
it('glans baseline ratio: dg0 = ratio × shaft Ø (scales with diameter)', () => {
  expect(GLANS_BASE_RATIO).toEqual({ min: 0.75, max: 1.25, step: 0.05, default: 1.0 });
  const dia = diameterFromGirth(10.4);                 // shaft Ø
  // ratio 1.0 → dg0 = shaft Ø (today's behavior)
  const a = estimate({ lengthCm: 12.7, baseGirthCm: 10.4, shaftCc: 5, glansCc: 2, baseGlansDiameterCm: 1.0 * dia });
  expect(a.glans.dg0).toBeCloseTo(dia, 6);
  // ratio 1.25 → dg0 25% larger; condom UNCHANGED
  const b = estimate({ lengthCm: 12.7, baseGirthCm: 10.4, shaftCc: 5, glansCc: 2, baseGlansDiameterCm: 1.25 * dia });
  expect(b.glans.dg0).toBeCloseTo(1.25 * dia, 6);
  expect(b.condomLow.w).toBe(a.condomLow.w);
});
