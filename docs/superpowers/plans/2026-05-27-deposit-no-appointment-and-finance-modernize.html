Deposit-without-appointment + Finance deposit modernize — Implementation Plan

For agentic workers: REQUIRED SUB-SKILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use HTML checkboxes for tracking. Spec: docs/superpowers/specs/2026-05-27-deposit-no-appointment-and-finance-modernize-design.html.

Goal: Add a ไม่นัดหมาย (no-appointment) deposit mode to AppointmentFormModal (writes a deposit-only doc), and modernize the Finance DepositPanel create form in place (เลือกลูกค้าภายหลัง + มัดจำสำหรับ), with full money-field parity preserved.

Architecture: Surgical extension of the EXISTING code (per user directive "อิงของเก่า") — no new modal, no new design. Reuse createDeposit (deposit-only writer), VisitPurposePicker (purpose chips), and the existing pickLater temp-customer pattern. be_deposits gains 3 optional string fields. The advisor→100%-seller behavior the appointment modal already does is inherited unchanged.

Tech Stack: React 19 + Vite + Firebase (Firestore) + Tailwind. Tests: Vitest + RTL + Rule I flow-simulate + Rule Q L2 (real-prod admin-SDK) + Rule Q L1 (user hands-on).

Decisions: Q1 = Modernize-in-place (two forms). Q2 = Port-all (no money field dropped). ที่ปรึกษา = appt-modal-only (→ seller); มัดจำสำหรับ = reuse นัดมาเพื่อ chip set.

Test ordering (Rule K + Rule I override of the skill's per-task TDD): lib tasks (T2, T3) are test-first; the UI tasks (T4–T6) build structure first; the test bank (T7–T9) lands after the UI structure is reviewable.
