V114 Receipt-Info Toggle Implementation Plan

For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use HTML checkboxes (<input type="checkbox">) for tracking.

Goal: Add an in-preview toggle ("ที่อยู่") to SalePrintView + QuotationPrintView headers that switches the customer block between a full receipt-info layout (taxId + address + phone + name) and a compact HN · โทร. single-line layout. State persists per-device via localStorage with a shared key across both views.

Architecture: Single shared hook useReceiptInfoToggle() writes/reads localStorage['lover_receipt_show_address']; cross-tab sync via window 'storage' event. Pure client UI — no backend, no rules, no migration. Default OFF (PDPA-friendly per Q3=B); both views consume the same key so toggling in either tab affects both immediately (Q5=A). Switch chrome lives inside the existing print:hidden sticky header — never appears on the printed PDF.

Tech Stack: React 19, Vite 8, Tailwind 3.4 (print:hidden + print:show utilities), Vitest 4.1 + RTL (@testing-library/react), Lucide icons. Existing V113-C mergedReceiptInfo walks snapshot+live priority chain — no new resolver needed.

Parent specs/lessons: V111 (buy-fetcher snapshot) + V112-A (updateBackendSale resolver) + V113 (renderer live-resolve) + V113-C (receiptInfo block live-resolve). Spec: docs/superpowers/specs/2026-05-23-receipt-info-toggle-design.html.

Out of scope: No backend writes, no firestore.rules, no audit doc, no migration, no per-customer override, no new printed-format mode, no backfill. Pure renderer toggle.

Flow

F-A — First open (localStorage empty)

Admin clicks 🖨 on a sale row→ SalePrintView mounts→ useReceiptInfoToggle reads localStorage→ key not set → DEFAULT_SHOW=false→ compact render: HN + phone single line

F-B — Toggle + print

Admin clicks switch (or label "ที่อยู่")→ setShowAddress(true)→ localStorage write "true"→ React re-render→ receipt-info block appears→ Admin clicks 🖨 พิมพ์ → window.print() → PDF includes full block (switch chrome hidden by print:hidden parent)

F-C — Cross-receipt persistence + cross-view shared key

SalePrintView closes (state OFF)→ Admin opens DIFFERENT sale→ hook reads localStorage = "false" → renders compact
Admin toggles ON in Sale→ localStorage = "true"→ Closes preview→ Opens QuotationPrintView→ Same hook reads same key → renders full block (Q5=A shared)

F-D — Cross-tab storage-event sync

Tab A: Admin toggles OFF→ localStorage write fires 'storage' event on OTHER tabs→ Tab B's hook subscribes to window 'storage' event→ readStorage() returns false → setShowAddressState(false)→ Tab B re-renders compact mode (no manual refresh)

F-E — Print CSS hides switch chrome

Header bar already has print:hidden on parent div (line 307 Sale / 169 Quotation)→ Switch + label sit INSIDE this bar→ window.print() → @media print drops the bar entirely→ Printed page shows ONLY the A4 document surface (sale-print-surface / quotation-print-surface)

describe('V114.H — useReceiptInfoToggle hook', () => {
  beforeEach(() => {
    try { localStorage.removeItem(STORAGE_KEY); } catch {}

  it('H1: defaults to false when localStorage empty (Q3=B PDPA-friendly)', () => {
    const { result } = renderHook(() => useReceiptInfoToggle());
    expect(result.current.showAddress).toBe(false);
  });

  it('H2: reads existing "true" from localStorage on mount', () => {
    localStorage.setItem(STORAGE_KEY, 'true');
    const { result } = renderHook(() => useReceiptInfoToggle());
    expect(result.current.showAddress).toBe(true);
  });

  it('H2b: reads existing "false" from localStorage on mount', () => {
    localStorage.setItem(STORAGE_KEY, 'false');
    const { result } = renderHook(() => useReceiptInfoToggle());
    expect(result.current.showAddress).toBe(false);
  });

  it('H3: setShowAddress(true) persists "true" to localStorage', () => {
    const { result } = renderHook(() => useReceiptInfoToggle());
    act(() => result.current.setShowAddress(true));
    expect(result.current.showAddress).toBe(true);
    expect(localStorage.getItem(STORAGE_KEY)).toBe('true');
  });

  it('H3b: setShowAddress(false) persists "false" to localStorage', () => {
    localStorage.setItem(STORAGE_KEY, 'true');
    const { result } = renderHook(() => useReceiptInfoToggle());
    act(() => result.current.setShowAddress(false));
    expect(result.current.showAddress).toBe(false);
    expect(localStorage.getItem(STORAGE_KEY)).toBe('false');
  });

  it('H4: cross-tab storage event updates state', () => {
    const { result } = renderHook(() => useReceiptInfoToggle());
    expect(result.current.showAddress).toBe(false);
    act(() => {
      localStorage.setItem(STORAGE_KEY, 'true');
      window.dispatchEvent(new StorageEvent('storage', { key: STORAGE_KEY, newValue: 'true' }));
    });
    expect(result.current.showAddress).toBe(true);
  });

  it('H4b: storage event for UNRELATED key does NOT affect state', () => {
    const { result } = renderHook(() => useReceiptInfoToggle());
    act(() => {
      window.dispatchEvent(new StorageEvent('storage', { key: 'some_other_key', newValue: 'whatever' }));
    });
    expect(result.current.showAddress).toBe(false);
  });

  it('H5: invalid localStorage value falls back to default false', () => {
    localStorage.setItem(STORAGE_KEY, 'garbage');
    const { result } = renderHook(() => useReceiptInfoToggle());
    expect(result.current.showAddress).toBe(false);
  });

  it('H6: setShowAddress coerces truthy to boolean true', () => {
    const { result } = renderHook(() => useReceiptInfoToggle());
    act(() => result.current.setShowAddress(1));
    expect(result.current.showAddress).toBe(true);
    expect(localStorage.getItem(STORAGE_KEY)).toBe('true');
  });

  it('H6b: setShowAddress coerces falsy to boolean false', () => {
    const { result } = renderHook(() => useReceiptInfoToggle());
    act(() => result.current.setShowAddress(true));
    act(() => result.current.setShowAddress(null));
    expect(result.current.showAddress).toBe(false);
    expect(localStorage.getItem(STORAGE_KEY)).toBe('false');
  });

  it('H6c: localStorage throw (private mode) — set still updates in-memory state', () => {
    const { result } = renderHook(() => useReceiptInfoToggle());
    const origSet = Storage.prototype.setItem;
    Storage.prototype.setItem = () => { throw new Error('QuotaExceeded'); };
    try {
      act(() => result.current.setShowAddress(true));
      expect(result.current.showAddress).toBe(true); // in-memory state updated
    } finally {
      Storage.prototype.setItem = origSet;
    }
  });
});

// src/hooks/useReceiptInfoToggle.js
// V114 (2026-05-23 EOD+1 LATE+2) — Receipt-info toggle shared by
// SalePrintView + QuotationPrintView. Persists per-device via localStorage;
// cross-tab sync via 'storage' event. Default OFF (Q3=B PDPA-friendly).
// Shared key (Q5=A) — toggling in either view affects both immediately.
//
// Pure renderer-level UI state. No backend, no rules, no migration.
// Parent: V111 (course-name) + V112-A (write resolver) + V113 (live-resolve)
// + V113-C (receiptInfo block). V114 is additive UI over V113-C.

import { useCallback, useEffect, useState } from 'react';

const STORAGE_KEY = 'lover_receipt_show_address';
const DEFAULT_SHOW = false;

function readStorage() {
  try {
    const raw = localStorage.getItem(STORAGE_KEY);
    if (raw === 'true') return true;
    if (raw === 'false') return false;
    return DEFAULT_SHOW;
  } catch {
    return DEFAULT_SHOW;
  }
}

export function useReceiptInfoToggle() {
  const [showAddress, setShowAddressState] = useState(readStorage);

  // Cross-tab sync: storage events fire on OTHER tabs when localStorage
  // changes in any tab.
  useEffect(() => {
    function onStorage(e) {
      if (e.key === STORAGE_KEY) setShowAddressState(readStorage());
    }
    window.addEventListener('storage', onStorage);
    return () => window.removeEventListener('storage', onStorage);
  }, []);

  const setShowAddress = useCallback((next) => {
    const val = !!next;
    setShowAddressState(val);
    try {
      localStorage.setItem(STORAGE_KEY, val ? 'true' : 'false');
    } catch {
      // private mode / quota — render still works, state just doesn't persist
    }
  }, []);

  return { showAddress, setShowAddress };
}
