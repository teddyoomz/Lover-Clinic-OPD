Sales Cancelled Sub-Tab Implementation Plan

For agentic workers: Execute task-by-task. Steps use HTML checkboxes for tracking. Spec: docs/superpowers/specs/2026-05-20-sales-cancelled-subtab-design.html.

Goal: Split cancelled sales out of the main "ขาย / ใบเสร็จ" list into a separate in-page sub-tab ("ยกเลิกแล้ว"), leaving the default "การขาย" tab showing only non-cancelled sales.

Architecture: One new pure helper (src/lib/saleSubTabFilter.js) + edits to one component (src/components/backend/SaleTab.jsx). Client-side split over the already-loaded sales array — no backend, no new fetch, no BSA change. Mirrors the V43-followup skipStockFilter.js pure-helper pattern and the StockTab segmented sub-tab pattern.

Tech Stack: React 19 (hooks), Vite, Vitest 4.1 + RTL, Playwright/preview for Rule Q L1. Thai UI.

Iron-clad notes: Rule N (targeted tests during iteration; full suite at batch end) · Rule I (flow-simulate at sub-phase end) · Rule Q V66 (real-browser/real-SDK before "verified") · Rule C1 (helper extraction = Rule of 3 testability, mirrors skipStockFilter) · NO deploy without explicit "deploy" this turn (V18) · preview against prod = read-only pill clicks ONLY, never click cancel/edit/save on real data (feedback_no_real_action_in_preview_eval).

// src/lib/saleSubTabFilter.js
// Sales-page sub-tab partition (2026-05-20). Splits the SaleTab list into the
// "การขาย" (non-cancelled) and "ยกเลิกแล้ว" (cancelled) sub-tabs. Pure JS — no
// React, no Firestore. Mirrors the V43-followup skipStockFilter.js pattern:
// single-source predicate so every reader agrees on what "cancelled" means.

/** A sale is cancelled iff its top-level status === 'cancelled'
 *  (sale.status, set by the cancel flow; beats payment.status). */
export function isCancelledSale(sale) {
  return sale?.status === 'cancelled';
}

/** Partition the loaded sales list for the given sub-tab.
 *  subTab === 'cancelled' → only cancelled; anything else → non-cancelled.
 *  Tolerates non-array input + null members (defensive). */
export function filterSalesBySubTab(sales, subTab) {
  const list = Array.isArray(sales) ? sales : [];
  if (subTab === 'cancelled') return list.filter(isCancelledSale);
  return list.filter((s) => !isCancelledSale(s));
}

import { dirname, join } from 'node:path';

const SALETAB = readFileSync(join(__dirname, '../src/components/backend/SaleTab.jsx'), 'utf8');

describe('F1 flow — load → switch tab → cancel → migrate', () => {
  it('F1.1 a cancelled sale migrates from active list to cancelled list', () => {
    // mirrors the real cancel flow: a sale's status flips to 'cancelled'
    let sales = [
      { saleId: 'INV-1', status: 'completed' },
      { saleId: 'INV-2', status: 'completed' },
    ];
    expect(filterSalesBySubTab(sales, 'active').map(s => s.saleId)).toEqual(['INV-1', 'INV-2']);
    expect(filterSalesBySubTab(sales, 'cancelled')).toEqual([]);

    // cancel INV-1 (reload returns it with status cancelled)
    sales = [
      { saleId: 'INV-1', status: 'cancelled' },
      { saleId: 'INV-2', status: 'completed' },
    ];
    expect(filterSalesBySubTab(sales, 'active').map(s => s.saleId)).toEqual(['INV-2']);
    expect(filterSalesBySubTab(sales, 'cancelled').map(s => s.saleId)).toEqual(['INV-1']);
  });

  it('F1.2 count line = selected sub-tab length (no double-count)', () => {
    const sales = [
      { saleId: 'A', status: 'completed' }, { saleId: 'B', status: 'cancelled' },
      { saleId: 'C', status: 'cancelled' },
    ];
    expect(filterSalesBySubTab(sales, 'active').length).toBe(1);
    expect(filterSalesBySubTab(sales, 'cancelled').length).toBe(2);

});
