Customer Patient-Link — ปุ่ม "🔗 ลิงก์ดูข้อมูล" ในหน้าดูข้อมูลลูกค้า (v2)

Date: 2026-05-25 (v2 corrected) · Status: Approved by user (design + mockup) · Implementation plan: docs/superpowers/plans/2026-05-25-customer-patient-link.html

Purpose / Problem

ผู้ใช้ต้องการปุ่ม "สร้างลิงก์ดูข้อมูล" ในหน้า CustomerDetailView เพื่อสร้างลิงก์ให้ ลูกค้า (anon, ไม่ต้อง login) เปิดดู นัดหมายครั้งต่อไป (พร้อมสาขา) + คอร์สคงเหลือ ของตัวเองได้ทุกเวลา. หน้าที่ลูกค้าเห็น = หน้า PatientDashboard เดิมที่ออกแบบไว้แล้ว (reuse 100%, ไม่ rebuild).

ground truth (อ่านโค้ดจริง): หน้า PatientDashboard ถูก rewire ไป be_* แล้ว (ไม่ตายหลัง V50) — fetchCoursesViaApi อ่าน getCustomer(be_customers)+getCustomerAppointments(be_appointments) เขียน latestCourses snapshot. AppointmentCard มีช่องแสดงสาขาอยู่แล้ว (line 289-294: {(a.branch||a.room) && <MapPin/>...}). CourseCard (เขียว/ส้ม/แดง) มีอยู่แล้ว. UI ทั้งหมด reuse — ไม่ต้องออกแบบใหม่.
ปัญหาที่แท้จริง (auth): be_customers+be_appointments+be_courses = allow read,write: if isClinicStaff() ทั้งหมด → anon อ่านไม่ได้. การ auto-sync ฝั่ง patient (anon) จึง fail เงียบ → นัดหมายว่าง (.catch(()=>[])) + คอร์สเห็นเฉพาะตอน admin เปิด. ⇒ ต้องใช้ server endpoint (admin SDK, token-gated) ดึงข้อมูลให้ anon. ไม่แตะ firestore.rules (be_* ยังล็อค clinic-staff).

Flow

A · clinic-staff สร้างลิงก์
กด 🔗 ในหน้าลูกค้า→ gen crypto token→ client write be_customers: patientLinkToken + patientLinkEnabled=true→ modal: ลิงก์ + QR + คัดลอก/เปิด/ปิด/เพิกถอน
B · ผู้ป่วยเปิดลิงก์ (anon · NO login)
เปิด ?patient=<token>→ PatientDashboard: GET /api/patient-view?token=→ endpoint (admin SDK) resolve token: be_customers OR opd_session (unified)→ อ่าน courses + be_appointments (resolve branchId→ชื่อสาขา, format วันที่เดือนเต็ม+พ.ศ., future-only)→ return รูปทรง latestCourses {courses, expiredCourses, appointments(+branch), patientName} + patientData + hn→ PatientDashboard render (view เดิม) — appointments+สาขาโชว์

token ไม่พบ/disabled → endpoint 404 → "ไม่พบข้อมูล"/"ปิดใช้งาน" (state เดิม). endpoint 404 → fallback path opd_session Firestore-listener เดิม (admin real-time, ไม่พัง).

Architecture

ชั้นรายละเอียด
Tokenbe_customers.patientLinkToken (crypto 128-bit hex) + patientLinkEnabled. สร้าง/เพิกถอนโดย clinic-staff (client SDK — be_customers เขียนได้โดย clinic-staff อยู่แล้ว).
api/patient-view.js (NEW)Public GET token-gated, admin SDK, NO Firebase auth needed. Unified resolve: be_customers WHERE patientLinkToken==token → courses; ELSE opd_sessions WHERE patientLinkToken==token → brokerProClinicId→getCustomer. แล้ว query be_appointments by customerId (future-only) + resolve branchId→ชื่อ (be_branches) + format date เดือนเต็ม+พ.ศ.. คืน {patientName, hn, courses[], expiredCourses[], appointments[](+branch)} field-minimized (ไม่มี national ID). 404 ถ้าไม่พบ/disabled.
PatientDashboard (reuse)เพิ่ม customer-mode: ลอง fetch(/api/patient-view?token=) → map ผลเป็น sessionData shape เดิม ({patientData, brokerProClinicHN, latestCourses}) → render เดิมทำงานเป๊ะ (patient card + AppointmentCard + CourseCard). gate auto-sync useEffect ไม่ให้รันใน customer-mode. 404 → fallback opd_session listener เดิม.
CustomerDetailViewrestructure แถวปุ่ม → Layout A. ปุ่ม 🔗 ม่วง เปิด CustomerPatientLinkModal. label "🔗 ลิงก์ ✓" เมื่อมี token.
CustomerPatientLinkModal (NEW)เลียนแบบ patient-link modal เดิม, customer-scoped.
backendClient.jsgenerateCustomerPatientLink · setCustomerPatientLinkEnabled · revokeCustomerPatientLink (client write be_customers).
UI patient viewไม่แก้ — AppointmentCard มีช่องสาขาแล้ว; CourseCard มีแล้ว. แค่ feed data ที่มี a.branch + วันที่เดือนเต็ม.
firestore.rulesไม่แตะ ✅ — be_customers/be_appointments/be_branches ยังล็อค clinic-staff. ไม่ต้อง Probe-Deploy-Probe.

Decisions (locked via Q&A + mockup with user)

  • Q1 = A: ลิงก์ระดับลูกค้า (อ่าน be_* สด).
  • Q2 (layout) = A: action group [แก้ไข·ผูก LINE·🔗 ลิงก์] + เส้นคั่น + [ลบลูกค้า] แยกแถว.
  • Data path = A2 unified server endpoint (admin SDK, token-gated) — resolve ได้ทั้ง be_customers token + opd_session token (ลิงก์เดิมในคิวก็ทำงาน anon ได้). ไม่เปิด anon-read be_*.
  • Reuse PatientDashboard view 100% (ไม่ rebuild) — ผู้ใช้ยืนยันจาก mockup.
  • นัดหมายต้องบอกสาขา — resolve branchId→ชื่อสาขา (AppointmentCard 📍 มีช่องแล้ว). + เดือนเขียนเต็ม ไม่ย่อ (THAI_MONTHS_FULL) + ปี พ.ศ.. แสดงเฉพาะนัดหมายในอนาคต.
  • Anon · NO login — endpoint validate ด้วย token เท่านั้น; customer-mode ไม่พึ่ง Firebase auth.
  • Modal = mirror patient-link เดิม: สร้าง/คัดลอก/QR/เปิด/เปิด-ปิด/เพิกถอน.
  • Field-minimization: ชื่อ·HN·courses·appointments(+สาขา). ไม่มีบัตร ปชช.

Acceptance Criteria

  1. หน้าลูกค้าแสดงปุ่ม 🔗 ลิงก์ (ม่วง) ในกลุ่ม action; "ลบลูกค้า" แยกแถวล่าง (Layout A) — desktop + mobile.
  2. กด 🔗 (ยังไม่มี token) → "สร้างลิงก์ดูข้อมูล" → เขียน token ลง be_customers → modal โชว์ลิงก์ + QR.
  3. Modal: คัดลอก / เปิด / QR / เปิด-ปิด / เพิกถอน ทำงานครบ.
  4. ลูกค้าเปิด ?patient=<token> โดยไม่ต้อง login → เห็นหน้าผู้ป่วยเดิม + ชื่อ + HN + นัดหมายครั้งต่อไป (เดือนเต็ม + 📍 สาขา) + คอร์สคงเหลือ (สด).
  5. endpoint resolve ได้ทั้ง be_customers token + opd_session token; 404 → fallback listener เดิม.
  6. token disabled → "ปิดใช้งาน"; ผิด/เพิกถอน → "ไม่พบข้อมูล".
  7. endpoint คืนเฉพาะ field ปลอดภัย; be_* ยังล็อค clinic-staff (ไม่แก้ firestore.rules); UI patient view ไม่ถูกแก้ (reuse).
  8. Rule Q: L2 endpoint verified บน prod จริง (TEST customer + appointments + branch resolve + cleanup) + L1 เปิดลิงก์จริงใน browser เห็นนัดหมาย+สาขา.

Risks / Open Questions

  • Public endpoint = attack surface — token crypto 128-bit + read-only + field-minimized; rate-limit เบา (optional).
  • App.jsx render gate — route ?patient= gate render จน anon-auth resolve (V16/V23). customer-mode ดึงจาก endpoint (ไม่พึ่ง auth) — anon-auth ที่ยังมีไม่เป็นพิษ; plan เช็คว่าไม่บล็อก.
  • appointments customerId matching — be_appointments.customerId ต้อง match id ที่ส่ง (V67 3-ID-layer risk). Rule R diag ตอน implement: ยืนยันลูกค้าในลิงก์มีนัดจริง + id ตรง.
  • PatientDashboard ไฟล์ใหญ่ — customer-mode เป็น branch เพิ่ม; Rule I flow-simulate ครอบทั้ง customer-mode + legacy opd_session fallback.
  • AVxx invariant — endpoint ต้อง field-minimize + be_* ห้ามเปิด anon-read (กัน regression).
