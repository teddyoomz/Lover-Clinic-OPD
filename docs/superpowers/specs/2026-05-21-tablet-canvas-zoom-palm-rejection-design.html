Tablet Canvas Zoom + Palm Rejection — Design Spec

Date: 2026-05-21 · Status: Approved by user ("approve ทุกส่วน ลงมือทำได้เลย") · Scope: TabletChartCanvas (the ?tablet=chart editor) only · Implementation plan: docs/superpowers/plans/2026-05-21-tablet-canvas-zoom-palm-rejection.html

Purpose / Problem

The tablet chart editor (TabletChartCanvas, Fabric v7 + perfect-freehand pen) currently has no zoom (canvas is fixed fit-to-frame) and only partial palm rejection — if (e.pointerType === 'touch' && penDownRef.current) return; rejects touch ONLY while a pen is already down. So a palm resting before the pen, and any 1-finger touch, currently draw. Clinicians want the behaviour of good iPad note/drawing apps (Procreate / GoodNotes): 2-finger pinch-zoom inside the frame for detail work, and resting the side of the hand (สันมือ) is recognised as palm, not pen, so it never draws.

Decisions (locked via Q&A with user)

  • Q1 = C (auto-adaptive input): pen seen this session → pen-only drawing (touch navigates, palm rejected); no pen yet → 1-finger draws (fallback); 2 fingers always zoom+pan; palm ignored in pen mode. Chosen because the clinic has mixed devices (iPad+Pencil and some stylus-less tablets).
  • Q2 = pinch + button reset: pinch-zoom 1× (fit) → 4×, zoom-to-centroid; pinch-out clamps to fit (natural reset); ⤢ button top-right = one-tap reset; 1-finger pans when zoomed; template stays crisp (viewport-zoom, not CSS scale).
  • Approach 1: Fabric viewportTransform + a thin raw-pointer-tracking gesture layer. Rejected: CSS transform (breaks getScenePoint + blurry), Fabric's old touch-gesture mixin (changed/dropped in v7, fragile).
  • Scope: TabletChartCanvas only. PC ChartCanvas is mouse-driven — pinch/palm N/A.

Acceptance Criteria

  1. In pen mode, a resting palm (touch) produces no stroke; only the pen draws.
  2. 2-finger pinch zooms 1× → 4×, clamped (never > 4× or < fit), centered on the pinch midpoint.
  3. Pinch-out returns to fit; the ⤢ button resets to fit in one tap; ⤢ is visible only when zoomed.
  4. The pen draws at the correct scene coordinates at any zoom level (via getScenePoint).
  5. While the pen is actively drawing, touch does not zoom (no mid-stroke zoom).
  6. Saving exports the full chart at canonical size regardless of current zoom (VPT reset on export); fabricJson coords unaffected.
  7. Loading a template / hydrating a re-edit starts at fit (zoom reset).
  8. In no-pen mode, 1 finger draws; a 2nd contact aborts the nascent stroke and starts pinch.

Risks / Open Questions

  • No-pen-mode palm rejection is best-effort — a palm resting before any pen is seen could draw (mitigated by 2/3-contact gesture detection; robust once a pen is seen). Accepted via choosing Q1=C.
  • Multi-touch is hardware-gated — synthetic events are isTrusted:false and Fabric/the router won't receive them (the more-tools lesson). Headless tests prove the gesture math + routing + export-VPT-reset + getScenePoint-under-zoom; real pinch/palm behaviour = user on-device L1 on the iPad.
  • Export-VPT-reset is the critical correctness invariant — a miss = saved charts cropped/zoomed. Locked by an acceptance test.
