Filler 2D Smart Auto-Scale (Model B, space-maximised)

Date: 2026-06-21 · Status: Approved by user (Model B + "ใช้พื้นที่คุ้มที่สุด") · Scope: src/components/FillerGraphic2D.jsx scale block only (+ its source-grep tests). Frontend-only, standalone filler bundle. Supersedes the "single fixed px/cm" approach from the cube-root spec.

Purpose / Problem

The cube-root-glans turn replaced the 2D scaling twice and broke two loved behaviours the user reported: (1) the auto-scale (10″ length fills the box so the drawing is always magnified as large as possible) was removed → fixed-scale made everything small; (2) the cross-section circle is now tiny vs its box. The user also flagged the head shape (already fixed) and wants exact glans:shaft proportion (the original ask). Goal: restore the old smart auto-scale, tuned to always use the available space, and keep the one real fix (unified thickness scale + old head shape).

The model (B — hybrid, space-maximised)

SIDE-VIEW (viewBox 480×168, CSS width:100% → resolution-independent):
 • thickness — ONE proportional scale for BOTH shaft Ø and glans Ø (→ glans:shaft EXACT):
     maxHalf    = SIDE_H/2 − (S_VPAD + DASH_OUT)           // band half-budget, room for the dashed outset
     THICK_BASE = 26 px/cm                                  // generous (typical Ø3.3 → ~86px full, the old look)
     thickScale = min(THICK_BASE, (maxHalf*2) / max(dLo, dgLo))   // grows with value; if the biggest Ø would
                                                            // overflow, scale BOTH down together → ratio still exact, never clips
     tShaftA=(dLo/2)*thickScale ; tGlansA=(dgLo/2)*thickScale ; glansLenA = tGlansA * 1.25  // OLD mushroom shape
 • length — AUTO-FILL the box width (10″ fills, short has a sensible floor):
     lenFrac     = clamp((lengthCm − lenMin)/(lenMax − lenMin), 0, 1)
     maxShaftLen = SIDE_W − X0 − RIGHT_MARGIN − GAP − glansLenA
     len         = MIN_SHAFT + lenFrac*(maxShaftLen − MIN_SHAFT)   // 10″→maxShaftLen (full) ; min→MIN_SHAFT

CROSS-SECTION (viewBox 240×240, CSS width:min(62%,250px)):
 • AUTO-FILL the box — the AFTER circle fills, BEFORE is proportional (→ after:before == dLo:d0 EXACT):
     csA = CS_MAX_R (≈100)         // after fills the box (was tiny)
     csB = csA * (d0 / dLo)        // before proportional → exact ratio

Decisions (locked via Q&A with user)

  • Model B (hybrid) + space-maximised — user: "B แต่สามารถปรับให้แสดงให้ได้เต็มพื้นที่เสมอ ใช้พื้นที่ได้คุ้มที่สุด".
  • Length auto-fills the box (10″ = full width; restored auto-scale) — "ขยายการแสดงผลให้ใหญ่ที่สุดเสมอ".
  • Thickness grows with value, ONE scale for shaft+glans → glans:shaft Ø exact; auto-shrinks both together only when the biggest Ø would overflow the band.
  • Cross-section auto-fills its box (the after-circle fills; before proportional) — restores "เคยจัดได้สวยงาม".
  • Old mushroom head shape (glansLen ∝ half-thickness) — already fixed, kept.
  • Responsive — SVG viewBox + width:100% + existing @media grid → best on every device, no per-device JS.
  • Accepted trade-off: length px-scale ≠ thickness px-scale (length auto-fills, girth grows at its own scale) — the old beloved behaviour the user re-requested.

Acceptance Criteria

  1. Length 10″ → the side-view shaft fills the box width (no wasted horizontal space); 2.5″ → visibly shorter.
  2. Increasing girth or glans cc → the drawing is visibly THICKER / the head visibly BIGGER (grows with value).
  3. glans-Ø-px ÷ shaft-Ø-px == dgLo ÷ dLo (one thickness scale) for any non-overflow config.
  4. The biggest config (max girth + 15cc head) does NOT overflow the band (thickScale auto-shrinks, ratio preserved).
  5. Cross-section AFTER circle fills its box (large, no wasted space); BEFORE radius ÷ AFTER radius == d0 ÷ dLo.
  6. Head = old mushroom shape (glansLen ∝ half-thickness).
  7. Renders well at desktop + mobile widths (SVG scales; rendered-pixel verified).

Risks / Open Questions

  • THICK_BASE tuning — 26 px/cm targets the old visual size; verify rendered-pixel at typical + extreme girth so it neither looks thin nor clips. Adjust the constant if needed.
  • Length not exact-to-girth — intentional (Model B). The filler effect is read from the after-vs-before dashed outlines + the numeric readouts, not the absolute length:girth px ratio.
  • 3D — unchanged; already real-cm + camera auto-frame (its own "fill the canvas"). Not in scope unless the user flags it.
