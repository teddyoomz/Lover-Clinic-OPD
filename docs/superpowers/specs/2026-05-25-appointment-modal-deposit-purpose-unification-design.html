Date: 2026-05-25 · Status: Approved by user (Q1=A · Q2=A · scope=create+edit · option-list=single-source · flip-away=choice-at-confirm) · Mockup (เปิดดูได้): docs/superpowers/specs/2026-05-25-appointment-modal-unification-mockup.html

Purpose / Problem

มี modal "สร้างนัดหมาย" ตัวเดียว (AppointmentFormModal.jsx) ที่เรียกจาก 4 ที่ แต่ปัจจุบัน section "รายละเอียดมัดจำ" โผล่เฉพาะตอนที่ caller ล็อก ประเภทเป็น deposit-booking (prop lockedAppointmentType) เท่านั้น — เมื่อผู้ใช้ เลือก "จองมัดจำ" ผ่าน radio (ในปฏิทินนัดหมาย / หน้าข้อมูลลูกค้า) section ไม่โผล่ ทำให้สร้างมัดจำจากที่นั่นไม่ได้. และช่อง "นัดมาเพื่อ" เป็น free-text (botox, filler…) ต่างจากปุ่ม "สร้างคิว" ใน Frontend ที่เป็น chip เลือกหลายตัว.

เป้าหมาย: (1) section มัดจำ auto-โผล่ทุกที่ที่เลือก "จองมัดจำ" ได้ (รวม create + edit); (2) "นัดมาเพื่อ" เป็น chip multi-select เหมือนสร้างคิว. แก้ที่ shared modal ตัวเดียว → มีผลทุก entry point.

State C · Edit — flip "จองมัดจำ" → ประเภทอื่น ขณะมีใบมัดจำผูก → confirm ตอนกดบันทึก
⚠ นัดนี้มีใบมัดจำผูกอยู่ (฿2,000)

คุณกำลังเปลี่ยนประเภทออกจาก "จองมัดจำ" — ต้องการจัดการใบมัดจำเดิมยังไง?

ลบใบมัดจำด้วย (ยกเลิก cascade) เก็บใบมัดจำไว้ ยกเลิก (ไม่บันทึก)

"ลบ" = reuse path ยกเลิกมัดจำเดิมของ DepositPanel (audit + reversal ครบ ไม่ลบดิบ). "เก็บ" = อัปเดตประเภทนัด ปล่อยใบมัดจำไว้ (ยังเห็นใน การเงิน → มัดจำ).

Decisions (locked via Q&A with user)

  • Q1 = A: เลือก "จองมัดจำ" (locked หรือ radio) → auto-โผล่ section + บังคับ ยอดมัดจำ>0 + บันทึกคู่ atomic เหมือน tab จองมัดจำเป๊ะ
  • Q2 = A: "นัดมาเพื่อ" = chip multi-select บังคับ ≥1 + "อื่นๆ" free-text; เก็บเป็น appointmentTo string เดิม (backward-compatible)
  • Scope = create + edit: section ทำงานทั้งสองโหมด
  • Option list = single source: แยก visitReasonOptions → constant กลาง ใช้ร่วม 3 ที่ (Rule C1)
  • Flip-away (edit, มีใบมัดจำ) = choice-at-confirm: ตอนกดบันทึกให้เลือก ลบ (cascade-cancel) / เก็บ / ยกเลิก — modal ไม่ลบเงินเงียบ

Acceptance Criteria

  1. Create + radio-เลือก "จองมัดจำ" → deposit section โผล่; เลือกประเภทอื่น → หาย
  2. Create จองมัดจำ → บังคับ ยอดมัดจำ>0; save เขียน be_deposits + be_appointments atomic (createDepositBookingPair)
  3. "นัดมาเพื่อ" = chip multi-select (10 ตัว + อื่นๆ); ไม่เลือกเลย → block save (scrollToError data-field="appointmentTo"); เก็บเป็น comma-string
  4. Edit นัดจองมัดจำที่มีใบมัดจำ → hydrate ฟิลด์มัดจำจากใบมัดจำเดิม; save = UPDATE ใบเดิม (ไม่สร้างซ้ำ)
  5. Edit + flip-to จองมัดจำ → save = สร้าง+ผูกใบมัดจำใหม่
  6. Edit + flip-away (มีใบมัดจำ) → confirm ลบ/เก็บ/ยกเลิก; "ลบ" ยกเลิกผ่าน cascade เดิม; "เก็บ" ปล่อยใบมัดจำไว้
  7. Edit hydrate "นัดมาเพื่อ" chips จาก string; legacy free-text → "อื่นๆ: …"
  8. Locked callers (tab จองมัดจำ / DepositPanel / Walk-in OPD-save) เหมือนเดิม 100% (Walk-in ล็อก no-deposit → ไม่มี section)
  9. visitReasonOptions = constant กลาง import โดย PatientForm + AdminDashboard + VisitPurposePicker
  10. Rule Q L2: real client-SDK e2e บน prod จริง (fixture TEST-) ยืนยัน create-pair + edit-update + flip paths; L1 hands-on by user post-deploy

Testing (Rule I + Q + N)

  • Rule I flow-simulate: radio→section→required→createDepositBookingPair payload · edit hydrate→update · flip-to→create · flip-away→confirm-delete/keep · chip↔appointmentTo round-trip · legacy free-text hydration
  • RTL: radio toggle แสดง/ซ่อน section · chip บังคับ block save · "อื่นๆ" เปิดกล่อง · flip-away confirm dialog 3 ปุ่ม
  • Rule Q L2: scripts/e2e-…mjs real client SDK บน prod (create จองมัดจำ จาก modal ปลดล็อก → ยืนยัน pair; edit update; flip delete via cascade) — fixture TEST-prefix + cleanup
  • AV invariant ใหม่: (a) deposit gate = effective type (ไม่ใช่ locked-only); (b) visitReasonOptions single-source (ห้าม inline copy ใหม่); (c) modal ไม่ลบ deposit นอกจากผ่าน cascade-cancel path
  • Rule N: targeted ระหว่างทำ; full suite ก่อน commit (แตะ PatientForm + AdminDashboard → ต้อง full)

Risks / Open Questions

  • update-deposit / cancel-deposit helper: edit-mode UPDATE + flip-away DELETE ต้องใช้ helper ที่ DepositPanel ใช้จริง — verify ชื่อ + shape ตอนเขียน plan (Rule Q-honest: ไม่เดา; ถ้าไม่มี update path ต้องออกแบบเพิ่ม)
  • Money safety: "ลบ" ต้อง route ผ่าน cascade-cancel เดิม (audit + reversal) — ห้าม deleteDoc ดิบ (audit-cascade-logic C16)
  • Linked-deposit lookup: edit ต้องหาใบมัดจำที่ผูกกับ appointment (โดย linkedAppointmentId / field จริง — verify ตอน plan)
  • Backward compat: appt เก่าที่ appointmentTo เป็น free-text ต้อง hydrate ไม่หาย (parseVisitPurposeText จัดการแล้ว — เพิ่ม test)
  • pickLater + บังคับนัดมาเพื่อ: chip บังคับเป็นอิสระจากลูกค้า → pickLater ยังทำงาน (purpose ≠ customer)
  • V21 / multi-reader: ย้าย visitReasonOptions แตะ PatientForm + AdminDashboard → full-suite + tests ของทั้งสอง
