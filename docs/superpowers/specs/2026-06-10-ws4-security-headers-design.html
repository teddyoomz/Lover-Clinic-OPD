WS4 — Security Response Headers + CSP

Date: 2026-06-10 · Status: Approved (user delegated judgment "เอาที่นายเห็นว่าเหมาะสมเลย") · Workstream: 4 of the Cybersecurity hardening (audit 2026-06-10)

Purpose / Problem

vercel.json shipped with NO headers block — the app sends zero security response headers. That leaves it open to clickjacking (no X-Frame-Options/frame-ancestors), MIME-sniffing (no nosniff), referrer leakage, and provides no CSP defense-in-depth against XSS for a PII-handling medical app. WS4 adds a global headers block. The one risky directive — CSP — is scoped to the app's REAL Firebase origins and L1-verified in a real browser on the production build before deploy (a wrong CSP would break the whole clinic, so verification is the gate).

Mockup Design

vercel.json · headers added (data shape)
"headers": [{ "source": "/(.*)", "headers": [
  { "key": "Content-Security-Policy", "value": "<see below>" },
  { "key": "Strict-Transport-Security",   "value": "max-age=63072000; includeSubDomains" },
  { "key": "X-Content-Type-Options",      "value": "nosniff" },
  { "key": "X-Frame-Options",             "value": "SAMEORIGIN" },
  { "key": "Referrer-Policy",             "value": "strict-origin-when-cross-origin" },
  { "key": "Permissions-Policy",          "value": "camera=(), microphone=(), geolocation=(), payment=(), usb=(), ..." },
  { "key": "X-DNS-Prefetch-Control",      "value": "off" }
]}]
Content-Security-Policy — scoped to the app's real surface (L1-verified non-breaking)
default-src 'self'; base-uri 'self'; object-src 'none';
script-src 'self' 'sha256-jhKhy…'(theme-flash) 'sha256-T+bx…'(body.ready);   ← HASHES, not 'unsafe-inline'
style-src 'self' 'unsafe-inline';        ← required: inline <style> + thousands of React inline styles
img-src 'self' data: blob: https:;       ← firebasestorage signed URLs + data:/blob: (PDF/canvas)
font-src 'self' data:;                   ← Sarabun = OS font (no Google Fonts)
media-src 'self' https: data: blob:;     ← staff-chat audio/video attachments
connect-src 'self' https://*.googleapis.com wss://*.googleapis.com https://*.firebaseio.com
            wss://*.firebaseio.com https://*.firebasestorage.app https://*.cloudfunctions.net
            https://www.gstatic.com;     ← broad-but-scoped: covers ALL Firebase endpoints, blocks exfil
worker-src 'self' blob:; manifest-src 'self'; frame-src 'self';
frame-ancestors 'self'; form-action 'self'; upgrade-insecure-requests

No 'unsafe-eval' (no eval/wasm in src or bundle — verified). No 'unsafe-inline' in script-src (the 2 inline scripts are hash-pinned). A drift test recomputes the hashes from index.html and fails the build if they fall out of sync with vercel.json.

Flow

Verify-before-deploy (CSP can't break prod)

scope CSP from real origins (grep eval/wasm/camera/fonts — none)→ compute SHA-256 of the 2 inline scripts (source===dist, stable)

→ L1: inject CSP meta into the prod build → vite preview → real browser → app renders 64KB DOM, inline scripts run (hashes OK), Firestore connects, ZERO console CSP violations

→ put final headers in vercel.json + hash-drift test (5/5)→ deploy vercel (build-only)→ curl prod → assert all 7 headers present

Decisions

  • CSP script-src = hashes, not 'unsafe-inline': real script-XSS protection; the 2 inline scripts (theme-flash + body.ready) are SHA-256-pinned; a drift test guards against silent breakage.
  • connect-src broad-but-scoped (*.googleapis.com): won't miss a Firebase endpoint (all under googleapis.com) while still blocking exfiltration to arbitrary domains.
  • L1-before-deploy is mandatory: a wrong CSP breaks the live clinic — verified on the production build in a real browser first.
  • Permissions-Policy disables camera/mic/geolocation/payment/usb: none used (no getUserMedia/capture in src).

Acceptance Criteria

  1. L1 (real browser, prod build): app renders, inline scripts run, Firestore connects, ZERO CSP violations. ✓ done
  2. vercel.json valid JSON; 7 headers; hash-drift + invariant test green (5/5). ✓ done
  3. Post-deploy: curl -I prod shows all 7 headers.
  4. No regression: full vitest still green (no src change).

Risks / Open Questions

  • Authenticated-only CSP break (residual): L1 tested the public landing (script/style/connect-firestore/inline). The authenticated app (storage images, chat media, PDF gen, FCM SW) uses the SAME already-proven directives (script/style 'self', connect *.googleapis.com, img https:, worker blob:) so the risk is low — but full authenticated L1 needs staff creds. Post-deploy: watch for any CSP-blocked feature; relax the specific directive if one surfaces (Rule A).
  • Inline-script drift: guarded by the hash-drift test (build fails if index.html inline scripts change without updating the CSP hashes).
