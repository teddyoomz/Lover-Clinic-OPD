Purpose / Problem

Three refinements to the public penile-filler size estimator, all user-requested:

  1. Initial glans (head) size is fixed. Real head-to-shaft ratio varies per person, but the baseline glans Ø is hard-wired to the shaft Ø. Users want to set their starting head size — with the default centered on today's balanced value, and still proportional to the chosen diameter.
  2. The 2D illustration is small while the screen has lots of empty space — it should use Desktop / iPad / mobile real-estate efficiently and read clearly.
  3. On mobile the order is wrong — controls sit above the illustration; the illustration should be on top, controls below.

Architecture

All changes are confined to the three filler files; no new dependency, no data, no API.

  • src/pages/FillerSimulator.jsx — NEW glansBaseRatio state (default 1.0) + slider control (between baseline and total-filler sections); pass baseGlansDiameterCm = glansBaseRatio × diameterFromGirth(baseGirthCm) into estimate(); grid → minmax(0,2fr) minmax(0,3fr) (40/60) with a 1fr single-column media query; CSS order on the controls/graphic panels so the graphic comes first ≤820px.
  • src/components/FillerGraphic2D.jsx — enlarge the side-view so the 2D fills the (wider) card: bigger anatomy as the hero, cross-section circle + legend kept but compact. Driven by the same est (single source); the head bulb already reads est.glans.visualLow.
  • src/lib/fillerStrings.js — NEW i18n keys: glansBase (TH "ขนาดหัวเริ่มต้น" / EN "Initial glans size") + small/normal/large markers (gSmall/gNormal/gLarge).
  • src/lib/fillerMath.js — no logic change (estimate() already accepts baseGlansDiameterCm); add a GLANS_BASE_RATIO range constant (min 0.75, max 1.25, default 1.0, step 0.05) for SSOT.

Decisions (locked via Q&A with user)

  • Q1 = B: Desktop illustration column wider — controls 40% / graphic 60% (biggest 2D while still side-by-side with the controls).
  • Q2 = Slider: Initial glans size is a smooth slider (เล็ก ↔ ปกติ ↔ ใหญ่), default centered, a ratio of the shaft Ø so it scales with the chosen diameter.
  • Mobile order (given): single-column → illustration on top, controls below; desktop unchanged (controls-left | graphic-right).
  • Glans ratio range: 0.75×–1.25× of shaft Ø, default 1.0× (= today's value), step 0.05.
  • No new result number for the head — it isn't part of condom/girth sizing; the slider affects the 2D/3D head bulb only.

Acceptance Criteria

  1. A "ขนาดหัวเริ่มต้น / Initial glans size" slider exists, default centered (ratio 1.0); at 1.0 the head Ø equals the shaft Ø (identical to current render).
  2. Changing the diameter/condom rescales the baseline head proportionally (ratio held); changing the ratio at a fixed diameter grows/shrinks the baseline head; the cc head-injection still adds on top.
  3. The slider changes the 2D AND 3D head bulb; condom + girth + diameter result cards are byte-identical regardless of the ratio (head excluded from condom sizing).
  4. Desktop: controls 40% / illustration 60%; the 2D visibly fills the wider card (no large empty band); no horizontal scroll.
  5. Phone + iPad-portrait (≤820px): single column with the illustration box ON TOP and the controls box BELOW; no horizontal scroll; the 2D is full-width and clearly larger than before.
  6. All copy stays formal in TH + EN (the new control included); pure-client (no Firestore/auth/PII); filler tests + build green; Rule Q L1 real-browser verified.

Risks / Open Questions

  • SVG enlargement vs viewBox: making the side-view bigger must not clip the corona/cross-section/legend at any card width; the exact viewBox + scale constants are worked out in the plan and verified live on Desktop/iPad/mobile widths.
  • Controls at 40% width: the condom <select> + sliders must stay comfortable in the narrower desktop column (they already wrap; verify the long condom option label doesn't overflow).
  • Glans ratio interaction with diameter mode: ratio multiplies diameterFromGirth(baseGirthCm) in BOTH condom-mode and Ø-mode, so the "scales with diameter" behavior holds either way.
