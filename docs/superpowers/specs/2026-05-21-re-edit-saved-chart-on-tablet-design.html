Spec — Re-edit a saved chart ON THE TABLET

Date 2026-05-21 · Status brainstormed + design APPROVED by user ("โอเค ลุยเลย") — HARD-GATE satisfied · Q1=A automatic object-level (raster fallback).

Problem. Clicking edit (✏️) on a saved chart opens the PC ChartCanvas ONLY — there is no way to send an EXISTING annotated chart back to the iPad to re-edit. New charts already show the "edit here / send to tablet" choice; edit should be consistent + the iPad must load the existing chart's prior annotations.

Goal

  • Edit (✏️) on a saved chart → open the EXISTING PcPairingModal (same as add-new): 🖥️ แก้ที่เครื่องนี้ / 📱 ส่งไปแก้ที่แท็บเล็ต.
  • Send-to-tablet for an existing chart → the iPad loads the prior annotations and the result merges back into the SAME chart slot.
  • Object-level when possible (prior strokes = movable/erasable), raster fallback when no re-editable fabricJson (legacy chart / pre-storage-deploy json upload denied).
  • Constraint: separate files; TreatmentFormPage stays untouched (ChartSection owns the relay, as today).

No-double-load invariant. When the reeditable json is the source, the canvas must NOT also loadTemplate the PNG (would bake annotations twice). The page resolves json FIRST; if reeditable → set initialFabricJson + skip raster. The canvas prefers initialFabricJson over templateImageUrl.

Flow

Data flow — PC → tablet → PC (the NEW editFabricJsonUrl leg in red)

1. PC ChartSection.handleEdit(idx) → set editingIdx=idx + canvasTemplate=charts[idx].template + pendingChart=charts[idx] → render PcPairingModal.
▼ user picks 📱 + a tablet
2. PC sendToTablet(tablet) → start({ tablet, template, patientLabel, templateDataUrl: pendingChart.dataUrl, editFabricJson: pendingChart.fabricJson, branchId }).
▼
3. PC hook start(): createChartEditSession (doc has editFabricJsonUrl:null) → uploadTransportImage('template', dataUrl) → if editFabricJson → uploadTransportJson('edit', json) (guarded) → updateChartEditSession({ templateImageUrl, editFabricJsonUrl }) in ONE patch (no flash).
▼ Firestore listener (instant-pop already fired on requested)
4. Tablet listener: resolve editFabricJsonUrl FIRST → downloadTransportJson → if isObjectLevelReeditable → setInitialFabricJson(json) + skip raster; ELSE → downloadTransportImageAsDataUrl(templateImageUrl) → setTemplateDataUrl.
▼
5. TabletChartCanvas: initialFabricJson reeditable → setDimensions(canvasW,canvasH) + loadFromJSON + relock object[0] + force white bg + baseline history (mirror PC ChartCanvas:69-79). ELSE → loadTemplate(templateImageUrl) (raster, current).
▼ user edits → ✓ บันทึก
6. Tablet onSave (UNCHANGED): PNG + fabricJson → updateChartEditSession({status:saved, resultImageUrl, resultFabricJsonUrl}).
▼
7. PC hook SAVED → onSaved({dataUrl, fabricJson, templateId, source}) → ChartSection.handleSave: editingIdx≥0 → replace the SAME slot (template = canvasTemplate, savedAt). Clear pendingChart+pendingTemplate.
