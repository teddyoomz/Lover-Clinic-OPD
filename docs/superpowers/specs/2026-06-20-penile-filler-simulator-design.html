Penile Filler Size Simulator — Design Spec

Date: 2026-06-20 · Status: Approved by user (Q1–Q5 + rev1 + rev2 locked) · Type: T2 customer-facing feature · Implementation plan: (writing-plans → next)

Purpose / Problem

คลินิกอยากมี "ของเล่นให้ความรู้" หน้าเดียวที่ส่งลิงก์ในแชทให้ลูกค้าทุกคนเปิดเองได้ เพื่อให้ลูกค้า นึกภาพออก + กะเกณได้คร่าวๆ ว่าถ้าฉีดฟิลเลอร์เพิ่มขนาด (penile girth augmentation ด้วย HA filler) กี่ cc แล้วขนาดจะเปลี่ยนประมาณไหน — และ ใส่ถุงยางไซส์ไหน / ใหญ่ขึ้นกี่ไซส์. ลูกค้าเลื่อนสไลเดอร์ แล้วกราฟิก (เหมือนจริง 2D หรือ 3D หมุนได้) ขยายสด + บอกขนาดใหม่เป็น "ช่วงประเมิน". เน้น กะเกณ ไม่ใช่รับประกัน. ไม่เก็บข้อมูลผู้ใช้ใดๆ.

ฐานความรู้ (research, อ้างอิงจริง): งานวิจัย HA penile girth augmentation รายงานรอบวงเพิ่มเฉลี่ย ~+2.5 ซม. ที่ ~16 cc (วัดตอนอ่อนตัว, พีค ~1 เดือน) ช่วง +1.5 ถึง +4 ซม. (≈0.13–0.19 ซม./cc); ฟิลเลอร์เพิ่มรอบวง ไม่เพิ่มความยาวจริง; อยู่ได้ ~6–24 เดือน, ยุบลงเหลือ ~55% ของพีคราว 1 ปี; ภาวะแทรกซ้อนต่ำ (ก้อน 2.2% / ติดเชื้อ 1% / ช้ำ 1.3%) และ HA ละลายกลับได้. (แหล่ง: PMC7230452 RCT · PubMed 20233296 · WJMH 2025 · Transl Androl Urol.) การแปลงไซส์ถุงยางอิง ISO 4074 nominal width + ไซส์จริงในตลาด (OneTouch TH 49/52/54/56 · Durex · Okamoto · ONE/MyONE 45–64).

Math engine — girth (ฝัง research; Approach A: geometry × k)

C0 = π · Ø₀                       // รอบวงเดิม (หรือมาจากถุงยาง: C0 = W₀/5)
C1_geo = √( C0² + 4π · V / L )    // cylinder-shell ideal (V=cc=cm³, L,C เป็น ซม.)
ΔC_geo = C1_geo − C0
ΔC_low  = k_realistic  · ΔC_geo   // k_realistic = 2.37
ΔC_high = k_optimistic · ΔC_geo   // k_optimistic = 3.32
C1_low/high = C0 + ΔC_low/high ;  Ø1 = C1/π
// k calibrate ที่ reference: ถุงยาง Regular 52mm (C0=10.4), L=11, V=16cc → band +2.0 ถึง +2.8 ซม.  (VERIFIED)
// length เกี่ยวในตัว (V/L): VERIFIED V16 → L8:+2.7 / L11:+2.0 / L15:+1.5 ; monotonic ตาม V

Math engine — condom conversion (EXACT, deterministic, ห้ามมั่ว)

width_mm    = girth_cm × 5            // ISO 4074 nominal width = ½ ของรอบวง
girth_cm    = width_mm / 5
diameter_cm = girth_cm / π = width_mm / (5π)
// snap: หาไซส์ในแลดเดอร์ที่ width ใกล้ required (= girth×5) ที่สุด, เสมอ → เลือกตัวใหญ่กว่า
// "+N ไซส์" = index(ไซส์ใหม่) − index(ไซส์เดิม) ในแลดเดอร์ด้านล่าง

Graphic & 3D mode (rev2)

  • Default = 2D เหมือนจริงเชิงกายวิภาค (FillerGraphic2D): silhouette แกน+ปลาย (glans)+corona + skin-tone shading; before เส้นประ + after ขอบแดง (ฟิลเลอร์); วงหน้าตัดประกอบ. ขยับสดตามคณิตศาสตร์. เบา เปิดเร็ว เสมอ.
  • 3D จริงหมุนได้ (Filler3D, Three.js/WebGL): โมเดล mesh กายวิภาค (cylinder แกน + glans rounded + corona), MeshStandardMaterial skin-tone, แสง 3 จุด, ลากหมุน (OrbitControls) + auto-rotate. radius ของแกน = ฟังก์ชันของ girth (C/π/2), ความยาว = L → ขยายสดตาม cc/L. lazy-load เฉพาะตอนกดปุ่ม 3D (ไม่ถ่วง initial load). WebGL ไม่รองรับ/โหลด fail → ซ่อนปุ่ม, เหลือ 2D.
  • Realism = "ลงรายละเอียด" (rev2) เชิงกายวิภาคทางการแพทย์ — รูปทรง/ผิว/แสงเงาเหมือนจริง แต่ เฟรมเป็นโมเดลการแพทย์ ไม่ sexualized (ไม่มีฉาก/บริบททางเพศ). ดู Risks เรื่อง content policy.
  • Both views ใช้ fillerMath ตัวเดียวกัน (single source of truth) — 2D scale + 3D mesh radius มาจากผลลัพธ์เดียวกัน → ตรงกันเสมอ.

Decisions (locked via Q&A + revisions)

  • Q1 = C: กราฟิก "รวม 2 มุม + ตัวเลข".
  • Q2 = ความยาวเป็น input ที่มีผลต่อรอบวง (ผ่าน V/L): สไลเดอร์ความยาว (ซม./นิ้ว) + geometry×k แต่ ไม่ทำนายความยาวเพิ่ม.
  • Q3 = ช่วงล้วน: realistic–optimistic band (หมายเหตุระยะเวลา 6–24 ด.).
  • Q4 = กรอกเอง + ไม่เก็บอะไรเลย: ephemeral ล้วน · ไม่เขียน Firestore · ไม่เก็บ PII.
  • Q5 = static public link.
  • Rev1: 6–24 เดือน · ความยาว ซม./นิ้ว · baseline 2 โหมด (Ø / ถุงยาง) · ผลลัพธ์โชว์ไซส์ถุงยาง + กี่ไซส์.
  • Rev2-a = 3D จริงหมุนได้ (WebGL/Three.js): default = 2D เหมือนจริง + ปุ่มสลับโหมด 3D (lazy-load, fallback 2D).
  • Rev2-b = realism "ลงรายละเอียด": กายวิภาคเหมือนจริง (รูปทรง/ผิว/แสงเงา) แบบโมเดลการแพทย์ ไม่ sexualized.

Acceptance Criteria

  1. เปิด ?play=filler โดยไม่ล็อกอิน → แสดงผลทันที, ไม่มี "Invalid Link" flash; โหมด 2D โหลดทันที (ไม่รอ three).
  2. เลื่อน/เปลี่ยนค่าทุกตัว → กราฟิก (2D หรือ 3D) + การ์ดผล 3 ใบ (รอบวง / Ø / ไซส์ถุงยาง) อัปเดตสด.
  3. กดปุ่ม 3D → lazy-load three → โมเดล 3D เรนเดอร์, ลากหมุนได้, ขยาย radius ตาม cc/L สด; กลับ 2D ได้.
  4. WebGL ไม่รองรับ → ซ่อนปุ่ม 3D, ใช้ 2D ได้ปกติ (ไม่ crash).
  5. three ไม่อยู่ใน initial bundle (verify: dynamic import; bundle หน้าอื่นไม่โต).
  6. ความยาว toggle ซม.↔นิ้ว แปลงถูก (1 นิ้ว = 2.54 ซม.) ค่าภายในเก็บเป็น ซม.
  7. baseline toggle 2 โหมดให้ C0 เดียวกัน (ถุงยาง 52 → รอบวง 10.4 / Ø 3.3 = เลื่อน Ø 3.3).
  8. condom conversion เป๊ะ: width=รอบวง×5; Ø=รอบวง/π; ถุงยาง→C0→snap กลับได้ไซส์เดิมทุกรุ่น; snap nearest-tie-larger; "+N ไซส์" = index diff.
  9. girth model ผ่าน anchor: ถุงยาง 52 (C0=10.4), L=11, V=16 → รอบวง ≈ 12.4–13.2 (+2.0/+2.8); V=10 → 11.7–12.2.
  10. length-sensitivity ถูกทิศ + monotonic ตาม V + band low ≤ high + ทุกเลข round; V>20 เตือน; clamp V≤30.
  11. 2D และ 3D ใช้ผลจาก fillerMath ตัวเดียวกัน (ขนาดตรงกัน).
  12. disclaimer ไทย+อังกฤษ เห็นชัด; ไม่ทำนายความยาวเพิ่ม; โน้ต girth-not-length + 6–24 ด. + ถุงยางประมาณ.
  13. ไม่มีการเขียน Firestore / ไม่มี PII.
  14. ธีม dark fire/ember, อ่านง่ายทั้ง desktop/mobile.

Testing (Rule Q — honest)

  • Unit — condom conversion (เป๊ะ): width=girth×5 / girth=width/5 / Ø=girth/π เทียบค่าจริง; snap nearest-tie-larger; ถุงยาง→C0→snap-back ทุกรุ่น; "+N ไซส์"; worked examples.
  • Unit — girth model: anchor (52mm·L11·V16→+2.0/+2.8), monotonic ตาม V, length-sensitivity, band ordering, clamp/warn, round, adversarial (0/ติดลบ/ใหญ่สุด/NaN).
  • Unit — unit toggle + 3D mesh dims: ซม.↔นิ้ว round-trip (×/÷ 2.54); mesh radius = girthToRadius(C) ถูก (C/(2π)), length=L → pure fn ทดสอบได้.
  • L1 จริง (Claude Preview / Rule S): เปิดหน้าจริง → เลื่อนสไลเดอร์ + สลับ baseline 2 โหมด + toggle นิ้ว + กดโหมด 3D → โมเดลเรนเดอร์ + ลากหมุน + ขยายตาม cc → ยืนยันตรงเครื่องคำนวณ. (dev server รัน Three.js ได้จริง → 3D L1 ทำได้เต็ม). pure client → ไม่มี Rule Q gap.
  • ไม่ต้อง L2 (ไม่มี Firestore query).

Risks / Open Questions

  • Content policy ตอนแชร์ (rev2 — สำคัญ): ภาพ realistic ลงรายละเอียด อาจโดน auto-moderation ของ LINE/FB ตอนแชร์ลิงก์/พรีวิว. ทางกัน: เฟรมเป็น "โมเดลการแพทย์" ไม่ sexualized + disclaimer + พิจารณา "แตะเพื่อดูภาพจำลอง" reveal gate (default บังภาพ, ผู้ใช้แตะเปิด) + OG/preview image เป็นโลโก้คลินิก ไม่ใช่โมเดล. (ยืนยันแนวทาง gate ตอน plan)
  • WebGL/perf บนมือถือ low-end: 3D อาจหน่วง/ไม่รองรับ → ปุ่ม 3D ซ่อนเมื่อไม่มี WebGL, mesh poly ต่ำ, จำกัด pixelRatio, dispose ตอนออกโหมด.
