Penile Filler Size Simulator — Design Spec

Date: 2026-06-20 · Status: Approved by user (Q1–Q5 locked) · Type: T2 customer-facing feature · Implementation plan: (writing-plans → next)

Purpose / Problem

คลินิกอยากมี "ของเล่นให้ความรู้" หน้าเดียวที่ส่งลิงก์ในแชทให้ลูกค้าเปิดเองได้ทุกคน เพื่อให้ลูกค้า นึกภาพออก + กะเกณได้คร่าวๆ ว่าถ้าฉีดฟิลเลอร์เพิ่มขนาด (penile girth augmentation ด้วย HA filler) กี่ cc แล้วขนาดจะเปลี่ยนประมาณไหน. ลูกค้าเลื่อนสไลเดอร์ (ความยาว / เส้นผ่านศูนย์กลางเดิม / จำนวน cc) แล้วกราฟิกขยายสด พร้อมบอกขนาดรอบวง/เส้นผ่านศูนย์กลางใหม่เป็น "ช่วงประเมิน" — เน้น กะเกณ ไม่ใช่รับประกัน. ไม่เก็บข้อมูลผู้ใช้ใดๆ.

ฐานความรู้ (research, อ้างอิงจริง): งานวิจัย HA penile girth augmentation รายงานรอบวงเพิ่มเฉลี่ย ~+2.5 ซม. ที่ ~16 cc (วัดตอนอ่อนตัว, พีค ~1 เดือน) ช่วง +1.5 ถึง +4 ซม. (≈0.13–0.19 ซม./cc); ฟิลเลอร์เพิ่มรอบวง ไม่เพิ่มความยาวจริง; อยู่ได้ ~12–24 เดือน, ยุบลงเหลือ ~55% ของพีคราว 1 ปี; ภาวะแทรกซ้อนต่ำ (ก้อน 2.2% / ติดเชื้อ 1% / ช้ำ 1.3%) และ HA ละลายกลับได้. (แหล่ง: PMC7230452 RCT 18 เดือน · PubMed 20233296 · WJMH 2025 · Transl Androl Urol complications review.)

Math engine (ฝัง research — Approach A: geometry × k)

C0 = π · Ø₀                       // รอบวงเดิม จากเส้นผ่านศูนย์กลางเดิม
C1_geo = √( C0² + 4π · V / L )    // cylinder-shell ideal (V=cc=cm³, L=ความยาว ซม.)
ΔC_geo = C1_geo − C0
ΔC_low  = k_realistic  · ΔC_geo   // k_realistic ≈ 2.15
ΔC_high = k_optimistic · ΔC_geo   // k_optimistic ≈ 3.00
C1_low/high  = C0 + ΔC_low/high
Ø1_low/high  = C1_low/high / π
// k calibrate ที่ reference Ø₀=3.0 (C0=9.4), L=11, V=16cc → band +2.0 ถึง +2.8 ซม.
// length เกี่ยวในตัว (V/L): ฐานสั้น → ΔC เยอะกว่า ตามที่ user ชี้
  • ทำไม × k: geometry เปล่าทำนาย ~+0.6 ซม./10cc แต่ผลจริงในงานวิจัย ~+2.5 ซม./16cc → คูณ k (~2–3×) ให้ตรงความจริง. k เป็น calibration constant เดียว, ยึด anchor test เป็นตัวล็อก.
  • ผลโชว์เป็น "ช่วง" (Q3): realistic = k_realistic, optimistic = k_optimistic. ไม่มี toggle พีค/อยู่ตัว — มีหมายเหตุบรรทัดเดียว "อยู่ ~12–24 เดือน ยุบลงบ้าง".
  • Clamp + warn: V ≤ 30 (hard); V > 20 → คำเตือน soft-cap; ถ้า optimistic Δ เกินช่วงงานวิจัย (เช่น ΔØ > ~1.5 ซม.) → โน้ต "เกินช่วงที่งานวิจัยรองรับ / เพดานความยืดผิวหนัง". round ทุกเลขที่โชว์.
  • ไม่ทำนายความยาวเพิ่ม (Q2): ความยาวเป็น input (ตัวหาร L) เท่านั้น.

Decisions (locked via Q&A with user)

  • Q1 = C: กราฟิก "รวม 2 มุม + ตัวเลข" — ภาพด้านข้าง + วงหน้าตัด + การ์ดตัวเลข.
  • Q2 = ความยาวเป็น input ที่มีผลต่อรอบวง (ผู้ใช้ยืนยันว่า length เกี่ยว ผ่าน V/L): สไลเดอร์ความยาวปรับได้ + เครื่องคำนวณใช้ geometry×k (length matters) แต่ ไม่ทำนายความยาวเพิ่ม + โน้ต "ฟิลเลอร์เพิ่มรอบวง ไม่เพิ่มความยาว".
  • Q3 = ช่วงล้วน: realistic–optimistic band, ไม่มี toggle พีค/อยู่ตัว (ใช้หมายเหตุระยะเวลาบรรทัดเดียวแทน).
  • Q4 = กรอกเอง + ไม่เก็บอะไรเลย: สไลเดอร์เริ่มที่ค่าเฉลี่ย ปรับเองได้ · ephemeral ล้วน · ไม่เขียน Firestore · ไม่เก็บชื่อ/เบอร์.
  • Q5 = static public link: เส้นเดียว ส่งในแชท เปิดได้ทุกคน ไม่ล็อกอิน.

Acceptance Criteria

  1. เปิด ?play=filler โดยไม่ล็อกอิน → หน้าแสดงผลทันที (มี/ไม่มีแบรนด์คลินิกก็ render ได้), ไม่มี "Invalid Link" flash.
  2. เลื่อนสไลเดอร์ทั้ง 3 → กราฟิก (ด้านข้าง + วงหน้าตัด) + การ์ดตัวเลข (รอบวง/Ø ใหม่เป็นช่วง) อัปเดตสดทันที.
  3. เครื่องคำนวณผ่าน anchor: Ø₀=3.0, L=11, V=16 → รอบวงใหม่ ≈ 11.4–12.2 ซม. (+2.0 ถึง +2.8).
  4. length-sensitivity ถูกทิศ: cc เท่ากัน, L น้อยลง → ΔC มากขึ้น (เป็น monotonic).
  5. monotonic ตาม cc: V มากขึ้น → C1 มากขึ้นเสมอ. band เรียงถูก (low ≤ high). ทุกเลขถูก round.
  6. V > 20 → คำเตือน soft-cap; V ถูก clamp ≤ 30; ค่าเกินช่วงงานวิจัย → โน้ตเตือน.
  7. มี disclaimer ไทย+อังกฤษ มองเห็นชัดทุกครั้ง; ไม่ทำนายความยาวเพิ่ม; มีโน้ต girth-not-length + ระยะเวลา.
  8. ไม่มีการเขียน Firestore / ไม่มี PII (verify: ไม่มี import backendClient/Firestore write ในไฟล์ใหม่).
  9. ธีม dark fire/ember, ไม่มีสีแดงบนชื่อ/HN (N/A — ไม่มีชื่อผู้ป่วย), อ่านง่ายทั้ง desktop/mobile.

Testing (Rule Q — honest)

  • Unit (เครื่องคำนวณ pure): anchor (16cc→+2.5-ish band), monotonic ตาม cc, length-sensitivity (สั้น→เยอะกว่า), band ordering, clamp V≤30, warn V>20, round, adversarial (0/ติดลบ/ใหญ่สุด/NaN). = runnable check ของ logic.
  • L1 จริง (Claude Preview / Rule S): เปิดหน้าจริงบน dev server → เลื่อนสไลเดอร์ → ยืนยันกราฟิก + ตัวเลขขยับถูก + ค่าตรงเครื่องคำนวณ. เป็น pure client → ไม่มี Rule Q gap (ไม่มี DB/auth/compound-query ให้ verify) — L1 ทำได้เต็ม.
  • ไม่ต้อง L2 (ไม่มี Firestore query).

Risks / Open Questions

  • k calibration: k เป็นค่าประมาณ (anchor งานวิจัยใช้ baseline เฉลี่ย; งานวิจัยไม่ได้ stratify ตามความยาว). L-sensitivity เป็นทิศทางจาก geometry — ถูกทิศ + กะเกณได้ แต่ขนาดความชันยังไม่มีข้อมูล validate. โชว์เป็น "ช่วงประเมิน" + disclaimer ชดเชย. k ปรับได้ภายหลังโดยไม่กระทบโครงสร้าง.
  • แบรนด์คลินิก: ถ้าอยากให้โลโก้/ชื่อคลินิก dynamic ต้อง anon-read clinic_settings/main (ตรวจว่า rule อนุญาต anon read หลัง WS1 security hardening ก่อน). MVP = ธีม fire ในตัว + ชื่อคลินิก hardcode/bundled → ไม่ต้องแตะ rules. (decision เล็ก — ทำ bundled ก่อน)
  • เนื้อหาอ่อนไหว: หน้าเปิด public — โทนต้องคลินิก/สุภาพ (schematic, ไม่ photoreal), disclaimer ชัด. ไม่มี input PII → ความเสี่ยง privacy ต่ำ.
  • ชื่อ route: ?play=filler เป็น default; ถ้าอยากได้ path สวยกว่า (เช่น /simulator) ปรับได้ตอน plan.
