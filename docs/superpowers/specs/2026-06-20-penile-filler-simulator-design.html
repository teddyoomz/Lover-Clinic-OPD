Penile Filler Size Simulator — Design Spec

Date: 2026-06-20 · Status: Approved by user (Q1–Q5 + revision round 1 locked) · Type: T2 customer-facing feature · Implementation plan: (writing-plans → next)

Purpose / Problem

คลินิกอยากมี "ของเล่นให้ความรู้" หน้าเดียวที่ส่งลิงก์ในแชทให้ลูกค้าทุกคนเปิดเองได้ เพื่อให้ลูกค้า นึกภาพออก + กะเกณได้คร่าวๆ ว่าถ้าฉีดฟิลเลอร์เพิ่มขนาด (penile girth augmentation ด้วย HA filler) กี่ cc แล้วขนาดจะเปลี่ยนประมาณไหน — และ ใส่ถุงยางไซส์ไหน / ใหญ่ขึ้นกี่ไซส์. ลูกค้าเลื่อนสไลเดอร์ แล้วกราฟิกขยายสด + บอกขนาดใหม่เป็น "ช่วงประเมิน". เน้น กะเกณ ไม่ใช่รับประกัน. ไม่เก็บข้อมูลผู้ใช้ใดๆ.

ฐานความรู้ (research, อ้างอิงจริง): งานวิจัย HA penile girth augmentation รายงานรอบวงเพิ่มเฉลี่ย ~+2.5 ซม. ที่ ~16 cc (วัดตอนอ่อนตัว, พีค ~1 เดือน) ช่วง +1.5 ถึง +4 ซม. (≈0.13–0.19 ซม./cc); ฟิลเลอร์เพิ่มรอบวง ไม่เพิ่มความยาวจริง; อยู่ได้ ~6–24 เดือน, ยุบลงเหลือ ~55% ของพีคราว 1 ปี; ภาวะแทรกซ้อนต่ำ (ก้อน 2.2% / ติดเชื้อ 1% / ช้ำ 1.3%) และ HA ละลายกลับได้. (แหล่ง: PMC7230452 RCT · PubMed 20233296 · WJMH 2025 · Transl Androl Urol.) การแปลงไซส์ถุงยางอิง ISO 4074 nominal width + ไซส์จริงในตลาด (OneTouch TH 49/52/54/56 · Durex · Okamoto · ONE/MyONE 45–64).

Math engine — girth (ฝัง research; Approach A: geometry × k)

C0 = π · Ø₀                       // รอบวงเดิม (หรือมาจากถุงยาง: C0 = W₀/5)
C1_geo = √( C0² + 4π · V / L )    // cylinder-shell ideal (V=cc=cm³, L,C เป็น ซม.)
ΔC_geo = C1_geo − C0
ΔC_low  = k_realistic  · ΔC_geo   // k_realistic = 2.37
ΔC_high = k_optimistic · ΔC_geo   // k_optimistic = 3.32
C1_low/high = C0 + ΔC_low/high ;  Ø1 = C1/π
// k calibrate ที่ reference: ถุงยาง Regular 52mm (C0=10.4), L=11, V=16cc → band +2.0 ถึง +2.8 ซม.  (VERIFIED)
// length เกี่ยวในตัว (V/L): VERIFIED V16 → L8:+2.7 / L11:+2.0 / L15:+1.5 ; monotonic ตาม V

Math engine — condom conversion (EXACT, deterministic, ห้ามมั่ว)

width_mm    = girth_cm × 5            // ISO 4074 nominal width = ½ ของรอบวง
girth_cm    = width_mm / 5
diameter_cm = girth_cm / π = width_mm / (5π)
// snap: หาไซส์ในแลดเดอร์ที่ width ใกล้ required (= girth×5) ที่สุด, เสมอ → เลือกตัวใหญ่กว่า
// "+N ไซส์" = index(ไซส์ใหม่) − index(ไซส์เดิม) ในแลดเดอร์ด้านล่าง

Decisions (locked via Q&A + revision)

  • Q1 = C: กราฟิก "รวม 2 มุม + ตัวเลข" — ภาพด้านข้าง + วงหน้าตัด + การ์ดตัวเลข.
  • Q2 = ความยาวเป็น input ที่มีผลต่อรอบวง (ผ่าน V/L): สไลเดอร์ความยาวปรับได้ (ซม./นิ้ว) + geometry×k (length matters) แต่ ไม่ทำนายความยาวเพิ่ม + โน้ต girth-not-length.
  • Q3 = ช่วงล้วน: realistic–optimistic band, ไม่มี toggle พีค/อยู่ตัว (หมายเหตุระยะเวลา 6–24 ด. บรรทัดเดียว).
  • Q4 = กรอกเอง + ไม่เก็บอะไรเลย: ephemeral ล้วน · ไม่เขียน Firestore · ไม่เก็บ PII.
  • Q5 = static public link: เส้นเดียว ส่งในแชท เปิดได้ทุกคน ไม่ล็อกอิน.
  • Rev1-a: ระยะเวลา 6–24 เดือน.
  • Rev1-b: ความยาวเลือกหน่วย ซม. หรือ นิ้ว ได้ (toggle).
  • Rev1-c: ขนาดเดิมเลือกได้ 2 โหมด — เลื่อน Ø ตรงๆ หรือ กดไซส์ถุงยาง → แปลงเป็น Ø (ทั้งคู่ → C0 เดียวกัน).
  • Rev1-d: ผลลัพธ์โชว์ ไซส์ถุงยางใหม่ + เพิ่มกี่ไซส์ ด้วยการแปลงคณิตศาสตร์เป๊ะ (verified).

Acceptance Criteria

  1. เปิด ?play=filler โดยไม่ล็อกอิน → แสดงผลทันที, ไม่มี "Invalid Link" flash.
  2. เลื่อน/เปลี่ยนค่าทุกตัว → กราฟิก + การ์ดผล 3 ใบ (รอบวง / Ø / ไซส์ถุงยาง) อัปเดตสด.
  3. ความยาว toggle ซม.↔นิ้ว แปลงค่าถูก (1 นิ้ว = 2.54 ซม.) ค่าภายในเก็บเป็น ซม.
  4. baseline toggle 2 โหมดให้ C0 เดียวกัน: เลือกถุงยาง 52 → รอบวง 10.4 / Ø 3.3; เลื่อน Ø 3.3 → ได้ผลเท่ากัน.
  5. condom conversion เป๊ะ: width_mm = รอบวง×5; Ø = รอบวง/π; ถุงยาง→C0→snap กลับได้ไซส์เดิมทุกรุ่น.
  6. snap rule = nearest width, tie→larger; "+N ไซส์" = ผลต่าง index ในแลดเดอร์ 8 รุ่น.
  7. girth model ผ่าน anchor: ถุงยาง 52 (C0=10.4), L=11, V=16 → รอบวงใหม่ ≈ 12.4–13.2 (+2.0/+2.8); V=10 → 11.7–12.2.
  8. length-sensitivity ถูกทิศ (cc เท่ากัน, L น้อยลง → ΔC มากขึ้น) + monotonic ตาม V + band low ≤ high + ทุกเลข round.
  9. V > 20 → คำเตือน soft-cap; clamp V ≤ 30; ค่าเกินช่วงงานวิจัย → โน้ตเตือน.
  10. disclaimer ไทย+อังกฤษ เห็นชัด; ไม่ทำนายความยาวเพิ่ม; โน้ต girth-not-length + 6–24 ด. + ถุงยางเป็นค่าประมาณ.
  11. ไม่มีการเขียน Firestore / ไม่มี PII (verify: ไม่มี import backendClient/Firestore write).
  12. ธีม dark fire/ember, อ่านง่ายทั้ง desktop/mobile.

Testing (Rule Q — honest)

  • Unit — condom conversion (เป๊ะ, ตามคำสั่ง "เทสมาด้วย"): width=girth×5 / girth=width/5 / Ø=girth/π เทียบค่าจริง (girth 9–14 → width 45–70, Ø 2.86–4.46); snap nearest-tie-larger; ถุงยาง→C0→snap-back consistency ทุกรุ่น; "+N ไซส์" index diff; worked examples (10.5→Regular, 12.0→XL60).
  • Unit — girth model: anchor (52mm·L11·V16→+2.0/+2.8), monotonic ตาม V, length-sensitivity (สั้น→เยอะกว่า), band ordering, clamp V≤30, warn V>20, round, adversarial (0/ติดลบ/ใหญ่สุด/NaN).
  • Unit — unit toggle: ซม.↔นิ้ว round-trip (×/÷ 2.54).
  • L1 จริง (Claude Preview / Rule S): เปิดหน้าจริง → เลื่อนสไลเดอร์ + สลับ 2 โหมด baseline + toggle นิ้ว → ยืนยันกราฟิก + การ์ด 3 ใบขยับถูก ตรงเครื่องคำนวณ. เป็น pure client → ไม่มี Rule Q gap.
  • ไม่ต้อง L2 (ไม่มี Firestore query).

Risks / Open Questions

  • k calibration: k เป็นค่าประมาณ (งานวิจัยไม่ได้ stratify ตามความยาว); L-sensitivity ถูกทิศจาก geometry. โชว์เป็น "ช่วงประเมิน" + disclaimer ชดเชย; ปรับ k ได้ภายหลังไม่กระทบโครงสร้าง.
  • ถุงยาง = ค่าประมาณ: ISO ±2mm + ลาเท็กซ์ยืด (1 ไซส์ครอบคลุมรอบวง ~±0.5–0.75 ซม.) → เป็น "ไซส์เริ่มต้นที่ควรลอง" ไม่ใช่การรับประกันฟิต. แลดเดอร์ใช้ชุด mainstream 8 รุ่น (design choice, document แล้ว) — "+N ไซส์" ขึ้นกับแลดเดอร์นี้.
  • flaccid vs erect: งานวิจัยวัดรอบวงตอน "อ่อนตัว" แต่ถุงยางอิงรอบวงตอน "แข็งตัว" — ค่าเป็นการกะเกณข้ามสภาวะ (ไม่มีข้อมูล erect-gain ในงานวิจัย). disclaimer + "ค่าประมาณ" ครอบคลุม. (Rule Q-honest: ส่วน geometry/แปลงถุงยาก = เป๊ะ verify ได้; ส่วน efficacy ฟิลเลอร์ = ประมาณการมีงานวิจัยรองรับ.)
  • "Magnum" ≠ XL: ชื่อการตลาดไม่ตรง width (Trojan Magnum shaft ~54mm). แลดเดอร์ยึด width จริง ไม่ยึดชื่อ.
  • ชื่อ route: ?play=filler default; ปรับเป็น path สวยกว่าได้ตอน plan.
