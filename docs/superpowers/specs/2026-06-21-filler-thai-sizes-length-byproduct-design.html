Date: 2026-06-21 · Status: Approved by user (Q&A locked) · Builds on: the 2026-06-21 closest-to-real recalibration (shipped) · Plan: docs/superpowers/plans/2026-06-21-filler-thai-sizes-length-byproduct.html

Purpose / Problem

Two owner-driven refinements to the just-shipped ?play=filler simulator. The math (girth k-ladder, glans, defaults) stays exactly as shipped — only the condom mapping + one new result row change.

  1. Thai-familiar condom dropdown. The current INPUT dropdown uses the real MyONE 9-rung ladder (45/47/49/51/53/55/57/60/64). Thai customers don't recognize 51/53 — in Thailand 52 is the everyday standard. The dropdown (where a customer selects their CURRENT size) must use sizes Thai customers recognize, extended up through the genuinely-largest sizes sold in the world so any customer can state their real current size.
  2. Result = closest computed size (mm), not a ladder rung. The condom RESULT must show the size closest to the actual computed girth (raw nominal width, rounded to the nearest mm) as a range — NOT floored to a retail rung. The customer reads the number and picks the closest real condom themselves. (Owner, verbatim: "เอา size มาตรฐานมาอ้างอิงแค่ใช้ตรง dropdown ... แต่ตรงที่แสดงผล ให้แสดงผลไซส์ใกล้ที่คำนวนได้ที่สุด แล้วให้คนที่รู้ข้อมูลนี้ ไปเลือก size ถุงยางเอง".) This also removes the old "clamp at the top rung" problem.
  3. Surface the length by-product. Filler adds FLACCID length (an anti-retraction splint, not true erect elongation). Today that fact is buried in the footnote — surface it as a dedicated result box WITH the warning (owner: "แสดงผลพลอยได้ที่เป็นความยาวด้วยพร้อมคำเตือนเหมือนตอน spec").

Amendments — systematic-debugging 2026-06-21: (a) cc range raised back to [5, 50] (owner). (b) the length by-product is NOT a flat 1.6 — research (PMC9809476 + PMC8987147: flaccid length +2.55 cm peak/1mo → +1.65 cm durable/12mo at ~15–21 mL) shows it is dose-dependent + saturating. Modeled as a varying durable–peak range ~+{low} – +{high} ซม. = MAX · (1 − e^(−shaftCc / 10)) (durable MAX 2.0 cm / peak 3.0 cm); 0 when no SHAFT filler (glans filler does not splint the shaft). A result that ignored every input was not a result.

Architecture

src/lib/fillerMath.js

  • CONDOM_LADDER → real-world widths [45,49,52,54,56,58,60,64,69,72] with Thai-descriptor labels (กระชับพิเศษ/กระชับ/มาตรฐาน/ใหญ่/ใหญ่พิเศษ/…/ใหญ่สุดในโลก). 45–56 = Thai retail (Durex/Onetouch/Okamoto); 58–72 = global large (MyONE 58/60/64, Pasante Super King 69, My.Size Pro 72 = world max). Integer-fraction values NOT needed (widths are integers) but labels are plain data — still hidden by the obfuscator only where numeric.
  • NEW FLACCID_LENGTH_GAIN_CM = 16 / 10 (= 1.6; integer-fraction so the obfuscator's numbersToExpressions hides it).
  • estimate(...) result fields: add condomWidthLow = Math.round(c1Low·5), condomWidthHigh = Math.round(c1High·5), lengthGainCm = FLACCID_LENGTH_GAIN_CM. Keep condom0 (base = selected). The floored condomLow/condomHigh/sizesUp* are no longer read by the UI — keep or drop (drop to stay lean; check no other reader).
  • condomForGirth / condomIndexForGirth stay (still map the dropdown selection → index, and the base width); they're no longer used for the RESULT. The "beyond" field is already always false.

src/pages/FillerSimulator.jsx

  • Default condomIdx → index of the 52 rung (index 2 in the new ladder).
  • Condom hero ResultCard → render {est.condomWidthLow} – {est.condomWidthHigh} มม. (single number when equal) + sub-line t('pickClosest'). Remove the oldVal (เดิม) + the delta (sizesUp) on the condom card only.
  • Add a length box DIRECTLY AFTER the condom hero (above รอบวง/เส้นผ่าศูนย์กลาง): label t('resLength'), value ~+{r1(est.lengthGainCm)} ซม., warning sub-line t('lengthWarn').
  • REMOVE from render: the range-legend line (t('rangeLegend')) AND the note line (t('note')). Keep the bottom t('disclaimer').

src/lib/fillerStrings.js

  • NEW keys (TH+EN): pickClosest ("เลือกถุงยางไซส์ใกล้เคียงค่านี้" / "pick a condom closest to this"), resLength ("ความยาว (ตอนอ่อน)" / "length (flaccid)"), lengthWarn ("พยุงไม่ให้หด ไม่ใช่ยืดจริง" / "anti-retraction splint, not true elongation").

Decisions (locked via Q&A with user)

  • Q1 = comprehensive world ladder: dropdown [45,49,52,54,56,58,60,64,69,72] — Thai-familiar middle + global-large up to 72 mm. Default rung = 52 (มาตรฐาน). Labels = number + Thai descriptor on anchors (กระชับพิเศษ/กระชับ/มาตรฐาน/ใหญ่/ใหญ่พิเศษ); plain number for the rest. NO "ใหญ่สุดในโลก" on 72 (owner revision).
  • Q2 = range mm only: condom result shows the computed nominal-width range "{low} – {high} มม." + "เลือกไซส์ใกล้เคียงเอง". No เดิม, no "+N ขนาด", no ladder floor/clamp.
  • Q3 = length box right after the condom box: the length by-product box is placed DIRECTLY AFTER the condom hero (above รอบวง / เส้นผ่าศูนย์กลาง), with the warning sub-line. Value fixed ~+1.6 cm.
  • Owner revision — remove from the screen: the range-legend ("ค่าซ้าย = ระยะคงตัว · ค่าขวา = ขนาดใหญ่ที่สุด(ช่วงแรก)") AND the note ("ฟิลเลอร์เพิ่มเส้นรอบวงและส่วนหัว · คงอยู่ประมาณ 12–18 เดือน… ±~1 ซม.") are NO LONGER RENDERED. The bottom legal disclaimer stays.
  • Unchanged: theme, layout, 2D/3D graphics, sliders, the girth/diameter/glans math + defaults, disclaimer.

Acceptance Criteria

  1. Dropdown lists exactly 45,49,52,54,56,58,60,64,69,72 (no 47/51/53) with Thai-descriptor labels; default selected = 52.
  2. Condom result = round(c1Low·5)–round(c1High·5) mm (raw, never floored to a rung); at defaults shows 56 – 59 มม. with "เลือกไซส์ใกล้เคียงเอง" and NO เดิม / NO +N.
  3. A 4th result card shows "ความยาว (ตอนอ่อน) ~+1.6 ซม." + warning "พยุงไม่ให้หด ไม่ใช่ยืดจริง".
  4. Theme/layout/sliders/2D/3D/girth/diameter unchanged; full vitest green; build clean; obfuscated build value-identical; Rule Q-vis L1 screenshot matches this mockup.

Risks / Open Questions

  • Result > biggest dropdown size: fine now — the result is raw mm (e.g. could read "73 – 78 มม." for a huge base+filler); the customer just picks the closest real product. No clamp, no "beyond" tag (intended).
  • Label verbosity: 10 rungs with Thai descriptors — keep labels short; descriptor only on anchors if the row feels long. Owner can tweak copy.
  • Length value fixed vs range: fixed ~+1.6 (matches the shipped spec/footnote + ±~1 disclaimer). Not dose-scaled (the splint effect is roughly constant).
