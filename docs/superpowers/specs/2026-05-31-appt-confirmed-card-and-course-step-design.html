Purpose / Problem

Three user-requested refinements (all cosmetic-shell — no flow/logic change), grounded in the REAL components:

  • ① Confirmed appointments don't stand out + don't bubble up. In the Frontend "วันนี้" list, a confirmed (กดคอนเฟิร์มนัด → status='confirmed' = "ยืนยันแล้ว") appointment only gets a thin left accent bar — easy to miss. Staff want it to (a) change the whole card colour clearly (theme-correct both Dark + Light) and (b) jump to the TOP of "today" (then sorted by time among confirmed), real-time. Reason: late walk-ins mean staff confirm out of time-order and need the confirmed queue surfaced first.
  • ② The "course" OPD step looks like an alarm. The finished-but-no-course-deducted state renders amber ! ยังไม่ตัด (warning styling). Some visits legitimately deduct no course (e.g. wound check / ดูแผล) → the amber "!" is too harsh / reads as an error. Make it a muted/faded dot like the neutral steps, labelled "ไม่ตัดคอร์ส".
  • ③ The course step is missing from the customer-detail treatment history. The Frontend "วันนี้" OPD card shows a 4-step stepper incl. "course"; the customer-detail (customer=LC-…) treatment-history row shows only 3 steps. Add the course step there too — keeping that view's existing coloured (teal/amber) connectors (option 3B).

Decisions (locked via visual Q&A)

  • ① = A — sky-tinted box (light wash + sky border + sky left bar) + reorder confirmed-to-top (today tab), real-time. Sky = the app's existing "ยืนยันแล้ว" colour language (cohesive). Recolour applies whenever confirmed (all tabs, cohesive); reorder is today-tab only.
  • ② = A — muted gray dot + label "ไม่ตัดคอร์ส" (drop amber + "!"). "ตัดแล้ว" stays violet ✓.
  • ③ = B — add the course step in the customer-detail history but KEEP that view's teal/amber coloured connectors (do NOT unify to the gray "สถานะ OPD" stepper).

Acceptance Criteria

  1. Frontend "วันนี้": confirming an appt → its card turns clearly sky-tinted (AA-readable in Dark AND Light) AND moves above all not-yet-confirmed rows; among confirmed, ordered by time; reflects on a 2nd device with no refresh.
  2. Pending/other-status cards + all existing buttons/handlers/data-testids unchanged (cosmetic-shell).
  3. OPD "course" step when finished-without-deduction shows a muted "ไม่ตัดคอร์ส" (no amber, no "!") on BOTH the Frontend card and the customer-detail history; when deducted shows violet ✓ "คอร์ส".
  4. Customer-detail treatment history shows 4 steps (incl. course) with its original teal/amber connectors; courseDeducted reflects the real treatment (verified on a row that DID deduct vs one that didn't).
  5. Tests: V139 stepper tests updated to the new state name + muted contract; new tests for sortApptsConfirmedFirst + the card-tint className + the TreatmentHistoryRow course wiring; full vitest green + build clean.
