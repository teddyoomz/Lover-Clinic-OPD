Filler 2D — Marching-Ants Baseline + Cross-Section Grow-Scale

Date: 2026-06-21 · Status: Approved by user (Q answered) · Scope: src/components/FillerGraphic2D.jsx only (+ its source-grep test). Two small refinements on the just-shipped Model B 2D.

Purpose / Problem

(1) The "เดิม" (baseline) dashed outline is static — the user wants it to animate as clockwise marching-ants, smooth (the "หลังฉีด" red outline keeps its existing breathe). (2) After the Model-B turn the cross-section is hard-pinned to max size (csA = CS_MAX_R always) — it must instead GROW with girth like the side-view thickness (small girth → small circle, big girth → fills), per the user's scale system.

Design

1 · Marching ants on the BASELINE only
   • <style>: @keyframes fgAnts { to { stroke-dashoffset: -9; } }   // -9 = one (5+4) dash cycle → seamless loop
              .fg-ants { animation: fgAnts 0.6s linear infinite; }   // smooth, continuous
              @media (prefers-reduced-motion: reduce){ .fg-ants{ animation:none } }
   • add className="fg-ants" to the showBaseline paths ONLY:
       - side-view baseline <path> (the faint "เดิม" mushroom outline)
       - cross-section baseline <circle> (csB)
   • clockwise: mushPath is drawn clockwise (top→right→head→bottom→left); negative dashoffset marches in the
     draw direction = clockwise. (If the circle marches the wrong way visually, flip the offset sign for it.)
   • the "หลังฉีด" red outline (fg-revBreathe) is UNCHANGED.

2 · Cross-section grows with girth (replaces the always-max)
   const CS_BASE = 44;   // px-per-cm for the cross-section; caps-at-fill near Ø4.5 (typical Ø3.3 → r≈73, sizeable)
   const csScale = Math.min(CS_BASE, (CS_MAX_R * 2) / Math.max(dLo, 0.1)); // grows with Ø, never overflows the box
   const csA = (dLo / 2) * csScale;     // AFTER — grows with girth, fills the box at large girth
   const csB = (d0  / 2) * csScale;     // BEFORE — proportional → after:before == dLo:d0 EXACT

Flow

CSS keyframe → baseline outlines · dLo → csScale → csA/csB
#Step
1@keyframes fgAnts animates stroke-dashoffset 0 → −9 (one dash cycle) linear infinite
2.fg-ants applied to the baseline <path> + baseline <circle> → they march clockwise
3reduced-motion → animation none (a11y)
4csScale = min(CS_BASE, box/dLo) → grows with girth, caps at the box
5csA=(dLo/2)·csScale (after) · csB=(d0/2)·csScale (before) → after:before == dLo:d0

Decisions (locked via Q&A with user)

  • Marching ants on the "เดิม" baseline ONLY (user picked this) — clockwise, smooth (linear infinite), reduced-motion-safe; "หลังฉีด" keeps its breathe.
  • Cross-section grows with girth (csScale = min(CS_BASE, box/dLo)) — not always-max; caps at fill; after:before exact.
  • CS_BASE = 44 — generous so typical girth is sizeable while still growing/capping (tunable after rendered-pixel check).

Acceptance Criteria

  1. The "เดิม" baseline dashed outline visibly marches clockwise + smooth (continuous, seamless loop); reduced-motion disables it.
  2. The "หลังฉีด" red outline is unchanged (still breathes).
  3. Cross-section AFTER circle radius grows with girth (small girth → smaller; large girth → fills the box ~r100); never overflows.
  4. Cross-section before-radius ÷ after-radius == d0 ÷ dLo (exact).
  5. Rendered-pixel verified in the live preview at small + large girth (cross grows) and the ants march.

Risks / Open Questions

  • Circle march direction — SVG circle winding may make the offset sign march counter-clockwise; verify rendered + flip the circle's offset if needed.
  • CS_BASE tuning — 44 targets a sizeable typical; adjust by rendered-pixel if the user wants typical bigger/smaller.
