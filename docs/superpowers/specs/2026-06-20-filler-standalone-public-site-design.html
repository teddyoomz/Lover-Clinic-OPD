Date: 2026-06-20 · Status: Approved by user ("ที่เหลือ ok") · Implementation plan: docs/superpowers/plans/2026-06-20-filler-standalone-public-site.html

Purpose / Problem

หน้าจำลองการเพิ่มขนาด (?play=filler) ปัจจุบันอยู่ใน OPD app เดียวกัน. เวลาส่งลิ้งสาธารณะให้ลูกค้า ลิ้งนั้นชี้ไปที่ lover-clinic-app.vercel.app ซึ่งเป็นโดเมนเดียวกับระบบ OPD ทั้งหมด — bundle ที่ลูกค้าโหลดมีโค้ด/SDK ของ OPD (firebase config ฯลฯ) ปนอยู่. ต้องการแยกหน้านี้ออกไปอีก Vercel project (URL loverclinic.vercel.app) ที่บรรจุแต่หน้า filler ล้วนๆ ไม่มีทางแตะ OPD ได้เลย — เพื่อความปลอดภัยขั้นสุดเมื่อแชร์ลิ้งสู่สาธารณะ. หน้าเดิมใน OPD ยังคงอยู่ (สำหรับใช้ภายใน). แก้ไฟล์ filler ครั้งเดียว → deploy ทั้ง 2 ที่.

การค้นพบสำคัญ (สิ่งที่ทำให้งานนี้สะอาด): หน้า ?play=filler self-contained 100% อยู่แล้ว — App.jsx return <FillerSimulator/> ก่อน auth gate ทุกตัว, และ import graph ของมันมีแค่ React → fillerMath.js / fillerStrings.js / FillerGraphic2D.jsx / Filler3D.jsx (lazy three) + โลโก้ static 2 ไฟล์ + inline <style> block (ไม่พึ่ง index.css/Tailwind, ไม่มี firebase, ไม่มี component ของ OPD). ดังนั้น build แยกที่ mount <FillerSimulator/> จะได้ bundle ที่ ไม่มี firebase/OPD โดยโครงสร้าง — ความปลอดภัยการันตีด้วย tree-shaking ไม่ใช่แค่ convention.

Decisions (locked via Q&A with user)

  • Q1 = A: Repo เดียว, build 2 target — single source, ไม่มี drift (Rule of 3).
  • Q2 = A: npm run deploy:filler คำสั่งเดียว deploy ทั้ง OPD + standalone (ยังต้องพิมพ์ "deploy" อนุญาตทุกครั้ง — V18).
  • Q3 = A: ใส่ <title> ไทย + favicon คลินิก + og:image/og:title/og:description ให้ลิ้งสวยใน LINE/FB.
  • Q4 = B: ถ้า loverclinic.vercel.app ถูกจอง → หยุดถาม user ก่อน ไม่เดาชื่อเอง (hard gate ใน plan).
  • og:image: ทำเองด้วย tool, โลโก้จริงเท่านั้น (lover-clinic-logo-dark.png) บนพื้น brand dark + ข้อความไทย. src/assets/hero.png ไม่ใช่รูปคลินิก → ไม่ใช้. ไม่มี AI/รูปปลอม.

Acceptance Criteria

  1. เปิด loverclinic.vercel.app (หรือ vite preview ของ dist-filler) → render เหมือน ?play=filler เป๊ะ: โลโก้จริง, โทน dark/light toggle, 2D/3D, TH/EN, contact buttons, watermark, มือถือ no-overflow + tap ≥44px. Console สะอาด.
  2. build-guard: grep dist-filler หา firebase|firebaseConfig|getFirestore|AdminDashboard|signInWith = 0 matches. DevTools → Sources บนหน้าจริงไม่มี firebase.
  3. obfuscation ยังทำงานบน standalone: grep dist-filler ไม่เจอ K_REALISTIC / 1.8 / 2.3 / condomIndexForGirth / dCgeo (เหมือน R9).
  4. แก้ไฟล์ filler 1 ครั้ง → npm run deploy:filler → ทั้ง 2 URL อัพเดตตรงกัน (ไม่มี drift).
  5. แชร์ loverclinic.vercel.app ใน LINE/FB → preview card ขึ้นโลโก้คลินิก + ชื่อ + คำอธิบาย (og meta ทำงาน).
  6. filler tests เดิม 98/0 + full vitest ไม่เปลี่ยน (vitest รัน serve → config/obfuscation ใหม่ไม่กระทบ).

Risks / Open Questions

  • URL availability: loverclinic.vercel.app อาจถูกจองทั่วโลกบน Vercel. Q4=B → เช็คตอนสร้าง, ถ้าถูกจองหยุดถาม user. (hard gate — ไม่ build/deploy ต่อจนกว่าจะได้ชื่อ)
  • Vercel CLI auth/org: ต้องมี account link + org ใน environment นี้. ถ้า CLI ไม่ได้ auth → ขอ user login หรือใช้ Vercel MCP. (เช็คตอนเริ่ม plan)
  • กลไก deploy 2 project จาก repo เดียว: ใช้ VERCEL_PROJECT_ID override; ตรวจว่า repo-level vercel.json (มี maxDuration ของ backup endpoints — V122) ไม่ทำให้ build filler พัง → ถ้าชน ใช้ --local-config vercel.filler.json.
  • รูปถ่ายคลินิกจริง: ตอนนี้ไม่มีเป็น asset (hero.png ไม่ใช่). og:image ใช้โลโก้จริง. ถ้า user ส่งรูปถ่ายคลินิกจริงมาทีหลัง → เปลี่ยน og:image ได้.
