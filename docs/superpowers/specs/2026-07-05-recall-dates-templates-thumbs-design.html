Purpose / Problem

  1. Recall dates กำกวม — ชิป "06/07" (dd/mm ไม่มีปี) + ป้าย "6 ก.ค." (ไม่มีปี) อ่านพลาดได้ · user: "แสดงวันที่แบบเต็มเสมอกันพลาด"
  2. "Recall วันนี้" สับสนตอนวันนี้ว่าง — RecallList ข้าม section ว่าง → เห็นแต่กลุ่มพรุ่งนี้ใต้หัวข้อ "Recall วันนี้" ดูเหมือนเป็นของวันนี้
  3. Template editor modal ซ้อน/แว๊ปสลับ — modal `fixed` render ใต้ DOM เมนูใน TFP; ancestor ที่มี transform/filter สร้าง containing block → modal โผล่ "ในช่อง" และเด้งสลับกับ "ข้างนอก"
  4. Dropdown template โหลดนานผิดปกติ — getDocs รอ server round-trip ทุกการเปิดเมนู (มือถือ + long-polling = ช้าเป็นวินาที) ทั้งที่ข้อมูลจิ๋ว + user ต้องการเห็น create/edit/delete ทันทีไม่ต้อง refresh
  5. TFP รูปเยอะโหลดช้า — ทุก grid โหลดรูปเต็มจาก Storage; ต้องโหลดแค่ thumbnail แล้วค่อยโหลดเต็มตอน zoom + ประหยัด traffic

Flow

④ Template dropdown real-time (แทน getDocs-ตอนเปิด)
TFP mount→ listenToOpdNoteTemplatesByBranch subscribe (onSnapshot, branch-scoped)→ เปิดเมนู = แสดง state ปัจจุบันทันที (ไม่มีรอโหลด)
staff สร้าง/แก้/ลบ (เครื่องไหนก็ได้)→ Firestore write (local = latency compensation)→ snapshot fire→ เมนูอัพเดตทันที — ไม่ต้อง refresh/เปิดใหม่

unsubscribe ตอน TFP unmount · แถว "กำลังโหลด" เหลือเฉพาะช่วงก่อน snapshot แรกเท่านั้น · error (เช่นสิทธิ์) → แถวเทา + built-in ใช้ได้เสมอ

⑤ Thumbnail — upload ใหม่ + backfill รูปเก่า (Q3=B)
เลือกรูปใน TFP→ upload รูปเต็ม (เดิม)+ canvas thumb ~320px JPEG (reuse pipeline staff-chat)→ upload thumb ควบคู่→ entry = {dataUrl, storagePath, thumbUrl, thumbStoragePath}
แสดงรูปย่อทุก surface→ img.thumbUrl || img.dataUrl + loading="lazy"→ กด zoom → Lightbox src=รูปเต็ม
backfill (Rule M): scan be_treatments→ entry มี storagePath แต่ไม่มี thumbUrl→ download → sharp resize 320 → upload thumb→ stamp thumbUrl ลง doc (dry-run → --apply, idempotent, audit)

Architecture

① วันที่เต็ม (Q1=B — "6 ก.ค. 2569")

  • src/lib/recallResolvers.js — NEW export formatThaiFullDate(iso) → "6 ก.ค. 2569" (พ.ศ. = ปี+543, เดือนจาก THAI_MONTHS_SHORT); _formatThaiShortDate ทุก callsite (status chip "เลื่อนไป", formatPairBadge) เปลี่ยนมาใช้ตัวเต็ม
  • RecallRow.jsx — dateDisplay (ชิป "06/07") → formatThaiFullDate; recallDateChip ("เลื่อนถึง dd/mm/yyyy") → formatThaiFullDate (uniform)
  • RecallSectionHeader.jsx — รับ todayISO → วันนี้/พรุ่งนี้ แสดง " · {formatThaiFullDate}" ต่อท้าย label (เกินกำหนด/สัปดาห์/ภายหลัง = ช่วง ไม่มีวันเดียว → label เดิม)
  • โหมด full (backend รายการ) ได้ผลอัตโนมัติ — components ร่วมกัน; modal ใดที่ render วันที่ recall → ใช้ formatThaiFullDate (กวาดตอน implement)

② Empty state (Q2=A — compact mode เท่านั้น)

  • Readers (กวาด V12 ทุก surface ที่ render รูปย่อจาก before/after/other/lab/chart images): src={img.thumbUrl || img.dataUrl} + loading="lazy"; zoom/Lightbox = full dataUrl

  • Storage rules: thumb อยู่ path เดิมของ treatment uploads (image/* ครอบอยู่แล้ว — ยืนยันตอน implement; ถ้าต้องแก้ rules = แจ้งใน plan)

  • Deploy: implement → hunt loop (agent ≤2/รอบ) จน 0 → หยุดรอคำสั่ง "deploy" (ไม่ auto)

Acceptance Criteria

  1. ทุกวันที่ใน recall surfaces (ชิปการ์ด, หัวข้อกลุ่มวันนี้/พรุ่งนี้, ป้ายเลื่อนถึง/เลื่อนไป, PairBadge, modal ที่โชว์วันที่) = "d เดือนย่อไทย ปีพ.ศ." — ไม่มี "dd/mm" เปล่าหรือ "d ก.ค." ไม่มีปี เหลืออยู่
  2. Recall วันนี้ ตอนวันนี้ว่าง: เห็น "วันนี้ · {วันที่เต็ม} · 0 รายการ" + กล่อง "✓ ไม่มี Recall วันนี้" ชัดเจน — ไม่มีทางเข้าใจผิดว่ากลุ่มพรุ่งนี้คือของวันนี้; ค้าง/พรุ่งนี้ ว่างก็มีกล่อง ✓ ของตัวเอง
  3. เปิด modal สร้าง/แก้ template → โผล่กลางจอที่เดียว นิ่ง ไม่มีสำเนาในช่อง OPD Card ไม่แว๊ปสลับ (ทดสอบ hover/scroll ระหว่างเปิด)
  4. เปิด dropdown template → เห็นรายการทันที (ไม่มี "กำลังโหลด" ค้างหลัง snapshot แรกของหน้า); สร้าง→โผล่ทันที ลบ→หายทันที แก้→เปลี่ยนทันที โดยไม่ปิด/เปิดเมนูใหม่ (พิสูจน์ L2 ด้วย listener จริง + RTL)
  5. TFP ที่รูปเยอะ: grid โหลดเฉพาะ thumb (~15-30KB/รูป) + zoom โหลดรูปเต็ม; backfill ทำให้รูปเก่ามี thumbUrl ครบ (dry-run นับ → --apply → re-run 0 writes)
  6. full vitest 0 fail + build clean + hunt loop converge → หยุด ไม่ deploy จนกว่าจะสั่ง

Risks / Open Questions

  • Backfill ดาวน์โหลดรูปเต็มทุกใบ (อาจหลาย GB ถ้ารูปเยอะมาก) — รัน local ครั้งเดียว, dry-run รายงานจำนวน/ขนาดก่อน apply
  • sharp เป็น native dependency (devDependency สำหรับ script เท่านั้น — ไม่เข้า bundle)
  • รูป legacy inline base64 ไม่ได้ thumb (skip) — แต่ไม่มี network cost อยู่แล้ว
  • listener เพิ่ม 1 subscription ต่อการเปิด TFP — collection จิ๋ว, cost ต่ำ
  • วันที่ พ.ศ. ใน recall: ยึด precedent ของ recall surfaces เดิม (ชิปเลื่อนนัดใช้ พ.ศ. อยู่แล้ว) — ต่างจาก backend finance ที่ใช้ ค.ศ. ตามกฎ 04 (คนละ surface, จงใจ)
