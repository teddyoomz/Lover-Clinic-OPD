Date: 2026-06-20 · Status: Approved (round-2 Q&A) · Builds on: v1 (shipped, commit 62cd64fd — shaft girth + condom, all VERIFIED) · v1 spec: 2026-06-20-penile-filler-simulator-design.html

Purpose / Problem (v2 delta)

v1 จำลอง ฉีดลำตัว (shaft girth) → รอบวง/Ø/ไซส์ถุงยาง. v2 เพิ่ม 8 อย่างตาม user:

  1. ฉีดหัว (glans) — คลินิกแบ่งฉีดทั้งลำตัว+หัว; หัว ไม่มีผลต่อไซส์ถุงยาง (ยึดลำตัว). research: 2cc → หัว Ø +0.5cm (+16%); ~0.25 cm Ø/cc (±30%).
  2. Split-bar (Q1=A) — แถว cc เป็น "cc รวม + สไลเดอร์แบ่ง% + แถบ stacked ลำตัว(แดง)|หัว(ส้ม)".
  3. ทรงหัวเหมือนจริง — silhouette ทรงเห็ด (shaft → sulcus → corona flare → glans bulb → tip) ให้ลูกค้านึกออกทันที.
  4. ตัด reveal-gate 18+ — โชว์ภาพได้เลย.
  5. ความยาว — step 0.1 นิ้ว · max 10 นิ้ว (= 25.4 ซม., ทั้งสองหน่วยตันจุดเดียวกัน) · default 5 นิ้ว (display นิ้วเป็นหลัก).
  6. EN version — ปุ่มสลับ TH/EN (default TH); ทุก string i18n.
  7. Light theme — ใช้ palette คลินิกจริง (light sakura + dark fire) + ปุ่มสลับ.
  8. Polish — /audit ปรับ design ให้สวย+เข้าตีม.

Decisions (locked)

  • Q1 = A: split-bar = cc รวม + สไลเดอร์แบ่ง% + แถบ stacked ลำตัว|หัว.
  • Q2 = โอเค: หัวทรงเห็ดเหมือนจริง + light/dark + ตัด gate + การ์ดหัว → ทำต่อ.
  • Glans (research): empirical 0.25–0.32 cm Ø/cc; baseline glans Ø = shaft Ø (derive, ไม่เพิ่ม input); cc 0.5–4 จาก split; ไม่มีผลถุงยาง; "ประมาณ".
  • EN: toggle TH/EN, default TH, ทุก string i18n.
  • Light: clinic palette (light + dark) + toggle.
  • Length: นิ้ว default 5, step 0.1, max 10 (=25.4ซม. ทั้งสองตันจุดเดียว).

Acceptance Criteria

  1. cc รวม + split% → ลำตัว/หัว แยกถูก (shaftCc+glansCc=total); แถบ stacked สะท้อนสัดส่วน.
  2. ฉีดหัวเพิ่ม → กราฟิกหัวโต + การ์ด "ขนาดหัว" เดิม→ใหม่ อัปเดต; รอบวง/Ø/ถุงยางไม่เปลี่ยนตาม glans cc (เปลี่ยนตาม shaft cc เท่านั้น).
  3. glans anchor: dg0 3.1 + 2cc → ≈3.6cm (±0.05).
  4. ทรงเห็ด render (shaft+corona+glans bulb) ทั้ง 2D + 3D; หัวโต/ลำตัวโตแยกสเกลได้.
  5. ไม่มี reveal-gate / blur / "แตะเพื่อดู" ในโค้ดเลย.
  6. ความยาว: เปิดมา default 5.0 นิ้ว; step 0.1 นิ้ว; ลากตันที่ 10 นิ้ว; สลับ ซม. แล้วตันที่ 25.4 (จุดเดียวกัน); round-trip นิ้ว↔ซม. ถูก.
  7. ปุ่ม TH/EN สลับทุก string (label/หน่วย/disclaimer/การ์ด); default TH.
  8. ปุ่ม ☀/🌙 สลับ light/dark; light = clinic palette, อ่านออกชัด (AA), ไม่มีสีแดงบนชื่อ (N/A); ทั้ง 2 ธีมสวย เข้าตีม.
  9. shaft math เดิม (v1) ยังถูก (anchor 52·L11·shaftCc16 → +2.0/+2.8); full vitest + build เขียว; three ยัง lazy chunk.
  10. ไม่เขียน Firestore / ไม่เก็บ PII.

Testing (Rule Q — honest)

  • Unit: glans (anchor 3.1+2→3.6, band low≤high, 0cc→no change, monotonic, adversarial) · estimate shaft+glans split (glans cc ไม่กระทบ condom/รอบวง) · length range/step/unit round-trip · shaft v1 cases ยังผ่าน.
  • flow-simulate: split chain (total+%→shaft/glans→cards) · glans-independent-of-condom · EN strings present · no-reveal-gate source-grep · purity (no Firestore).
  • L1 (Claude Preview): เลื่อน split → ลำตัว/หัวโตแยก + การ์ดถูก · เพิ่ม glans cc → ถุงยางไม่ขยับ · สลับ TH/EN + light/dark · length 0.1นิ้ว/ตัน10 · 2D+3D mushroom. (pure client → ไม่มี Rule Q gap; screenshot tool อาจ time out → ดูภาพ = user.)

Risks / Open Questions

  • Glans ±30% ประมาณ: rate จาก 1-study cohort (PE-treatment, glans target เดียวกัน); baseline glans Ø ไม่มี dataset ใหญ่ → derive จาก shaft Ø. โชว์ "ประมาณ" + disclaimer. (shaft = exact verified; glans = approximate-grounded.)
  • i18n scope: EN ครอบ UI strings + disclaimer; ชื่อไซส์ถุงยาง (Regular/Large…) เป็นอังกฤษอยู่แล้ว.
  • Theme: ใช้ local data-theme + clinic var values (หน้า public ไม่อ่าน clinic_settings) — bundled palette, fallback ปลอดภัย.
  • 3D mushroom + glans-separate-scale: mesh ซับซ้อนขึ้นเล็กน้อย; poly ต่ำ, dispose เดิม. WebGL fallback → 2D.
  • Polish/audit: รัน /audit หลัง implement; ปรับ spacing/hierarchy/contrast ให้เข้าตีม.
