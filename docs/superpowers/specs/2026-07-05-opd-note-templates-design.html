Purpose / Problem

แพทย์ต้องจดประวัติซ้ำรูปแบบเดิมทุกครั้งในช่อง CC ของ TFP. ต้องการ dropdown ตรงคำว่า "OPD Card" (เหนือช่อง CC) ให้เลือก template ที่มีอยู่หรือสร้างใหม่เองได้ โดย template ที่สร้างใหม่บันทึกต่อสาขา. เมื่อเลือกแล้วข้อความ template เข้าไปอยู่ในช่อง CC โดยอัตโนมัติ. มี template บังคับ 1 อัน = "สมรรถภาพทางเพศ" (เนื้อหาตามไฟล์ .docx ด้านล่าง). ข้อบังคับ layout: ปุ่ม "บันทึกข้อมูลซักประวัติ" (เขียว, คอลัมน์ซ้าย) กับ "บันทึกสำหรับแพทย์" (ม่วง, คอลัมน์ขวา) ต้อง bottom-align ตรงกันเป๊ะเหมือนเดิม.

Flow

Flow 1 — เลือก template → append เข้า CC (Q2=A)
แพทย์กด "📄 template จดประวัติ ▾"→ เปิดเมนูครั้งแรก: lazy-load listOpdNoteTemplates({branchId})→ เมนูแสดง: built-in "สมรรถภาพทางเพศ" (บังคับ) + template สาขา + "+ สร้างใหม่"→ กดเลือกรายการ→ setOpdField('symptoms', cc ? cc + "\n\n" + content : content)→ sync-in effect ของ OPDFieldWithPrev ([value]) พาเข้า textarea→ เมนูปิด (action ไม่ persist selection)

Race-safety: blur ของ textarea fire ก่อน click บนเมนูเสมอ → parent state commit ผ่าน flushSync ก่อน append (กลไก OPDFieldWithPrev เดิม ไม่ต้องแก้).

Flow 2 — สร้าง / แก้ไข / ลบ template ของสาขา (Q3=A) + data-flow
"+ สร้างใหม่" หรือ ✎ บนแถว→ modal ชื่อ+เนื้อหา (validate: ทั้งคู่ไม่ว่าง)→ saveOpdNoteTemplate (scopedDataLayer → stamp branchId ผ่าน _resolveBranchIdForWrite)→ Firestore be_opd_note_templates/{id}→ refresh list ในเมนู
🗑 บนแถว→ confirm ธรรมดา→ deleteOpdNoteTemplate(id)→ refresh list

Built-in "สมรรถภาพทางเพศ" เป็น constant ในโค้ด — ไม่มี ✎/🗑, ไม่อยู่ใน Firestore, มีเสมอทุกสาขา.

Architecture

  • NEW src/lib/opdNoteTemplateValidation.js — pure: MANDATORY_OPD_NOTE_TEMPLATES (built-in "สมรรถภาพทางเพศ" — เนื้อหา verbatim ด้านล่าง), validateOpdNoteTemplate, normalizeOpdNoteTemplate (V14: ไม่มี undefined leaves), appendTemplateToCc(existing, content) (append + "\n\n" คั่น).
  • src/lib/backendClient.js — Layer 1: listOpdNoteTemplates({branchId, allBranches}) (safe-by-default ตาม BS-13 template), saveOpdNoteTemplate (stamp _resolveBranchIdForWrite), deleteOpdNoteTemplate. id = OPDT-{ts}-{cryptoHex} (Rule C2).
  • src/lib/scopedDataLayer.js — Layer 2 auto-inject wrappers (BS-1).
  • NEW src/components/OpdNoteTemplateMenu.jsx — ปุ่ม pill + เมนู + modal สร้าง/แก้ไข + confirm ลบ; props: {isDark, onInsert(content)}; ไม่รู้จัก TFP state (boundary สะอาด). ESC ผ่าน useEscToClose (LIFO); modal ตาม AV78.
  • src/components/TreatmentFormPage.jsx — จุดแตะเดียว: วาง <OpdNoteTemplateMenu onInsert={...}/> ใน children slot ของ SectionHeader "OPD Card" + handler append ผ่าน setOpdField('symptoms', ...). ไม่แตะกลไก column-balance เดิม (CC grow + flex-col).
  • firestore.rules — NEW match be_opd_note_templates: allow read, write: if isClinicStaff(); → เข้า batch rules ที่ staged อยู่ (deploy พร้อม TFP cards) + probe ใหม่ (anon write → 403) ใน Rule B.
  • BSA bookkeeping — จัด be_opd_note_templates เป็น branch-scoped ใน branch-collection-coverage matrix (BC1.1).

Decisions (locked via Q&A with user)

  • Q1 = A: dropdown เป็นปุ่ม pill ชิดขวาในแถวหัวข้อ "OPD Card" (children slot ของ SectionHeader) — ไม่เพิ่มความสูง column.
  • Q2 = A: เลือก template = append ต่อท้ายข้อความเดิมใน CC (เว้นบรรทัดคั่น "\n\n") — ไม่ทำข้อมูลหาย, ซ้อนหลาย template ได้, ไม่มี confirm popup.
  • Q3 = A: จัดการครบใน dropdown — สร้าง (modal) / แก้ไข ✎ / ลบ 🗑 โดย staff ทุกคนที่เข้า TFP; built-in บังคับแก้/ลบไม่ได้.
  • Storage = แนวทาง 1: NEW branch-scoped collection be_opd_note_templates (ไม่ใช้ clinic_settings เพราะ world-readable + เขียนทับกันได้; ไม่ใช้ system_config เพราะ universal).
  • Built-in = hardcoded constant (precedent: DEFAULT_RECALL_TEMPLATES) — มีเสมอทุกสาขา ลบไม่ได้ ไม่ seed ลง Firestore.
  • Lazy load ตอนเปิดเมนูครั้งแรก + refresh หลัง mutation — ไม่ใช้ real-time listener (template เปลี่ยนไม่บ่อย, ponytail).

Acceptance Criteria

  1. ปุ่ม "📄 template จดประวัติ ▾" อยู่ในแถวหัวข้อ OPD Card เหนือช่อง CC; เปิดเมนูเห็น "สมรรถภาพทางเพศ" (badge บังคับ, ไม่มี ✎/🗑) เป็นรายการแรกเสมอทุกสาขา.
  2. เลือก template ตอน CC ว่าง → CC = เนื้อหา template ตรงตามต้นฉบับ; ตอน CC มีข้อความ → ข้อความเดิมคงอยู่ + template ต่อท้าย (คั่นด้วยบรรทัดว่าง) แล้ว save doc ได้ค่าครบ.
  3. สร้าง template ใหม่ที่สาขา A → เห็นเฉพาะสาขา A (สลับ BranchSelector ไปสาขา B แล้วไม่เห็น — พิสูจน์ด้วย Rule Q L2 client SDK จริงบน prod).
  4. แก้ไข ✎ อัปเดตชื่อ/เนื้อหา; ลบ 🗑 (confirm) แล้วหายจากเมนู; modal ไม่ปิดเมื่อคลิก backdrop (AV78) + ESC ปิดเฉพาะชั้นบนสุด (LIFO).
  5. ปุ่ม "บันทึกข้อมูลซักประวัติ" (เขียว) กับ "บันทึกสำหรับแพทย์" (ม่วง) ยัง bottom-align ตรงกันเป๊ะ (source-grep + กลไก CC-grow เดิมไม่ถูกแตะ).
  6. anon write ไป be_opd_note_templates → 403 (probe ใหม่ใน Rule B).

Risks / Open Questions

  • Live-gated on rules deploy: collection ใหม่ default-deny จนกว่า firestore.rules จะ deploy (batch เดียวกับ TFP cards) — ก่อน deploy เมนูจะโหลด template สาขาไม่ได้ (แสดง built-in ได้เสมอ + error เงียบเป็น list ว่าง; ไม่ block การใช้ built-in).
  • เนื้อหา template ใช้ tab จัดคอลัมน์ (จาก Word) — textarea แสดง tab ได้แต่ความกว้างต่างจาก Word เล็กน้อย; ยึด verbatim ตามไฟล์ ผู้ใช้แก้ template เองได้ภายหลังถ้าอยากปรับ (built-in แก้ไม่ได้ → ถ้าอยากปรับ format ให้แจ้งแก้ constant).
  • ไม่มี edit-concurrency guard (2 คนแก้ template เดียวกัน = last-write-wins) — ยอมรับได้ ไม่ใช่ Rule T domain (ไม่ใช่ balance RMW).
