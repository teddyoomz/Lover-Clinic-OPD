Purpose / Problem

The Tablet Chart Editor relay (shipped + live) lets the clinician annotate a chart template on an iPad with an Apple-Pencil pressure pen, then merges the result into the patient's charts[] via the existing PC save path. Today the tablet canvas (PenCanvas.jsx) is a flat raster: it has the lovely perfect-freehand pressure pen + highlighter + a raster eraser + undo/redo/clear, but no object model — you cannot select, move, resize, or layer anything, and there are no shapes, arrows, or text. Clinicians need a professional annotation toolset (mark regions, draw arrows to spots, label with text, box areas, then nudge/resize those marks) — the same capability the PC editor (ChartCanvas.jsx, Fabric.js) already has, but with the pressure-pen feel preserved.

Goal: upgrade the tablet editor to a full object-based annotation tool — select / move / resize, shapes (line, arrow, rectangle, circle), text, a freeform color picker — without losing the perfect-freehand pressure pen, and without touching the proven relay state machine, presence, or the one TFP prop. Out of scope: changing the PC ChartCanvas, the session relay protocol semantics, or the charts[] consumers (display/print).

Architecture

One genuinely net-new module (the pressure brush) + a Fabric-based canvas that reuses the proven ChartCanvas.jsx patterns (image fit-to-ratio + locked background, shapes, text, JSON-history undo/redo, clear-all, toDataURL export). The relay/session/presence layer is untouched; only the transport payload grows by one JSON blob.

Decisions (locked via Q&A with user)

  • Editing model = B: select / move / resize (object-based), not raster-only.
  • Pen engine = Hybrid perfect-freehand: keep the pressure pen (perfect-freehand) as a custom Fabric brush — NOT Fabric's constant-width PencilBrush.
  • Architecture: Fabric v7 object model (reuse ChartCanvas patterns) + pressureBrush.js. Chosen over a hand-built object model on the raster canvas (re-invents Fabric, higher bug surface).
  • Toolset: Select · Pen(pressure) · Highlighter · Line · Arrow · Rectangle · Circle · Text · Eraser · Undo/Redo/Clear.
  • #2 Eraser (upgraded "support all work types"): object-granular — tap-to-delete + scrub-to-delete (drag intersects → remove), uniform across strokes/shapes/text/arrows, no new dep, no raster compositing. + Clear-all + Undo. True sub-stroke pixel erase = scoped stretch (see Risks), never blocks the feature.
  • Color: 6 quick swatches + freeform 🎨 picker (any hex).
  • Sizes: 3 presets (thin/med/thick) → pen + shape stroke width.
  • #6 Save (upgraded "support all work types"): flatten PNG (unchanged charts[] contract — displays/prints) + transport full fabricJson as a second Storage blob → merged chart entry carries real object data (lossless / re-editable-ready), doc stays URL-only. (Full loadFromJSON object re-edit on open = uses the existing PNG-as-background pattern for now; see Risks.)

Acceptance Criteria

  1. On the real iPad (L1): pen draws pressure-tapered strokes; each is a selectable object after drawing.
  2. Select tool: tap any mark → transform handles appear; drag moves it; corner handles resize; rotate works.
  3. Each shape (line, arrow, rect, circle) draws by dragging at arbitrary size; text places + types via on-screen keyboard.
  4. Eraser: tap removes one object; scrub-drag removes every object touched along the path.
  5. Color: a swatch AND the 🎨 picker both change the active tool's color; 3 sizes change stroke width.
  6. Undo/redo step through every commit (incl. move/resize/delete); Clear removes all marks but keeps the template.
  7. Save → PC merges a flattened PNG into charts[] exactly as today (display + print unaffected) AND the chart entry carries a non-null fabricJson.
  8. Relay unchanged: instant-pop, heartbeat/presence, busy-vs-offline, cancel, orphan-sweep, no-hang-on-save all still pass (regression).
  9. Aspect ratio preserved (the 72ea7585 fix): face 4:5 / body 1:2 render un-stretched.
  10. Verification: Rule I flow-simulate + source-grep + a real-browser L1 pass on the deployed iPad path (Rule Q — mock-only is insufficient).

Risks / Open Questions

ItemPlan
PressureBrush is the main risk (net-new). Fabric v7 BaseBrush API + reading e.pressure from pointer events + emitting a selectable Path.Validate FIRST — spike: brush draws a tapered preview AND commits a movable Fabric.Path before building the rest. If pressure isn't reliably forwarded by Fabric's pointer pipeline, fall back to capturing pointer events on the upper canvas directly (as PenCanvas does today) and add the Path to Fabric on up.
Palm rejection inside a Fabric canvas (Fabric owns pointer handling).Gate touch-pointers while a pen pointer is active at the canvas event layer (port the existing penActiveRef logic).
True sub-stroke pixel erase (erase HALF of one long stroke). Fabric v7 core has no EraserBrush.Scoped stretch, non-blocking. Evaluate @erase2d/fabric (maintained Fabric v6/7 eraser) within a time-box during impl; if it integrates cleanly with the pressure brush, add it as a 2nd eraser mode. Otherwise ship tap+scrub object-delete (fully covers chart-annotation needs).
Full object re-edit on open via loadFromJSON. PC ChartCanvas deliberately avoided it ("no loadFromJSON hang") and uses the saved PNG as a flat background.This spec captures fabricJson (lossless) but does NOT change the open-existing-chart flow. Enabling true object re-edit (validate v7 loadFromJSON doesn't hang) = a separate future task.
Bundle size — Fabric (~303 KB) is already bundled (PC uses it); the tablet route currently doesn't pull it.Lazy-import fabric in TabletChartCanvas (dynamic import('fabric'), same as ChartCanvas) so the tablet route only loads it when an editor opens.
Flatten fidelity — exported PNG must match on-screen incl. the template background.Reuse ChartCanvas export (toDataURL multiplier:2) with the template as a locked, sent-to-back FabricImage.
