Doctor-only header + Recall-วันนี้ blink — Design Spec

Date: 2026-06-29 · Status: Approved by user ("ok") · Scope: 2 UI-only changes on the AdminDashboard นัดหมาย page · Deploy class: frontend-only (no firestore.rules, no data ops → NO Probe-Deploy-Probe).

Purpose / Problem

หน้า นัดหมาย (AdminDashboard) มี 2 จุดที่ user ขอแก้:

  1. Header "หมอเข้าวันนี้" — ปัจจุบัน badge-row แสดง chip ทั้งหมอ (sky) + ผู้ช่วย (purple) และถ้าว่างทั้งคู่จะขึ้นข้อความ ไม่มีพนักงานเข้างาน วันนี้ ซึ่ง user ไม่ต้องการ. ต้องการให้ส่วนนี้ โฟกัสที่หมอ: แสดงว่าวันนี้หมอใครเข้า, ถ้าไม่มีหมอเข้าให้ขึ้น ไม่มีแพทย์เข้า.
  2. ปุ่ม "Recall วันนี้" — ต้อง กระพริบไม่หยุด ถ้ายังมี recall ค้างสถานะ "รอโทร" แม้แต่อันเดียว. พนักงานต้องโทรครบทุกรายการ (→ done/closed) ปุ่มถึงจะหยุดกระพริบ. = signal ที่ดึงให้พนักงานโทรจนหมด list.

Decisions (locked via Q&A)

  • Q1 = A (เฉพาะหมอ): header แสดงเฉพาะ chip หมอ; ตัด chip ผู้ช่วยออก; ไม่มีหมอ → ไม่มีแพทย์เข้า (แม้มีผู้ช่วยเข้า).
  • Q2 = A (สลับสีเต็มใบ): blink แบบ chat-tab — สลับ เทา↔แดงเต็มใบ + glow, 1s ∞.
  • Q3 = A (วันนี้ + เกินกำหนด): ใช้ badge count เดิม (eff ≤ วันนี้, ไม่ done/closed) เป็นตัวสั่ง blink — single source of truth กับ badge.
  • Default-1 (no-answer): recall no-answer auto-snooze +3 วัน → หลุดจากนับวันนี้ → ปุ่มหยุดกระพริบ (ถือว่าโทรแล้ว แม้ไม่รับ).
  • Default-2 (a11y): prefers-reduced-motion → ไม่กระพริบ, ขอบแดงเด่นนิ่งแทน (signal ยังอยู่).

Acceptance Criteria

  1. tab today/tomorrow มีหมอเข้า → เห็น chip หมอ (🩺 ชื่อ + เวลา) เหมือนเดิม; ไม่เห็น chip ผู้ช่วย.
  2. tab today/tomorrow ไม่มีหมอเข้า → เห็น ไม่มีแพทย์เข้า (ไม่เห็น ไม่มีพนักงานเข้างาน อีกเลยทั้ง codebase); แม้มีผู้ช่วยเข้าก็ยังขึ้น ไม่มีแพทย์เข้า.
  3. มี recall ค้าง (count > 0) → ปุ่ม Recall วันนี้ กระพริบสลับสีเต็มใบไม่หยุด (ทั้งตอน inactive และ active).
  4. โทรครบทุกรายการ (count → 0) → ปุ่มหยุดกระพริบทันที (real-time, count มาจาก listener เดิม).
  5. prefers-reduced-motion: reduce → ปุ่มไม่กระพริบ แต่ยังมีขอบแดงเด่นเป็น signal.
  6. ไม่แตะ recall/count logic, firestore.rules, data; npm run build clean; full vitest เขียว.

Testing (Rule Q / I / N)

  • source-grep regression: ไม่เหลือ literal ไม่มีพนักงานเข้างาน ใน AppointmentHubDoctorCards.jsx · มี ไม่มีแพทย์เข้า · ไม่มี appt-hub-assistant-card · RecallTogglePill มี blink-class toggle ตาม count · index.css มี 2 keyframe + reduced-motion.
  • RTL (AppointmentHubDoctorCards): doctorShifts present → chip หมอ; doctorShifts ว่าง → ไม่มีแพทย์เข้า; assistantShifts present + doctorShifts ว่าง → ยัง ไม่มีแพทย์เข้า และไม่มี assistant chip (Rule I flow-simulate ของ Q1=A).
  • RTL (RecallTogglePill): mock count>0 → button มี class recall-pill-blink (inactive) / recall-pill-blink-active (active); count=0 → ไม่มี.
  • Rule Q L1 (real browser, Claude Preview per Rule S): รัน dev (ชี้ prod Firebase), เลือกสาขา นครราชสีมา (badge "1" = มี pending จริง) → ดูด้วยตา (Rule Q-vis): (a) header หมอถูกต้องตามจริง, (b) ปุ่ม Recall กระพริบจริง — screenshot, (c) toggle prefers-reduced-motion → หยุดกระพริบ มีขอบแดงแทน. ถ้าไม่มี pending วันนี้บนสาขาที่ทดสอบ → seed TEST- recall (Rule M/Rule R) แล้วลบหลังเทส.
  • Rule N: small UI change → targeted (touched-area + flow-simulate) ระหว่างทำ, full vitest ก่อน commit/end-of-batch.

Risks / Open Questions

  • ตัด assistant ทิ้งข้อมูล: ข้อมูลผู้ช่วยเข้างานหายจาก header นี้ (user เลือก Q1=A เอง). ตารางผู้ช่วยเต็มยังดูได้ที่หน้า ตั้งค่าตารางคลินิก / schedule view (TodaysDoctorsPanel เป็น doctor-only sidebar คนละจุด).
  • blink ตอน active: เลือก glow-pulse (active variant) แทน full-swap เพราะ pill แดงอยู่แล้ว + พนักงานกำลังอ่าน list — ยัง "กระพริบจน count=0" ตามสั่ง แต่ไม่จัดจ้านเกินจนรบกวนการอ่าน. ถ้า user อยากให้ active สลับสีจัดเต็มด้วยก็ปรับ keyframe ได้.
  • edge no-answer: ถ้า admin re-snooze no-answer มาเป็น ≤ วันนี้เอง → จะนับ+กระพริบ (ถือว่า "ยังต้องโทร") — สอดคล้องเจตนา.
  • L1 ต้องมี pending recall จริง บนสาขาที่ทดสอบเพื่อเห็น blink; ถ้าไม่มีต้อง seed TEST- fixture (ลบหลังเทส).
