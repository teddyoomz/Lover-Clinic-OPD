Modal Scroll Lock (universal sweep) — Design Spec

Date: 2026-07-07 · Status: Approved by user (Approach A) · Implementation plan: docs/superpowers/plans/2026-07-07-modal-scroll-lock.html

Purpose / Problem

User report (verbatim): "เปิด modal pop-up ขึ้นมา แล้วการเลื่อนนิ้ว หรือเลื่อน scroll ของ mouse มันไปเลื่อนหน้าที่ background page แทน มันไม่ไปเลื่อนหน้าใน modal ที่ pop-up ขึ้นมา ในหลายๆจุดของโปรแกรม อยากให้ไล่ตรวจสอบแล้วแก้ให้สมบูรณ์ทั้งหมด ทุกที่"

Inventory: 77 ไฟล์ / 133 จุด มี fixed inset-0 overlay. มี scroll containment อยู่แค่ StaffChatPanel (V82-fix7-bis) + 1 modal ใน PatientForm. จุดชี้ขาด: หน้า backend/admin เลื่อนด้วย inner scroller (AdminDashboard.jsx:5065 overflow-y-auto flex-1) ไม่ใช่ body → ล็อค body อย่างเดียวไม่พอ, ต้องตัด scroll chain ที่ตัว overlay ด้วย.

Mockup Design

Behavior fix — mockup = กลไก 2 ชั้น (data-shape ของ DOM/CSS) ไม่มี visual change ใดๆ เมื่อ modal ปิด/เปิด.

ชั้นที่ 1 — html-level lock (hook ตัวเดียว, ref-counted)
// NEW src/lib/useModalScrollLock.js  (วางข้าง useEscToClose.js — precedent V82-fix7-bis)
useModalScrollLock(active = true)
  mount/active   → lockCount++ ; ครั้งแรก: วัด scrollbar gutter →
                   html.dataset.modalOpen = '1' + --scroll-lock-gutter
  unmount/!active→ lockCount-- ; ครั้งสุดท้าย: ลบ attr + ลบ gutter var

// index.css (เพิ่ม ~4 บรรทัด)

html[data-modal-open] body { touch-action: none; padding-right: var(--scroll-lock-gutter, 0px); }

Architecture

  • NEW src/lib/useModalScrollLock.js — hook เดียว, module-level lockCount, side-effect เฉพาะ document.documentElement attr + CSS var. ไม่มี dependency ใหม่. SSR-safe guard (typeof document).
  • src/index.css — เพิ่ม rule html[data-modal-open] ~4 บรรทัด (แยกจาก html[data-staff-chat-open] เดิม — คนละ semantics).
  • Sweep ต่อไฟล์ (กลุ่ม 1+2, ~60 ไฟล์): (a) เรียก hook ใน component root ของ modal (ตัวที่ render ตอน open เท่านั้น → useModalScrollLock(true); ตัวที่ mount ค้างรับ prop open → useModalScrollLock(open)) (b) ชั้น fixed inset-0 นอกสุดที่รับ event เติม overflow-y-auto overscroll-contain (c) audit panel: ต้องมี max-h-[≤90vh] + overflow-y-auto — ตัวไหนขาดให้เติม ไม่งั้นเนื้อหายาวจะเข้าถึงไม่ได้หลังล็อค (d) โครง sibling-backdrop (backdrop แยกจาก wrapper) → ใส่ที่ layer เต็มจอที่รับ event.
  • ห้ามแตะ: handler/state/props ของ modal ทุกตัว (cosmetic-shell rule), StaffChatPanel lock เดิม, dropdown กลุ่ม 3, print/full-screen กลุ่ม 4.
  • Sweep แบบ inline (ไม่ fan-out agent — memory lock feedback_no_large_agent_fanout), batch commit ตาม Rule K work-first-test-last.

Decisions (locked via Q&A with user)

  • Q1 = กลุ่ม 1+2: modal + lightbox/drawer/palette ล็อคหมด; dropdown/anchored เล็ก (กลุ่ม 3) ไม่ล็อค.
  • Q2 = backdrop no-op: scroll บน backdrop ไม่ขยับอะไรเลย (มาตรฐาน Stripe/Linear); scroll บนเนื้อหา modal เลื่อนเนื้อหา modal.
  • Approach = A: shared hook (ชั้น 1) + backdrop containment sweep (ชั้น 2). ไม่ทำ ModalShell refactor (C) และไม่ใช่ hook-only (B — ไม่ครอบ inner-scroller pages).
  • StaffChatPanel: คงกลไก V82-fix7-bis เดิม (docked panel, mobile-only lock — semantics ต่างจาก modal).

Acceptance Criteria

  1. เปิด modal ตัวแทนจากทุก surface (backend tab modal / TFP modal / AdminDashboard inline modal / customer-facing) → wheel + touch บน backdrop และส่วนว่างของ panel → background นิ่ง 100% ทั้งหน้า body-scroll และหน้า inner-scroller.
  2. เนื้อหา modal ที่ยาวกว่า panel → scroll ภายในเลื่อนได้ปกติ ทั้ง wheel และนิ้ว.
  3. ปิด modal → scroll position ของหน้าเดิมไม่กระโดด + เลื่อนหน้าได้ปกติทันที (ref-count ถึง 0).
  4. Modal ซ้อน modal → ปิดตัวบนแล้วยังล็อค, ปิดหมดแล้วปลด.
  5. ทุก modal ในกลุ่ม 1+2 เนื้อหายาวเกิน viewport ยังเข้าถึงได้ครบ (panel max-h + overflow audit — ไม่มีตัวไหนโดนล็อคขัง).
  6. ไม่มี visual change: pixel parity ตอน modal ปิด = 0 diff; ตอนเปิดไม่มี layout shift (gutter compensation).
  7. Dropdown กลุ่ม 3 + print/full-screen กลุ่ม 4 + StaffChatPanel = พฤติกรรมเดิมทุกประการ.
  8. Classifier test ครอบทั้ง 77 ไฟล์ — ทุกไฟล์ที่มี fixed inset-0 ต้องอยู่กลุ่มใดกลุ่มหนึ่ง (hook+contain หรือ sanctioned พร้อมเหตุผล); ไฟล์ modal ใหม่ในอนาคตไม่ผ่าน test ถ้าไม่ใส่ hook → AV205.
  9. Rule Q L1: real browser ยิง wheel + touch จริงบน modal ตัวแทนแต่ละกลุ่ม + วัด scrollTop ของ background ก่อน/หลัง = ต้องเท่าเดิม.

Risks / Open Questions

  • iOS < 16: overscroll-behavior ไม่รองรับ → touch บน backdrop ในหน้า inner-scroller อาจ chain ทะลุได้ (ชั้น 1 ยังกัน body-scroll pages ให้). iPad คลินิกเป็นรุ่นใหม่ → residual ยอมรับ + จดใน AV205.
  • flex-center + overflow clip: เติม overflow-y-auto บน backdrop ที่ items-center — ถ้า child สูงกว่า viewport หัว modal จะโดน clip. Mitigation: audit ข้อ (c) บังคับ child มี max-h ≤ 90vh เสมอ → ไม่เกิด.
  • Sweep 60 ไฟล์ = mechanical risk: แก้ทีละไฟล์ inline + eyeball โครงจริงต่อไฟล์ (โครง backdrop ไม่เหมือนกัน) + parity screenshot ตัวแทน; V21 fixups คาดว่ามี (source-grep tests เดิมล็อค class string ของ backdrop บางตัว).
  • Modal ที่ mount ค้าง (render เสมอ + ซ่อนด้วย state): ต้องส่ง open เข้า hook ไม่ใช่ hard-true — ตรวจต่อไฟล์ตอน sweep.
