Filler 2D — Red dashed breathe+glow + fainter baseline

Date: 2026-06-20 · Status: Approved by user · Scope: src/components/FillerGraphic2D.jsx (2D only) · Page: ?play=filler (pure-client) · Implementation plan: docs/superpowers/plans/2026-06-20-filler-red-line-breathe-glow.html

Purpose / Problem

The 2D filler illustration draws two dashed outlines: a red "หลังฉีด" (after-injection) outline and a faint "เดิม" (baseline) outline. Two issues:

  1. The red outline is static and easy to miss. User wants it to breathe + glow — fade fully out then return — so it stands out and looks more polished ("ให้ดูชัดขึ้น แล้วดูดีขึ้น").
  2. The faint baseline dash is too opaque (dark-theme white at α 0.5), reading as a strong white line that fights the skin-tone fill behind it ("เส้นประจางขาวเข้มไป รบกวนสีเนื้อด้านหลัง").

Pure visual/CSS change. No math, logic, Firestore, auth, or 3D touched. The faint baseline stays static (only the red outline animates) so it remains a calm reference.

Flow

สว่าง+เรืองสุด (held 0–38%) → จางลง (38–62%) → หายสุด opacity 0 (held 62–72%) → จางกลับมาสว่าง (72–100%) ↺
#Render / CSS wiring
1FillerGraphic2D renders one <style> block (keyframes fgRevBreathe + fgRevGlow, class .fg-revBreathe, prefers-reduced-motion guard) once at the top of its returned fragment.
2Each red outline is its OWN fill="none" element overlaying the static skin-fill body (side-view path + cross-section circle), carrying className="fg-revBreathe" + strokeOpacity="0.6". Animating opacity/glow on the fill:none outline fades ONLY the dashed line — the skin body never moves. (A first cut put the class on the combined fill+stroke element → the WHOLE body faded; fixed by splitting.)
3Browser runs the CSS animation: element opacity 1→0 (with hold) × filter: drop-shadow red glow → none. Red line breathes + glows, vanishes fully, returns.
4prefers-reduced-motion: reduce → animation: none → element opacity 1, no filter → red line static at its original 0.6 (no motion).
5The beforeStroke const (line 45) lowered: dark rgba(255,255,255,0.5)→0.25, light rgba(15,23,42,0.42)→0.21. Faint baseline (lines 88, 106) stays static — no animation.

Architecture

Single-file change in src/components/FillerGraphic2D.jsx:

  • One scoped <style> in the wrapping <div> — keyframes + class + reduced-motion. CSS keyframes are document-global once injected, so a single block covers both inner <svg>s. (CSS is not touched by the build-time obfuscator, which targets only the 4 filler JS files' formula constants.)
  • Split each red outline into its own fill="none" element overlaying the static skin-fill body (side + cross). The animation lives on the fill:none outline only, so opacity→0 + glow fade the LINE, never the skin body. strokeOpacity="0.6" sets the lit peak.
  • One const edit on beforeStroke (line 45) for the fainter baseline.

Decisions (locked via Q&A with user)

  • Q1 = C: Breathing + Glow (not breathing-only, not glow-only).
  • Q2 = 1 (Calm), modified: slow ~3.6s; opacity floor = 0 (fully invisible, brief hold) — not 0.2; lit+glow peak held longer (~0–38% of cycle).
  • Baseline = A (α 0.25): dark α 0.5 → 0.25, light α 0.42 → 0.21 (fainter, still readable as a reference; stays static).
  • Scope: 2D only. 3D has no dashed line — untouched. Faint baseline does NOT animate.
  • Accessibility: prefers-reduced-motion: reduce disables the animation (static = original look).

Acceptance Criteria

  1. On ?play=filler (2D), the red "หลังฉีด" outline (both side-view + cross-section) breathes: holds lit+glowing, fades to fully invisible (opacity 0), holds, returns — ~3.6s loop. Verified in a real browser (Rule Q-vis: seen, not asserted).
  2. The lit/glow peak is visibly held (dwell), not a momentary flash.
  3. The faint "เดิม" baseline is clearly fainter than before and no longer fights the skin tone; it does NOT animate.
  4. With prefers-reduced-motion: reduce, the red line is static at its original look (no motion).
  5. Math / computed results / 3D / Firestore unchanged: filler-math.test.js + existing flow-simulate blocks stay green; full vitest green.
  6. Source-grep regression (v5.6 block) locks: .fg-revBreathe on both red strokes, both keyframes present, reduced-motion guard present, beforeStroke uses 0.25/0.21 (anti-regression on 0.5/0.42).

Risks / Open Questions

  • SVG filter: drop-shadow perf on low-end mobile — gentle (≤6px) + slow (3.6s) + single animated element per SVG → negligible. Acceptable.
  • Baseline too faint to read — mitigated: α 0.25 chosen by user from a live before/after (0.5 / 0.28 / 0.22 / 0.16 → settled 0.25); still visible as a reference. Tunable if needed.
