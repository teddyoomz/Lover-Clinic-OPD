ED Score Box — กดดูรายละเอียดของครั้งอื่นๆ

Date: 2026-06-18 · Status: Approved by user (Q1=A, Q2=A) · Scope: Frontend-only, ONE component (src/components/backend/EDScoreBox.jsx) · no data / no firestore.rules / no API · Plan: 2026-06-18-ed-score-round-select.html

Purpose / Problem

ในหน้าข้อมูลผู้ป่วย (CustomerDetailView) กล่อง "สมรรถภาพ · ED Score" แสดงรายละเอียด (chips ADAM/IIEF-5/MRS/PE) ของ ครั้งล่าสุดอย่างเดียว. ครั้งก่อนๆ ปรากฏแค่ตัวเลขย่อในลิสต์ "ประวัติทุกครั้ง" และ กดไม่ได้. ผู้ใช้ต้องการกดแถวครั้งอื่นในลิสต์ แล้วให้ chips ด้านบนเปลี่ยนไปแสดงรายละเอียดเต็มของครั้งนั้น (ผู้ใช้: "กดครั้งอื่นๆแล้วแสดงรายละเอียดของครั้งๆอื่นๆลงบนช่องแสดงคะแนน ... เพราะครั้งอื่นๆมันแสดงแต่ตัวเลขใน list เฉยๆ").

Architecture

  • ไฟล์เดียว: src/components/backend/EDScoreBox.jsx. ไม่มีไฟล์ใหม่, ไม่แตะ assessmentRoundsCore.js / edScoreDisplay.js (re-use deriveRounds / latestPerType / scoreForType / formatRoundDate ที่มีอยู่).
  • State: เพิ่ม const [selectedRoundId, setSelectedRoundId] = useState(null).
  • Derived: const selectedRound = selectedRoundId ? rounds.find(r => r.id === selectedRoundId) : null; (ทำหน้าที่ guard ในตัว — ถ้าหาไม่เจอ → null).
  • Chip source: เมื่อ selectedRound != null → render chips จาก selectedRound โดยตรง (per-type: วัด→scoreForType, ไม่วัด→"—"); เมื่อ null → latestPerType เดิมทุกอย่าง (รวม olderTag).
  • Header: สลับข้อความตาม state ตามตาราง Flow + pill "← ไปที่ครั้งล่าสุด" โผล่เฉพาะ selected state.
  • History rows → buttons: ทั้งแถวเป็น <button type="button" role="button" aria-pressed={selected} aria-label="ดูผลการประเมิน ครั้งที่ N"> onClick = select; ปุ่มลบ/badge "รับเข้า" เป็น nested element, ปุ่มลบมี e.stopPropagation(). แถวที่เลือก = วงส้ม (เหมือน mockup AFTER).
  • ไม่แตะ: send button (always "ครั้งที่ {nextN}"), empty state, expand/collapse toggle, deleteAssessmentRound, theme classes (CHIP map), data-testid เดิม.

Decisions (locked via Q&A with user)

  • Q1 = A: กดแถวในลิสต์เพื่อเลือก + ปุ่ม "← ไปที่ครั้งล่าสุด" กลับ; แถวที่เลือก highlight; ลิสต์ยังพับ/กางเหมือนเดิม (ไม่บังคับกางค้าง, ไม่ใช้ลูกศร prev/next).
  • Q2 = A: ค่าเริ่มต้น (selectedRoundId === null) = คงพฤติกรรมเดิมทุกอย่าง (merged latestPerType "ครั้งล่าสุด" + olderTag). การกดดูครั้งเก่าเป็นการ เพิ่ม snapshot view — zero change ต่อ default. ไม่ทำให้ทุกมุมมองเป็น snapshot ครั้งเดียว.

Acceptance Criteria

  1. เปิดกล่อง (ยังไม่กดอะไร) → แสดงเหมือนเดิมเป๊ะ: header "ครั้งล่าสุด: ครั้งที่ {N}", chips = merged latestPerType (พร้อม olderTag ถ้ามี). ไม่มี regression.
  2. กางลิสต์ → กดแถว "ครั้งที่ 1" → chips ด้านบนเปลี่ยนเป็นค่า ADAM/IIEF/MRS/PE ของครั้งที่ 1 (type ที่ครั้งนั้นไม่ได้วัด = "—"); header เป็น "กำลังดู: ครั้งที่ 1 · {date}"; แถวที่ 1 highlight; ปรากฏปุ่ม "← ไปที่ครั้งล่าสุด".
  3. กดปุ่ม "← ไปที่ครั้งล่าสุด" → กลับสู่ค่าเริ่มต้น merged เดิม (selectedRoundId = null), ไม่มี highlight.
  4. กดไอคอนลบบนแถว → ลบได้ตามเดิม (ไม่ select แถว); ถ้าลบแถวที่กำลังดูอยู่ → เด้งกลับ merged default.
  5. กดแถว "ครั้งที่ 2" (intake "รับเข้า" หรือ followup) → เลือกได้เหมือนกันทุกแถว.
  6. คีย์บอร์ด: แถวเป็นปุ่มจริง (focusable, Enter/Space เลือกได้); aria-pressed ตรงสถานะ; ปุ่มกลับมี aria-label.
  7. ดาร์ก + ไลท์ธีม: highlight วงส้ม + pill อ่านออกทั้งสองธีม; ไม่ใช้แดงกับชื่อ/HN (rule 04).

Risks / Open Questions

  • ความสม่ำเสมอของ default vs selected: default = merged, selected = round-only. ในข้อมูลลูกค้าปัจจุบันเหมือนกัน; เมื่อครั้งล่าสุดวัด type น้อยกว่าครั้งเก่า ค่าจะต่างกันได้ — ตั้งใจตาม Q2=A (default โชว์ค่าล่าสุดที่รู้, selected โชว์ snapshot จริงของวันนั้น). header แยกชัด ("ครั้งล่าสุด:" vs "กำลังดู:") กันสับสน.
  • Verify (Rule Q): L2 = unit/RTL บน EDScoreBox จริง (mount, click row → chip text เปลี่ยน, click back → กลับ merged, stopPropagation บน delete). L1/pixel = USER hands-on บน authed CustomerDetailView (ไม่มี staff creds สำหรับ automated backend L1 ใน session นี้ — honest gap).
  • ไม่มี data migration / firestore.rules / API → ไม่ต้อง Probe-Deploy-Probe; deploy = vercel frontend-only เมื่อสั่ง.
