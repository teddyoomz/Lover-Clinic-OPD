Staff Chat Resizable Panel — Design Spec

Date: 2026-05-31 · Status: Approved by user (Q1=A, Q2=A; "spec → plan → implement เลย") · Implementation plan: docs/superpowers/plans/2026-05-31-staff-chat-resizable-panel.html

Purpose / Problem

บน desktop กล่อง staff chat (StaffChatPanel) ตอนนี้เป็น 360×480 คงที่ ยึดมุมล่างขวา — ผู้ใช้รายงานว่า "บางทีรู้สึกเล็กไป". ต้องการให้ ลากมุมซ้ายบนปรับความกว้าง+ความสูงได้อิสระ (มุมล่างขวายึดติดจอเหมือนเดิม), UI ขยาย-หด + จัดระเบียบข้อความ ลื่นๆ ทันทีตามเมาส์, และ จำขนาดไว้ที่เครื่อง (per-device) → ย่อ (กด −) แล้วเปิดใหม่ หรือ มีคนทักแล้วแชทเด้งขึ้น = กลับมาขนาดที่ผู้ใช้คนนั้นลากเลือกไว้โดยอัตโนมัติ. มือถือไม่เปลี่ยน (ยังเต็มจอ). ฟีเจอร์นี้ เพิ่มแบบ additive — ไม่แตะ flow แชทเดิม (ส่ง/ย่อ/เด้ง/cursor/unread).

Flow

Flow 1 — Resize drag (smooth, no React re-render mid-drag)
pointerdown บน grip→ setPointerCapture + record {x,y,w,h}→ pointermove: w=startW−Δx, h=startH−Δy → clampSize → เขียน panelRef.style ตรงๆ (DOM, ไม่ setState)→ browser reflow bubbles/list เอง (flex + 80%)→ pointerup: setState(size) + setPanelSize() → localStorage
Flow 2 — Auto-restore (minimize→reopen / someone messages→pop-up) — FREE via mount
กด − (minimize) / มีคนทัก (auto-popup)→ useStaffChat setMinimized(true/false)→ minimized=false → StaffChatPanel MOUNTS ใหม่→ mount: getPanelSize() ?? DEFAULT → clampSize(viewport) → inline size→ เด้งมาขนาดที่ผู้ใช้เลือกไว้ อัตโนมัติ
Flow 3 — Window resize / orientation (กันหลุดจอ)
window 'resize'→ matchMedia('(min-width:768px)') → desktop?→ re-read current → clampSize(new vw,vh) → ถ้าเปลี่ยน เขียน style + setState (persist clamped)→ กล่องพอดีจอเสมอ

desktop↔mobile crossing: ถ้าจอเล็กลงต่ำกว่า 768px → ปล่อย class เดิม (fullscreen), ไม่ apply inline size. กลับเป็น desktop → apply saved size อีกครั้ง.

Architecture

3 ชิ้น additive — ไม่แตะ logic แชทเดิม (messages/send/minimize/popup/cursor/unread):

ไฟล์ชนิดหน้าที่
src/lib/staffChatPanelSize.jsNEW (pure) localStorage helper (device-wide key staffChat.panelSize.v1): getPanelSize() / setPanelSize(s) / clearPanelSize() + ค่าคงที่ DEFAULT={w:360,h:480} MIN={w:360,h:480} MARGIN=32 + pure clampSize({w,h},{vw,vh}). invalid/missing → null (caller ใช้ DEFAULT).
src/hooks/useStaffChatPanelResize.jsNEW desktop gate (matchMedia(min-width:768px) + listener); state size; panelRef; gripProps (pointer handlers, direct-DOM mid-drag, commit+persist บน pointerup); window-resize re-clamp; double-click grip → reset DEFAULT. คืน {isDesktop, size, panelRef, gripProps}.
src/components/staffchat/StaffChatPanel.jsxMODIFIED เรียก hook. desktop: ref={panelRef} + inline style={{width,height}} (ทับ md:w-[360px]) + render grip ⤡ มุมซ้ายบน. mobile: ไม่ render grip/ไม่ใส่ inline → overlay เต็มจอเดิม 100%. header/banner/children เดิมทั้งหมด.

Smoothness: pointermove เขียน panelRef.current.style.width/height ตรงๆ (ไม่ setState → list 50 ข้อความไม่ re-render) → 60fps; commit state + persist เฉพาะตอน pointerup. Reflow: ฟรีจาก layout เดิม (flex-1 min-h-0 overflow-y-auto + bubble max-w-[80%] break-words).

Decisions (locked via Q&A with user)

  • Q1 = A: จุดจับ = มุมซ้ายบนจุดเดียว (ปรับกว้าง+สูงพร้อมกัน) — ตรงตามที่ผู้ใช้สั่ง; ไม่มี edge handles.
  • Q2 = A: เล็กสุด = 360×480 (ห้ามเล็กกว่าปัจจุบัน), ใหญ่สุด = vw−32 × vh−32 (เกือบเต็มจอ "อิสระ"); clamp อัตโนมัติเมื่อจอเล็กลง.
  • By request (locked): desktop เท่านั้น (mobile เต็มจอเดิม) · มุมล่างขวายึดติด (ไม่ย้ายตำแหน่ง) · per-device (device-wide, ไม่ใช่ per-branch) · restore ทั้ง minimize-reopen และ auto-popup.
  • Polish: double-click grip → reset DEFAULT 360×480 (รวมไว้; ถ้าไม่อยากได้บอกตัด).

Acceptance Criteria

  1. Desktop: ลากมุมซ้ายบน → กล่องกว้าง+สูงเปลี่ยนตามเมาส์ลื่นๆ, มุมล่างขวาอยู่กับที่, ข้อความ reflow ทันที.
  2. ขนาด clamp อยู่ใน [360×480 , vw−32 × vh−32] เสมอ (ลองค่า adversarial: ลากเลยขอบ/จอจิ๋ว/NaN ก็ไม่หลุด).
  3. ลากเสร็จ → ขนาดถูกเก็บใน localStorage (device-wide).
  4. กด − แล้วเปิดใหม่ → กล่องกลับมาขนาดที่เก็บไว้ (ไม่ใช่ 360×480).
  5. มีคนทัก (auto-popup) → กล่องเด้งมาขนาดที่เก็บไว้.
  6. ย่อ window เล็กกว่าขนาดที่เก็บไว้ → กล่อง clamp พอดีจอ ไม่หลุดขอบ.
  7. Mobile (<768px): ไม่มี grip, ยังเต็มจอเหมือนเดิม (zero regression).
  8. ไม่มีการเปลี่ยนพฤติกรรมแชทเดิม (ส่ง/ย่อ/เด้ง/cursor/unread/upload/sticker/unsend) — additive ล้วน.

Risks / Open Questions

  • grip ทับ header: grip 18px มุมซ้ายบนซ้อนกับ 💬 ของ header rose — ทำให้ subtle + cursor:nwse-resize; ผู้ใช้ขอ "ขอบซ้ายบน" จึงคาดหวังจับตรงนั้นพอดี. (ไม่บัง interactive อื่นเพราะ header ซ้ายสุดเป็นข้อความ ไม่ใช่ปุ่ม.)
  • Rule Q L1: harness ลากกล่อง mockup/real-mounted ได้ (Chrome MCP) แต่ widget จริงอยู่หลัง auth (AdminDashboard) → assembled L1 = ผู้ใช้ทำเองหลัง deploy (ลื่น/reflow/persist ข้าม minimize+popup/2 ธีม).
  • SSR/test env: matchMedia + localStorage ต้อง guard (typeof window) เพื่อไม่ throw ใน jsdom/SSR.
