Sales table redesign — clean & compact & responsive

Date: 2026-06-01 · Status: Approved by user (Q1=A clean&compact · Q2=A compact-icons · Q3=A source→tag) · Plan: docs/superpowers/plans/2026-06-01-sales-table-redesign.html (next)

Purpose / Problem

ตารางหน้าการขาย (SaleTab.jsx) ดูไม่สวย (user verbatim: “ตอนนี้แม่งไม่สวยจริงๆ จะออกแบบใหม่ทั้งตารางก็ได้”). ปัญหา 4 จุด: (1) badge “จาก OPD Card” อยู่ในคอลัมน์เงิน + flex-wrap ดันยอดเป็น 2 บรรทัด; (2) status chip ไม่มี nowrap → “ชำระแล้ว” ตัดบรรทัดในคอลัมน์แคบ; (3) ปุ่มจัดการ 5 ปุ่ม p-2.5 ใหญ่+ยาว; (4) แถวสูงไม่เท่ากัน. + ต้อง responsive ฉลาดทุกขนาด desktop รวม Windows DPI scaling (125/150% → CSS px แคบลง) + ทุก resolution.

Decisions (locked via Q&A with user)

  • Q1 = A: Clean & compact table (คงโครงตาราง, แก้ 4 จุด + fluid responsive). ไม่ใช่ card-row.
  • Q2 = A: ปุ่มจัดการ = ไอคอนกะทัดรัดโชว์ตลอด (~28px, hover highlight). ไม่ซ่อน hover, ไม่ overflow menu.
  • Q3 = A: source badge → tag ในรายการขาย (คอลัมน์เงินสะอาด).
  • Responsive: fluid + truncate(ลูกค้า/รายการขาย) + min-w + overflow-x-auto. สวย 1024→2560 + Windows DPI scale.

Acceptance Criteria

  1. คอลัมน์ ยอดสุทธิ + ยอดชำระจริง = เลข font-mono ชิดขวา บรรทัดเดียว (nowrap), ไม่มี source badge; paid คงสี tone (green/amber/gray).
  2. SaleSourceTag โชว์ tag บนสุดของ รายการขาย สำหรับ source ∈ {treatment, exchange, share, addRemaining}; source ว่าง/form → ไม่โชว์; ทั้ง 2 ตีม.
  3. SaleStatusPill: nowrap, dot+label, rounded-full, 6 สีตาม resolveSaleStatus; “ชำระแล้ว” ไม่ตัดบรรทัด; ทั้ง 2 ตีม AA.
  4. ปุ่มจัดการ compact (~28px, gap-0.5, hover bg, ชิดขวา); ทุก onClick/testid/conditional-pay/stopPropagation เดิมครบ.
  5. Responsive: ลูกค้า+รายการขาย truncate ellipsis; เลขที่/เงิน/สถานะ/จัดการ nowrap; table min-w กันยุบ; ไม่ overflow ตั้งแต่ ~1024 CSS-px; scroll นุ่มเมื่อแคบกว่า min-w. ดูดี 1024/1280/1440/1920.
  6. Cosmetic-shell: ไม่เปลี่ยน flow/logic/data; ไม่มี fetch/write; row-click (เปิด detail) + ทุก handler/testid เดิมคงอยู่. pagination 30/หน้า (เพิ่งเพิ่ม) คงทำงาน.

Risks / Open Questions

  • Rule Q-vis gap: live window-resize ใน harness ผม limited → responsive verify ผ่าน fixed-width preview (1024/1280/1440) + assembled L1 บน SaleTab จริง + จอ scale จริง = USER post-deploy.
  • min-w จูน: ต้องเลือก min-width ที่ fixed-cols ไม่เบียด (≈ เลขที่ + 2×money + status + actions ≈ 560-620px) + flexible cols truncate ได้ — จูนตอน impl.
  • วันที่ column: mockup ตัด วันที่ ออกเพื่อโชว์กว้าง — ของจริงคงไว้ (nowrap, narrow).
