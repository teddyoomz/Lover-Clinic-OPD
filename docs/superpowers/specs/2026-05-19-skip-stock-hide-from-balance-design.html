Hide Skipped Products from Stock Balance + Edit Shortcut

Date: 2026-05-19 NIGHT+5 EOD
Status: Approved by user (Q1=B · Q2=B · Q3=A · Q4=A)
Origin: User pain — 4 service products (Shock wave, ผ่าตัดทำหมันชาย, ติดตามอาการกับแพทย์, เพิ่ม ตัดเส้นสองสลึง) with skipStockDeduction:true appear in ตารางยอดคงเหลือ rows with -1/-16/-100/-1 negative qty. Visual noise prevents admin from scanning real stock; "ไม่ตัดสต็อคจะเสือกเข้าระบบ stock ทำไม"
Implementation plan: [link to plan.html — TBD]
Parent V-entries: V43 (skip-stock direct-product flag) · V46 (Rule O productId-as-identity) · V54 (BS-13 listener safe-by-default) · V81 (live-update onSnapshot pattern)

Purpose / Problem

The "ไม่ตัดสต็อค" toggle on ProductFormModal (V43, 2026-05-08) lets admin mark a product as opt-out of stock decrement. The toggle works correctly at the deduction boundary (verified Rule Q L2 — Playwright spec tests/e2e/v43-skip-stock-deduction-toggle.spec.js 2/2 PASS).

However, products marked skipStockDeduction:true still appear in the balance table (ตารางยอดคงเหลือ) — both per-branch + central warehouse. They take up visual real-estate, often show stale or negative qty (Shock wave = -100), and serve no functional purpose because they're never deducted.

User directive (verbatim): "เพราะมันรบกวน User ในการใช้สายตา Scan หาสินค้าที่ตัดจริงๆ และยังดูไร้ประโยชน์ที่ถ้าไม่ตัดสต็อคจะเสือกเข้าระบบ stock ทำไม เปลือง Data เปลืองการ Management มากๆ"

Two-part solution:

  1. Filter products with skipStockDeduction:true out of the balance table — both per-branch + central + future branches
  2. Add "แก้ไข" shortcut button in Actions column → opens ProductFormModal on the row's product → live-update via onSnapshot listener so toggling the flag makes the row disappear instantly (no page refresh)

NEW audit invariants

  • AV97 — Every "stock balance reader" (any component rendering be_products + be_stock_batches as a balance table) MUST route products through filterOutSkippedProducts. Source-grep regression locks the contract. Sanctioned exceptions: ProductsTab (master CRUD list — admin needs to see all products to edit them), MovementLogPanel (history audit — immutable).
  • BS-18 — listenToProducts Layer 1 safe-by-default + Layer 2 auto-inject (mirror V54/BS-13). Sanctioned exceptions: NONE.

Branch coverage (per-branch + central + future)

  • Per-branch (existing): listenToProducts({branchId: resolveSelectedBranchId()}) — auto-inject per V54/BS-13 + BS-18.
  • Central warehouse: StockBalancePanel is shared between StockTab + CentralStockTab. CentralStockTab passes defaultLocationId = warehouse ID. The listener uses the same be_products source; the filter is product-level (not location-level) — so it applies identically.
  • Future-branch: filter is a product-level field — branch-agnostic at filter time. New branches created in the future automatically inherit the filter behavior (no code change needed).

Decisions (locked via Q&A with user)

Q1 = BLive-update via onSnapshot listener. Mirror V54/BS-13 listener pattern + V81 onSnapshot architecture. Covers multi-tab, multi-device, save-from-anywhere paths. Cost: 1 extra Firestore listener per StockBalancePanel mount (norm in BSA).
Q2 = BEdit button placement: rightmost in Actions column → [≡ ปรับ] [+ เพิ่ม] [✎ แก้ไข]. Sky-blue tinted (mirror ProductsTab edit convention) to differentiate from red destructive stock actions.
Q3 = AStrict hide, reversible. Batch data preserved in Firestore. Admin untoggle → row reappears with original qty (including -100 negative). Movement Log keeps historical entries (audit-immutable per Rule D). No "show hidden" checkbox. No auto-zero.
Q4 = AV48-style prof-grade 7-tier test bank, ~800-1200 assertions. Covers: unit + source-grep + Rule I flow-simulate + mulberry32 adversarial × 4 product types × 3 tiers + admin-SDK e2e on real prod + Playwright L1 + stress (race / concurrent / branch switch).

Acceptance Criteria (testable + verifiable)

AC1 — On StockBalancePanel mount with locationId = any branch: every product appearing in the rendered DOM table has be_products[productId].skipStockDeduction !== true (admin-SDK readback after L1 Playwright spec).
AC2 — On StockBalancePanel mount with locationId = central warehouse: same as AC1. (Symmetry test.)
AC3 — When admin clicks [✎ แก้ไข] in row: ProductFormModal opens with that row's product loaded into form state (verified via DOM input values).
AC4 (LIVE-UPDATE) — When admin toggles ไม่ตัดสต็อค ON + clicks บันทึก: within 5s, the row disappears from the DOM table without any page refresh, branch switch, or manual reload. Verified via Playwright L1 (real browser).
AC5 (REVERSIBILITY) — When admin untoggles ไม่ตัดสต็อค + saves: row reappears within 5s with the SAME qty as before (negative or positive preserved). Verified via Playwright L1.
AC6 (BRANCH ISOLATION) — Toggling skipStockDeduction on a product at branch A does NOT affect filter behavior at branch B (unless the product is shared cross-branch via allBranches). Verified via 3-tier adversarial e2e.
AC7 (FUTURE-BRANCH) — Create a NEW branch during the test run (TEST-BR-{ts}); load StockBalancePanel for that branch; verify same filter behavior as existing branches. Verified via adversarial e2e Layer 5.
AC8 (NO REGRESSION) — V43 e2e scripts/e2e-skip-stock-deduction.mjs still PASSES 39/39 (decision tree at deduction layer untouched).
AC9 (AUDIT) — /audit-anti-vibe-code AV97 grep passes (zero balance readers bypass filterOutSkippedProducts). /audit-branch-scope BS-18 passes (no UI bypasses Layer 2 listenToProducts).
AC10 (STRESS) — Concurrent saves from 2 tabs: both eventually converge to filtered state. Branch switch during open modal: modal stays for the original product; balance table reflects new branch immediately on close.

Risks / Open Questions

R1 — Listener cost: adding listenToProducts at StockBalancePanel mount = 1 more onSnapshot subscription per panel render. Mitigated by: (a) panel unmounts cleanly cleanup unsub; (b) cost scales linearly with be_products count (~606 docs in prod = trivial). No throttling needed.
R2 — productThresholdMap dependency cascade: changing one-shot → listener triggers a re-render on EVERY product update (not just skipStockDeduction). For 606 products, this is ~10ms per fire — negligible. If perf becomes an issue, narrow the listener to where('skipStockDeduction', '==', true) + Date-based + diff comparison.
R3 — Sanctioned-exception list discipline: ProductsTab + MovementLogPanel must NOT route through filterOutSkippedProducts (admin needs to see all products to edit them; movement log is history). AV97 closed exception list enforced via source-grep test. Adding a 3rd exception requires V-entry (per Rule P 7-step).
R4 — Concurrent toggle race: admin saves at branch A while another tab subscribes at branch B. Both listeners fire independently. Resolution: each panel's listener is independent — no shared state to race. Verified in Tier 7 stress.
R5 — Negative qty visibility loss: 4 current prod products have -100/-1/-16/-1 historical qty. After filter, admin loses direct visibility. Mitigation accepted (Q3=A): admin can untoggle to inspect; movement log still shows the historical writes. If recurring concern → V-entry for opt-in "show hidden" filter checkbox (future).

Out of scope (explicit)

  • Movement Log (history audit — immutable per Rule D)
  • Product picker dropdowns in Stock Order / Transfer / Withdrawal flows (admin may still want to receive stock for service products as audit; their choice)
  • Adjust / Add modals reached from other entry points (e.g. via central stock order form) — filter only applies to balance table view
  • ProductsTab master CRUD (admin needs all products visible to edit; sanctioned AV97 exception)
  • Auto-zero / auto-archive of existing batches when flag flips (Q3=A — strict hide, batches preserved)
