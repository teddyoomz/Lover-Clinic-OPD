Appointment Card Redesign — Design Spec (v2)

Date: 2026-05-26 EOD+6 (v2) · Status: ✅ Approved by user — both themes + all states verified in real browser (Rule Q-vis) · Scope: Q1=C band redesign · Q2=A theme-matched pills · Q3=A Editorial Ember · Q4=A stepper untouched (re-position only) · Q5 remove "⚙ OPD Lifecycle" header label · Q6 rename save button "บันทึกลง OPD" → "บันทึกเข้าระบบ" · Constraint: COSMETIC-SHELL — zero changes to any button wiring / flow / logic / data-testid / conditional.

Purpose / Problem

The appointment-hub card list (AppointmentHubRowCard.jsx) looks unrefined — especially in Light theme — and the OPD lifecycle strip is visibly broken: its badges (OpdLifecycleRow.jsx) use unconditional dark-only Tailwind classes (bg-emerald-900/30 text-emerald-300, bg-blue-900/30 text-blue-300, bg-slate-800/50 text-slate-500) with no light base and no light override in the data-theme override layer — so in Light theme they render green-on-green / washed (the screenshot). Goal: a genuinely beautiful, theme-correct card in both Dark + Light without touching any button's behavior.

Mockup Design — Dark + Light shown TOGETHER (left = 🌙 Dark, right = ☀️ Light)

Every state below renders in both themes side-by-side — no toggle needed. The round-circle สถานะ OPD stepper is the EXISTING Phase 28 component, drawn here as an exact replica of your screenshot (1 ซักประวัติ · 2 ข้าม · 3 เสร็จ · "—") — OFF-LIMITS, NOT redesigned/restyled/recolored, only re-positioned. Section 2 shows that same component in all of its REAL states so you can see บันทึกแล้ว / ซักประวัติแล้ว / แพทย์ลงบันทึกแล้ว / เสร็จแล้ว.

Flow

Band architecture — the card reads top-to-bottom instead of three cramped columns.

① HEADER BAND — HN · membership/ready chips · name (sky-blue, never red) · phone  |  right: LINE badge + status pill
② FINANCE STRIP — wallet · มัดจำ · ค่างชำระ · ยอดสั่งซื้อ (renders only if any present — conditional preserved)
③ DETAIL ZONE (2-col → 1-col mobile) — date + time chip + type/deposit/missed chips  |  ที่ปรึกษา/แพทย์/ผู้ช่วย/ห้อง + นัดมาเพื่อ
④ OPD FOOTER ZONE — OPD lifecycle pills (state A–E) + the สถานะ OPD stepper. NO header label (Q5: "⚙ OPD Lifecycle" text removed)
⑤ ACTION BAR — confirm / record-treatment / mark-complete (ember) · edit (sky) · cancel (rose) — every existing button + handler + data-testid, just relocated to a bottom bar

OPD lifecycle state → pill mapping (logic unchanged in opdSessionState.js — only the classes + the one save-button label change):

StateMeaningPills rendered (AFTER)
AOPD already saved🟢 ดูข้อมูล OPD (emerald)
Bno link sent yet🔵 ส่งลิ้งค์ (blue) · ⏳ ยังไม่มีข้อมูล (slate, disabled)
Clink sent, waiting🟢 ดูลิ้งค์ (emerald) · ⏳ รอลูกค้ากรอก (slate, disabled)
Dcustomer filled → review+save🟢 ดูลิ้งค์ · 🟢 ดูข้อมูล OPD (review) · 🔴 บันทึกเข้าระบบ (rose CTA, pulse)
Etransient post-save🟢 ดูข้อมูล OPD (emerald)

Architecture

  • src/components/admin/_apptHubStyles.js — add a new OPD_PILL token family (blue / emerald / slate-wait / rose-save) using the proven dual-tone pattern, mirroring STATUS_CHIP_CLS / TYPE_CHIP_CLS (light base + dark: override) so the pills render identically to their neighbour chips. Rule of 3: centralize here. (Tailwind dark: is OS-coupled — no darkMode config — but matching the neighbours is the correct, consistent fix; a systemic dark:→data-theme migration is out of scope.)
  • src/components/admin/OpdLifecycleRow.jsx — (a) swap the 4 hardcoded dark-only class strings for the new theme-aware tokens; (b) Q5: remove the OPD lifecycle header <span>; (c) Q6: rename the save button label บันทึกลง OPD → บันทึกเข้าระบบ (label text only). Predicates, props, onClick handlers, data-testid (incl. opd-save-btn-active), disabled state, and the State A–E logic stay byte-for-byte.
  • src/components/admin/AppointmentHubRowCard.jsx — re-lay-out the JSX into the 5 bands above. Every interactive element, every conditional (hasTreatmentForDay, rawStatus, isPastDate, showMarkCompleteBtn, …), every handler prop, every data-testid preserved verbatim — only wrapping markup + classNames move. AppointmentOpdStepperRow + TreatmentLifecycleStepper (round-circle สถานะ OPD stepper) are OFF-LIMITS — NOT edited, NOT restyled, NOT recolored (เด็ดขาด): only re-parented verbatim into the footer band. AppointmentLineBadge likewise re-parented unchanged.
  • No changes to opdSessionState.js, appointmentHubFilters.js, AppointmentHubView.jsx data flow, or any Firestore / endpoint / cron. Pure presentation.

Decisions (locked via Q&A)

  • Q1 = C — full band-architecture layout redesign (not a polish of the 3-column layout).
  • Q2 = A — OPD lifecycle pills = theme-matched dual-tone (blue / emerald / slate-wait / rose-save CTA), mirroring the card's existing chip tokens.
  • Q3 = A — Refined Editorial Ember: warm amber/orange accent across Dark + Light; Light = crisp white card + hairline slate border + soft shadow + warm hover.
  • Q4 = A — the round-circle สถานะ OPD stepper is completely untouched — only re-positioned (NOT even a color change). It is a shared Phase 28 component (TreatmentLifecycleStepper) also rendered in the Backend treatment-history view; recoloring would propagate there.
  • Q5 (2026-05-26 EOD+6 v2) — remove the ⚙ OPD lifecycle header label text from the footer band.
  • Q6 (2026-05-26 EOD+6 v2) — rename the OPD-save button label บันทึกลง OPD → บันทึกเข้าระบบ (label text only; onSaveOpd handler + data-testid="opd-save-btn-active" unchanged).
  • Hard constraints: (a) cosmetic-shell — never touch wiring/flow/logic/handlers/conditionals; (b) every data-testid preserved; (c) patient name stays sky-blue, never red; (d) theme via the existing dual-tone pattern, never new dark-only classes.

Acceptance Criteria

  1. OPD lifecycle pills AA-readable in both Dark and Light (no green-on-green / washed).
  2. Card renders cleanly in both themes + mobile (detail zone collapses to 1 col, no overflow).
  3. Every button in the old card present in the new one, with the same data-testid, same handler, same show/hide condition.
  4. "⚙ OPD Lifecycle" label gone; save button reads "บันทึกเข้าระบบ".
  5. สถานะ OPD stepper byte-identical (no class/color change) — only re-parented.
  6. Patient name never red.
  7. Full vitest green (V21 fixups for layout-assert tests); build clean; /audit-ui-cultural-a11y + /audit-react-patterns pass.
  8. Rule Q-vis: rendered pixels verified by eye in a real browser (Dark + Light) before "done".

Risks / Open Questions

  • Test churn (expected): the Scope-C layout change + the Q6 label change will break RTL/source-grep tests that assert the old structure / the old "บันทึกลง OPD" label (e.g. v118-*). These get V21 fixups — re-anchored on the preserved data-testids, not on layout/label. No real regression if every testid + handler is preserved.
  • Mockup vs React: this HTML hand-renders the design; the implementation maps it to the app's real Tailwind classes + CSS vars. Final sign-off is the real-browser pixel check (Rule Q-vis), not this mockup.
