Staff Chat Enhancements — Design Spec

Date: 2026-05-26 · Status: Approved by user (design + Q1–Q5 locked) · Surface: internal staff-to-staff chat (NOT customer FB/LINE chat) · Implementation plan: docs/superpowers/plans/2026-05-26-staff-chat-day-quote-unsend-stickers.html (pending)

Purpose / Problem

Four user-reported gaps in the staff chat, all on one surface:

  1. No day separators — messages render as a flat list. On days a doctor isn't in, there may be zero messages; reviewing the log later is confusing because there's no marker of which day a bubble belongs to.
  2. Quoted reply text too small — the reply/quote preview is text-[10px], almost unreadable.
  3. No unsend / delete — a wrong upload or mis-sent message cannot be removed; only a 30-day retention cron eventually clears it.
  4. No emoji / stickers — only plain text + @mentions + attachments today.

All four live in src/components/staffchat/* + useStaffChat + staffChatClient. Messages stay in be_staff_chat_messages (canonical artifacts/{APP_ID}/public/data/be_staff_chat_messages); attachment/sticker bytes stay under Storage staff-chat-attachments/{branchId}/{messageId}/.

Flow

Feature 1 · Day-grouping render flow (no Firestore change)
listener → messages[] (createdAt ASC after reverse)→ groupMessagesByDay() · Bangkok-TZ buckets · dual-shape createdAt→ [{dayKey, label, items[]}]→ render pill divider + bubbles per group
Feature 3 · Unsend flow (own message)
long-press / hover (deviceId === own)→ 🗑 + confirm dialog→ deleteDoc(be_staff_chat_messages/{id})+ delete Storage staff-chat-attachments/{branchId}/{id}/**→ onSnapshot removes from all devices

Own-only = client gate (deviceId is localStorage, not auth). Server rule = any clinic-staff may delete. Storage delete reuses the retention cron's folder-delete (Rule of 3). Best-effort: tolerate already-gone objects.

Feature 4 · Bundled sticker send (0 Firebase / 0 Storage)
pick bundled sticker→ message sticker:{kind:'bundled', id:'fluent/party'}→ no Storage write→ recipient renders from its own /public/stickers asset by id
Feature 4 · Custom sticker (library = IndexedDB; sent instance = temporary Storage)
+ add from URL / file→ IndexedDB (this device only · never Firebase)→ on SEND → upload to staff-chat-attachments/{branchId}/{messageId}/→ message sticker:{kind:'custom', url, storagePath, w, h}→ recipient renders from url→ 30-day retention auto-deletes (like any attachment)

AV134 (new invariant)

  • Unsend gate is own-only on the CLIENT (deviceId match); server rule allows clinic-staff delete (no per-user auth in staff chat).
  • Custom-sticker LIBRARY lives in IndexedDB only — it MUST NOT be written to Firestore/Storage as a catalog. Only the SENT instance touches Storage (under the attachment prefix, retention-covered).
  • Bundled stickers are ID references to in-repo assets — 0 Firebase / 0 Storage on send.
  • Delete = Firestore doc + Storage folder both removed (no orphan).

Decisions (locked via Q&A with user)

  • Q1 = A: Day separators — relative (วันนี้/เมื่อวาน) + full Thai BE date w/ weekday for older; centered pill, static (no sticky).
  • Q2 = A: Quote text 10px → 13px (near-normal; body ≈ 14px).
  • Q3 = A: Unsend — own messages only (client deviceId gate), hard-delete Firestore + Storage (vanish, no tombstone), confirm dialog.
  • Q4 = A: Emoji = unicode inserted in text. Stickers = 2-tier (bundled ID-ref 0 Firebase + custom IndexedDB library → temporary Storage on send, 30-day auto-clean).
  • Q5 → MIT: Bundled pack = Microsoft Fluent Emoji (MIT), curated ~20–24 moods, LICENSE committed. (User chose "find a CC0 cartoon pack"; MIT Fluent Emoji is the license-safe, higher-quality realization for a commercial clinic — accepted.)

Acceptance Criteria

  1. Messages spanning ≥2 Bangkok days render a divider between each day group; today/yesterday correct; older = full BE date + weekday.
  2. Quote label + snippet render at 13px.
  3. A 🗑 affordance appears ONLY on the viewer's own messages; on confirm, the Firestore doc AND every Storage object under that message's folder are gone (verified by real-prod L2 e2e); other devices stop showing it.
  4. Other users' messages show no delete affordance.
  5. Emoji picker inserts unicode at the textarea cursor (no Firebase write).
  6. Sending a bundled sticker writes sticker:{kind:'bundled',id} with zero Storage writes and renders on a second device from local assets.
  7. A custom sticker is stored in IndexedDB (not Firebase); on send it uploads to the attachment prefix, the message carries sticker:{kind:'custom',url,...}, renders on a second device, and is removed by the 30-day retention sweep.
  8. Rules deploy passes Probe-Deploy-Probe (anon delete on be_staff_chat_messages + staff-chat-attachments → 403/401; clinic-staff delete succeeds).

Risks / Open Questions

  • License: Fluent Emoji MIT — keep the LICENSE file in /public/stickers/; no in-app attribution required. (If true-CC0-only is mandatory, fallback = OpenMoji CC BY-SA w/ attribution — lower quality choice.)
  • Own-only is client-gated (deviceId, not auth) — accepted; consistent with the existing staff-chat identity model.
  • IndexedDB quota for the custom library (per device): bound count/size, show usage, evict oldest. Library loss on browser-data clear is acceptable (personal palette, re-addable).
  • Custom sticker after 30 days: the sent image is retention-deleted like any attachment → an old chat review shows a missing sticker. Same contract as attachments (acceptable).
  • Emoji dataset: default = curated categorized unicode set (lean, no heavy dep). Can swap to a fuller set later if needed — confirm at spec review if you want the full Unicode keyboard.
  • Date format: divider uses พ.ศ. (per the selected preview) even though backend UIs default to ค.ศ. — staff chat is internal; honoring the chosen visual.
