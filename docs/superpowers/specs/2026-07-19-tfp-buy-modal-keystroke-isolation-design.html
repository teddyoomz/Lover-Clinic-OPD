TFP Buy-Modal Keystroke Isolation (#20) — Design Spec

Date: 2026-07-19 EOD+2 · Status: Approved (user "ทำหมดเลย แต่ห้ามเพิ่มบั๊ค" — batch scope) · Origin: perf punchlist #20 (deferred: "money-critical 5.9k-line form — needs its own session") · Plan: plan

Purpose / Problem

พิมพ์ค้นหาใน buy modal (ช่อง "ค้นหาด้วยชื่อ") → setBuyQuery อยู่บน TreatmentFormPage ทั้งใบ (~5.3k บรรทัด) → ทุก keystroke re-render ทั้งฟอร์มเงิน (จุดเดียวที่เหลือจาก P2 ที่ user อาจยังรู้สึกบนเครื่องช้า) การ extraction step 3 (2026-07-19) ย้าย JSX ของ modal ไป TfpBuyModal.jsx แล้ว แต่ view-filter state ยังอยู่ที่ parent Fix = ย้าย view-filter state เท่านั้น (query / category / show-limit + derived filter) เข้า modal — เงินทุก state (checked / qty / disc / vat) + handlers (openBuyModal / toggleBuyCheck / confirmBuyModal) อยู่ที่ TFP เหมือนเดิม 100% (V13/V42/V162 gates)

Flow

Keystroke render path — ก่อน → หลัง
ก่อน: พิมพ์ 1 ตัวอักษร→ setBuyQuery (TFP state)→ TFP ~5.3k บรรทัด re-render ทั้งใบ→ modal re-render
หลัง: พิมพ์ 1 ตัวอักษร→ setBuyQuery (modal state)→ TfpBuyModal re-render เท่านั้น (TFP นิ่ง)
Reset semantics (ต้องเหมือนเดิม byte-for-byte)
เปิด modal (ปุ่มนอก)→ mount ใหม่ ({buyModalOpen && ...}) → state สด อัตโนมัติ
(แทน setBuyQuery('')ฯลฯ ใน openBuyModal ที่ถูกถอด)
สลับ type ขณะเปิด (select/sidebar)→ useEffect([buyModalType]) ใน modal → reset query/cat/limit
(แทน reset เดิมใน openBuyModal — พฤติกรรมเดิมเป๊ะ)
ติ๊ก/จำนวน/ส่วนลด/VAT/ยืนยัน→ TFP state + handler เดิมทุกอย่าง (per-click ไม่ใช่ per-keystroke — นอก scope)

Decisions

  • ย้ายเฉพาะ view-filter — ศูนย์การแตะ money state/handlers (แนวเดียวกับ extraction step 3 verbatim discipline)
  • Reset ผ่าน mount-fresh + effect-on-type — ไม่แก้ handler ใดใน TFP นอกจากลบ 3 บรรทัด reset ที่ตายแล้ว
  • ไม่ memo TfpBuyModal เพิ่ม (YAGNI — keystroke ไม่แตะ TFP แล้ว; ของอื่นเป็น per-click)

Acceptance Criteria

  1. Source-grep: buyQuery|buySelectedCat|buyShowLimit|buyFilteredItems หายจาก TreatmentFormPage.jsx ทั้งหมด (0 match) และอยู่ใน TfpBuyModal
  2. RTL: พิมพ์ใน modal → list กรองถูก · สลับ type → query/cat/limit reset · โหลดเพิ่ม +50 ทำงาน
  3. เทสเดิมทั้ง buy family (V162 qty-multiply / extraction locks / L1 Playwright spec) เขียวหมด — พฤติกรรมเงิน byte-เดิม
  4. Full vitest 0 fail + build clean

Risks / Open Questions

  • เทสเดิมที่ล็อค props ของ TfpBuyModal หรือ openBuyModal resets = V21 repoints ที่คาดไว้
  • L1 บน prod จริง (Playwright buy-modal spec 10/10 เดิม) รันซ้ำเป็น gate — ห้ามกดยืนยันด้วย data จริงนอก TEST fixture
