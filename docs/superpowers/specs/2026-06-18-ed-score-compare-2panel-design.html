ED Score — 2-Panel Round Compare Design Spec

Date: 2026-06-18 (rev 2 — trend removed) · Status: Approved by user (design + 4 Q&A; rev: drop ดีขึ้น/แย่ลง trend, badge-no-squeeze) · Scope: frontend-only, pure display (no Firestore query / rules / data change) · Implementation plan: docs/superpowers/plans/2026-06-18-ed-score-compare-2panel.html

Purpose / Problem

The ED Score box on CustomerDetailView already lets staff click a chip (ADAM / IIEF / MRS / PE) to open EDDetailModal — a per-question answer view for one round. Clinically, the value of repeat assessment is the change over time. Today staff must open the modal for round N, memorise it, close, re-open for round M, and eyeball the difference.

This feature upgrades the detail modal into a side-by-side two-panel compare (left / right), each panel showing one round's answers for the same assessment type, with a swap-left/right control, per-panel round pickers, a type switcher, and changed-row highlighting. Both rounds' scores are shown side-by-side so the clinician reads the change directly — the app does NOT assert "ดีขึ้น/แย่ลง" (removed per user; avoids a clinically risky direction call). It mirrors the proven TFP split-screen compare (useLayoutPreference + swap). Pure display — it reads the rounds already derived in the box; no new data path.

Mockup Design

Grounded in the real EDDetailModal / ED box theme: dark surface #101010, TONE accents (adam=orange-400, iief=red-400, mrs=pink-400, pe=slate-300), score banner + answer rows (question left / answer right). Changed rows tint sky-blue. Badges ("ล่าสุด" / "วันนี้") are flex:none + centered + nowrap so they never squeeze — the round picker truncates instead.

Flow

Open + state-transition flow
#TriggerWhat happens
1 Click an ED chip in EDScoreBox (already passes {type, round: measured}) Box opens EDDetailModal with type, the clicked round (= primary), and the full rounds[]. Modal computes compareId = autoPickCompareRound(rounds, primary, type) (nearest prior round by date that measured type; fallback nearest later; else null).
2 Render If compare === null OR rounds.length === 1 → single panel + hint "ยังไม่มีครั้งอื่นให้เทียบ". Else → 2 panels. Position from useLayoutPreference('ed-compare','left') (isPrimaryLeft). Each panel shows badges: "ล่าสุด" iff panelRound.id === hero.id, "วันนี้" iff formatRoundDate(...).isToday.
3 Type tab (ADAM/IIEF/MRS/PE) setActiveType. Both panels re-render that type for their currently selected rounds (round selections unchanged — predictable). A panel whose round lacks the type → "ไม่ได้ประเมิน {type} ในครั้งนี้".
4 Per-panel round dropdown setPrimaryId / setCompareId to any round in rounds[]. Each side independent ("เลือกครั้งไหนก็ได้").
5 Swap button ⇆ swap() from the layout hook flips isPrimaryLeft (persisted per-device, like TFP). Pure visual reorder of the two panels (desktop: left↔right; mobile: top↔bottom). Logical primary/compare unchanged.
6 Close: ✕ button OR ESC onClose() → setDetail(null) in box. AV78: backdrop click does NOT close.

Helper contracts (src/lib/edCompare.js)

// align two answer-row arrays by `n`; return { primary:[{...,changed}], compare:[{...,changed}] }
// changed = both present AND primary.answer !== compare.answer (string compare of rendered label)
markChangedRows(rowsPrimary, rowsCompare) → { primary, compare }

EDDetailModal new prop shape

  // round  : the clicked round = initial PRIMARY (guaranteed to carry `type`)
  // rounds : Round[] (all, for pickers + auto-compare)  — NEW
  // hero   : latest round (for the "ล่าสุด" badge)        — NEW

  // AV78 preserved: backdrop no onClick; ESC + ✕ close. role=dialog/aria-modal kept.

Badge layout rule (no-squeeze)

// the "ล่าสุด"/"วันนี้" badges must NEVER compress — when the panel header is tight
// (long date, mobile, both badges showing) the round picker truncates, not the badges.
badge  : flex:none; display:inline-flex; align-items:center; justify-content:center;
         white-space:nowrap;   // → centered label even when squeezed
picker : min-width:0; overflow:hidden; text-overflow:ellipsis; white-space:nowrap;
container: flex justify-between; badges wrapped in a flex:none group

Decisions (locked via Q&A with user)

  • Q1 = A (entry): click chip → open in 2-panel compare directly when ≥2 rounds (left = clicked round, right = previous auto). + enrichment: each panel shows a "ล่าสุด" badge when it is the hero round AND a "วันนี้" badge when its date is today, so staff can't misread which is which. 1 round → single panel + hint.
  • Q2 = A (type scope): both panels always show the SAME assessment type; a type-tab switcher (ADAM/IIEF/MRS/PE) switches both at once.
  • Q3 = A, revised (delta): highlight rows whose answer changed (sky-blue tint). No ดีขึ้น/แย่ลง trend is asserted (removed per user 2026-06-18 — avoids a clinically risky direction call); both rounds' scores are shown side-by-side for the clinician to read.
  • Q4 = A (mobile): on < lg, stack panels top/bottom; swap flips top↔bottom; swap control visible on mobile too.
  • Round pickers: per-panel dropdown listing all rounds (date + round#) — each side independently re-selectable.
  • Swap persistence: reuse useLayoutPreference('ed-compare') (per-device localStorage), same mechanism as TFP.
  • Compare auto-derives per (primary, type): the compare side is the nearest OTHER round measuring the active type (a manual compare pick sticks while it still has that type). The PRIMARY round is the anchor you clicked and stays put — if it lacks the active type, its panel shows the empty state ("ไม่ได้ประเมิน {type} ในครั้งนี้"); the compare side, being auto-derived to a round that HAS the type, never empties.
  • Badges never squeeze: flex:none + centered + nowrap; the picker truncates instead.

Acceptance Criteria

  1. Clicking an ED chip with ≥2 rounds opens a 2-panel modal: left = clicked round, right = the nearest prior round that measured the same type.
  2. Clicking an ED chip with exactly 1 round (or no other round measured the type) opens a single panel + the "ยังไม่มีครั้งอื่นให้เทียบ" hint; no swap control, no compare picker.
  3. Each panel shows the round#, date, a "ล่าสุด" badge iff it is the hero round, and a "วันนี้" badge iff its date is today.
  4. Type tabs switch BOTH panels to the chosen assessment; each panel's selected round is unchanged; a panel whose round lacks the type shows "ไม่ได้ประเมิน {type} ในครั้งนี้".
  5. Each panel's dropdown can select any round; the two sides are independent.
  6. The swap button flips panel positions and the choice persists across reloads (localStorage layout_pref:ed-compare).
  7. Rows whose rendered answer differs between the two shown rounds are highlighted; identical rows are not.
  8. Badges stay centered + uncompressed when the header is tight: "ล่าสุด"/"วันนี้" are flex:none + inline-flex items-center justify-center + whitespace-nowrap; the round picker truncates instead (verify on mobile + when both badges show).
  9. No ดีขึ้น/แย่ลง trend asserted; both rounds' scores shown side-by-side for the clinician to judge.
  10. AV78 holds: clicking the backdrop does NOT close; only ✕ or ESC closes. role="dialog" / aria-modal preserved; chips/tabs/pickers keyboard-operable. No red on any patient name/HN (rule 04).
  11. Full vitest green + build clean. No Firestore/rules/api/data change → frontend-only.

  • RTL (tests/ed-detail-modal.test.jsx, reworked): 2-panel vs single-panel vs empty-type; type tab switches both; per-panel picker; swap flips + persists; changed-row highlight; ล่าสุด/วันนี้ badges present + not-squeezed (whitespace-nowrap, flex:none); AV78 (backdrop no-close, ✕/ESC close); keyboard.
  • Source-grep regression: lock the AV78 no-backdrop-onClick contract, the useLayoutPreference('ed-compare') wiring, and reuse of buildEdAnswerRows/scoreForType (no re-implementation).
  • Rule Q honest gap: pure-display UI over already-loaded data → no new compound query / rules → L2 client-SDK N/A. L1 rendered-pixel on the authed CDV (the modal on a real customer, theme, swap, mobile stack, badge layout) = USER hands-on after merge, consistent with prior ED sessions. All logic is unit/RTL-covered.

Risks / Open Questions

  • Reworking EDDetailModal breaks its current test contract (single-round props). Expected — the rework rewrites ed-detail-modal.test.jsx to the new compare contract; the single-round path is preserved as the 1-panel fallback (its assertions carry forward).
  • Trend direction REMOVED per user (2026-06-18) — the modal no longer asserts ดีขึ้น/แย่ลง (which risked a wrong-direction clinical call). It shows both scores + highlights changed answers; the clinician judges. This removes what was the highest-risk piece.
  • Badge squeeze — explicitly handled: badges are flex:none + centered + nowrap; the long round-date picker truncates. Verified by mockup State A/D + an RTL assertion.
  • Modal width: 2 panels need more than the current max-w-xl. Widen to max-w-3xl (≈768px) on desktop; single-panel keeps max-w-xl. Mobile stacks regardless.
  • No open scope questions — all four design forks locked via Q&A; round-picker style + swap mechanism chosen by recommendation.
