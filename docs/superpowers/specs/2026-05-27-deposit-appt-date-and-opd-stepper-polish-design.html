Deposit-list appointment date + OPD stepper polish — Design Spec

Date: 2026-05-27 (EOD+9) · Status: Approved by user ("ok เขียน spec และ plan เลย") · Type: Pure-display / cosmetic-shell (no flow / logic / data / rules / serverless) · Plan: docs/superpowers/plans/2026-05-27-deposit-appt-date-and-opd-stepper-polish.html

Purpose / Problem

Two independent UI-polish items in the backend, both display-only (data already present; no behaviour change):

  1. Finance → มัดจำ deposit list: a deposit's appointment date is invisible in the list row — admin must open the detail modal (or hover "ไปที่นัด") to see when the appointment is. The date is already denormalised on the deposit doc (dep.appointment.date / .startTime); it just isn't surfaced in the list. Requested: show it per-row for deposits that have an appointment.
  2. Appointment tab → สถานะ OPD stepper: the "สถานะ OPD" stepper (1 ซักประวัติ → 2 แพทย์ → 3 เสร็จ) sits left-aligned in the OPD footer band, and the band colour (orange-50/40 light → "ขาวๆ", amber-500/[0.03] dark → faint warm) does not harmonise with either theme. Requested: centre-align + recolour the band to fit both themes.

Flow

① Deposit row — per-row render decision (data already on doc)

เงื่อนไข (ในแต่ละ row ของ filteredDeposits.map)แสดงใต้ชิป 🎯 purpose
dep.appointment?.date มีค่าลิงก์กดได้ 📅 นัด {fmtThaiDate(date)}{startTime ? ' · '+startTime : ''} → onClick → gotoApptDate(dep) → window.open(/?backend=1&tab=appointment-deposit&date=…)
ไม่มี appointment + status ∈ {active, partial} (= เงื่อนไขเดียวกับปุ่ม +สร้างนัด)ยังไม่นัด (amber hint, ไม่กดได้)
ไม่มี appointment + cancelled / refunded(ไม่แสดง — เหลือแค่ purpose หรือ —)

gotoApptDate(dep) = helper เล็กๆ ดึง logic จากปุ่ม "ไปที่นัด" เดิม (DepositPanel.jsx:858-865) มาใช้ร่วม → ปุ่มเดิม + ลิงก์วันที่ใหม่ เรียกตัวเดียวกัน (DRY, ปุ่มเดิม behaviour ไม่เปลี่ยน).

Architecture

  • Item ① files: src/components/backend/DepositPanel.jsx เท่านั้น — (a) เพิ่ม helper gotoApptDate(dep); (b) refactor onClick ปุ่ม "ไปที่นัด" เดิม (line ~858) ให้เรียก helper (behaviour เดิม); (c) ใน cell "มัดจำสำหรับ" (line ~804-813) เพิ่ม block ใต้ purpose: clickable date OR ยังไม่นัด. fmtThaiDate import อยู่แล้ว.
  • Item ② files: src/components/admin/AppointmentOpdStepperRow.jsx (1 บรรทัด: justify-center) + src/components/admin/AppointmentHubRowCard.jsx (1 บรรทัด: band classes). ไม่แตะ shared stepper.
  • Boundaries: ทั้งสอง = pure-display. ไม่มี Firestore write, ไม่มี data shape change, ไม่มี rules/storage/serverless/cron. ไม่ใช่ branch-scope concern (display only).
  • Cosmetic-shell discipline (feedback_cosmetic_shell_redesign_constraint): handlers/state/props เดิมคงทุกตัว; การเปลี่ยน = เพิ่ม display + reuse นำทางเดิม + เปลี่ยน className. ไม่ลด/เพิ่มฟังก์ชันปุ่มใด.

Decisions (locked via Q&A + Visual Companion)

  • ① placement = B: ใต้ "มัดจำสำหรับ" (ไม่เพิ่มคอลัมน์; เห็น purpose+วันนัดคู่กัน)
  • ① format = วันที่ + เวลา: 📅 นัด 28 พ.ค. 2569 · 14:30 (กันพลาด: ไม่มี startTime → ตัด "· เวลา")
  • ① no-appt = "ยังไม่นัด" (amber)
  • ① clickable = YES: กดวันที่ → เด้งไปที่นัด (reuse URL เดิม); ปุ่ม "ไปที่นัด" เดิมคงไว้
  • ② OPD band = Ember (C) + centre-align; วงกลม stepper เหมือนเดิม (shared, ไม่แตะ)

Acceptance Criteria

  1. Deposit row ที่มี appointment.date → แสดง 📅 นัด <dd MMM พ.ศ.> · <HH:MM> ใต้ purpose chip; ไม่มี startTime → แสดงวันที่อย่างเดียว.
  2. กดวันที่ → เปิดแท็บใหม่ที่ /?backend=1&tab=appointment-deposit&date=<apptDate> (เหมือนปุ่ม "ไปที่นัด" เดิม).
  3. Deposit row ไม่มี appointment + status active/partial → แสดง ยังไม่นัด (amber); cancelled/refunded → ไม่แสดง.
  4. ปุ่ม "ไปที่นัด" + "+สร้างนัด" เดิม behaviour ไม่เปลี่ยน.
  5. OPD stepper row จัดกลาง; band เป็น Ember ทั้ง dark + light; วงกลม/label/connector/เวลา เหมือนเดิมเป๊ะ.
  6. CustomerDetailView treatment-history stepper (shared component) ไม่เปลี่ยนแปลง.
  7. Build clean; full suite ไม่มี new red; source-grep regression ทั้ง 2 ไฟล์ผ่าน.

Risks / Open Questions

  • Rule Q-vis (L1): source-grep/RTL พิสูจน์ shape ไม่พอ — ต้องเปิด browser จริง (Chrome MCP) ดู pixel ทั้ง 2 ธีมก่อน claim เสร็จ. DepositPanel mount หนัก (Firebase/Branch) → list-row อาจ source-grep + flow-simulate + L1; AppointmentOpdStepperRow mount RTL ได้.
  • ปุ่มเขียว "ดูข้อมูลรับเข้า" บนแถบ Ember แดงจางๆ: green-on-faint-red — alpha ต่ำมาก ควรอ่านได้ปกติ; ยืนยันด้วยตา (Rule Q-vis).
  • Legacy deposit doc ที่ appointment.date มีแต่ไม่มี startTime → defensive (ตัดเวลาออก). ไม่มี appointment object เลย → ไป path "ยังไม่นัด".
  • Redundancy: วันที่กดได้ + ปุ่ม "ไปที่นัด" = 2 ทางไปที่เดียวกัน (ตั้งใจ; additive). ถ้า user อยากเอาปุ่มออกภายหลัง = 1-line.
