Deposit-without-appointment + Finance deposit modernize

Date: 2026-05-27 · Status: Approved by user (Q1 = Modernize-in-place, Q2 = Port-all) · Implementation plan: TBD (next: writing-plans) · Branch baseline: master 9209ec70, prod LIVE, 14892 tests/0 fail.

Purpose / Problem

Two related deposit flows are out of step:

  1. Appointment modal (AppointmentFormModal.jsx, 1669 lines) can already book a deposit (จองมัดจำ) with pick-customer-later or an existing HN — but it always requires an appointment (date/time) and always writes the deposit+appointment pair. Staff need to take a deposit without scheduling a visit yet.
  2. The Finance deposit-create form (DepositPanel.jsx, 1555 lines) is the older form: it can't pick a customer later, and a deposit with no appointment shows a blank มัดจำสำหรับ column (the reason lived only on the linked appointment).

Goal: add a ไม่นัดหมาย (no-appointment) deposit path to the appointment modal, and modernize the Finance deposit form in place (pick-later + a มัดจำสำหรับ reason field + visual polish) — preserving all existing money fields (commission/source/evidence).

Reuse / isolation

  • VisitPurposePicker — reused as-is for มัดจำสำหรับ in both surfaces (no fork).
  • The pickLater temp-customer pattern (amber name+phone + customerNameTemp/Phone) is mirrored from the appointment modal — same field names, same persisted shape, so the table's existing ลูกค้าจอง badge renders for both.
  • No new collection. be_deposits gains 5 optional string fields (reader = table/detail/edit + reports; writer = both create paths; can't live elsewhere — no appointment doc for no-appt deposits). Rule C3 satisfied.

Decisions (locked via Q&A with user)

  • Q1 = Modernize in place: keep DepositPanel as its own full-featured money form; appointment modal gains only the ไม่นัดหมาย toggle. Two forms, lower regression risk to the 1669-line modal.
  • Q2 = Port all (full parity): the Finance form keeps every money field (commission/ผู้ขาย, source, ref-no, evidence, time) — nothing dropped.
  • Defaults (accepted, no objection): มัดจำสำหรับ = reuse นัดมาเพื่อ VisitPurposePicker, stored top-level dep.purpose; no-appt deposit still supports + สร้างนัด later; when no-appt → hide date/time/หมอ/ห้อง/recurring and write deposit-only.
  • Open call (accepted): a no-appt deposit created from the appointment modal does not collect commission (sellers empty — lib-valid); commission added later via Finance edit. Full commission form lives in Finance.

Acceptance Criteria

  1. In the appointment modal, selecting จองมัดจำ shows a ไม่นัดหมาย checkbox; ticking it hides date/time/หมอ/ห้อง/recurring and the modal saves a be_deposits doc with no be_appointments doc.
  2. The no-appt deposit works with both an existing-HN customer and เลือกลูกค้าภายหลัง (temp name+phone persisted; customerId:'').
  3. มัดจำสำหรับ (= chosen appointmentTo) is stored top-level and shows in the Finance deposit table's มัดจำสำหรับ column for no-appt + Finance deposits.
  4. The no-appt deposit still offers + สร้างนัด on its row and schedules correctly afterward.
  5. Finance สร้างมัดจำ supports เลือกลูกค้าภายหลัง + a มัดจำสำหรับ field, while still capturing ผู้ขาย/ค่ามือ + source + ref-no + evidence (no field lost).
  6. Existing with-appointment deposit-booking (both entry points) is unchanged: still the atomic createDepositBookingPair.
  7. Branch-scope preserved: deposit stamped with the selected branch via _resolveBranchIdForWrite; list/table re-fetch on branch switch.
