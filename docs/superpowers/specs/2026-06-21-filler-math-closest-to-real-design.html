Purpose / Problem

The penile-filler size simulator was deliberately biased low ("ปัดเศษออกเอาต่ำไว้ก่อน" — floor the condom snap, anchor k at the low end). The user reversed that directive: research as thoroughly as possible, then compute as close to reality as the evidence allows. We ran an exhaustive cited research program — 5-facet baseline → 3-skeptic adversarial verification → a loop-until-dry research loop (9 rounds, converged when a round changed no constant), ~17 agents, every number traced to a primary source (RCTs, large cohorts, meta-analyses, ISO 4074, manufacturer data). This spec encodes the resulting closest-to-real calibration.

Three UX framings were locked with the user before research: (1) the headline shows the PEAK (~1-month) result, with the durable value shown alongside for honesty; (2) the user enters ERECT girth + length (condoms are sized to erect girth); (3) condom snap = FLOOR/round-down (research confirmed this is the correct retention rule) with the next-up neighbor shown.

Architecture

  • src/lib/fillerMath.js (core — pure math, single source of truth): replace the 2 flat k-constants with the k-ladder + peak-fade map; keep the exact geometry engine; correct CONDOM_LADDER to the real 9-rung MyONE; FLOOR snap kept + emit the neighbor-up rung; drop the +2→72 extension (ceiling 64); glans ΔØ/cc → 0.17 with a 2mL saturation cap + glans fade; re-scaled volume constants; SE-Asian default + range constants; new flaccid/erect length-delta constants (display-only). New helpers: girthGainCm(C0,L,V,k), condomRecommend(erectGirth) (returns {rung, neighborUp, beyond, category}), glansDeltaDiameter(doseMl) (saturating).
  • src/components/FillerSimulator.jsx (UI wiring): erect-input labels + SE-Asian defaults/ranges; headline = peak + durable-secondary; condom neighbor display + category; glans dose as a {0,2,3} discrete control (was continuous); volume slider re-ranged 5–30 with the >20 warning band; ±1cm band; length secondary line; honesty notes. Cosmetic-shell discipline: all changes consume fillerMath + copy — no new data sources, no behavior beyond surfacing the new constants.
  • src/lib/fillerStrings.js: TH/EN copy for peak/durable, condom neighbor, glans, length-splint, volume warning, honesty notes.
  • Obfuscator note: fillerMath.js is in the build obfuscator scope (formula protection) — keep the new k/condom/glans constants as integer fractions (value-identical) so numbersToExpressions hides the literals, exactly as the current code does.

Decisions (locked via Q&A + research)

  • Framing Q1 = peak headline + durable shown alongside — user chose "show peak"; research + adversarial flag (peak fades ~29%@12mo) → keep peak as headline but display durable co-equally (not hidden).
  • Framing Q2 = ERECT input — user enters erect girth+length; the geometric ΔC is computed on the erect base → the (smaller) erect gain. Optional flaccid path converts ×1.25 first.
  • Framing Q3 = FLOOR snap + neighbors — research confirmed round-down is the correct retention rule (MyONE/ISO); keep FLOOR, show the next-up rung.
  • Condom sized on the DURABLE girth — the lasting, honest, practical size (the research synthesis itself sized the condom on the durable girth). The peak is shown as a brief upper bound, not the recommended buy size.
  • Girth k: peak 1.90, durable(12mo) 1.22, full ladder (Yang2020 RCT / Ahn RCT / Wang2022 cohort).
  • Condom ladder corrected to the real MyONE 9-rung [45,47,49,51,53,55,57,60,64]; ceiling 64 (drop fictional 72).
  • Glans ΔØ/cc 0.17 (band 0.13–0.24); discrete 2–3mL saturating; own 85%@5yr fade; decoupled.
  • Volume default 16, range 5–30 (hard-cap 30, warn >20); dose-response saturating via geometry (n=103 regression: more mL ≠ more cm).
  • Defaults re-seeded SE-Asian (erect girth ~10.4cm, length ~13.4cm) — Western seeding overstated girth by ~2 condom sizes.
  • Length: filler increases FLACCID length (+1.65cm durable splint) but not erect/true (+0.6cm fading) — display-only secondary; L in the geometry stays the erect/stretched length.

Acceptance Criteria

  1. fillerMath.estimate with the SE-Asian erect defaults (C0=10.4, L=13.4, V=16, glans=2) returns peak girth gain ≈ +1.3cm and durable ≈ +0.85cm (matches Wang erect anchor), condom FLOOR rung (sized on the durable girth) with a neighbor-up, glans ΔØ ≈ +0.35cm.
  2. Doubling volume does NOT double the girth gain (saturation): gain(32mL) − gain(16mL) < gain(16mL) − gain(0).
  3. Condom recommendation uses the real ladder [45,47,49,51,53,55,57,60,64]; a required NW > 64 returns beyondMainstream:true; the old 72 ceiling is gone.
  4. Glans dose saturates: ΔØ(3mL) ≈ ΔØ(2mL); glans never changes the shaft girth or condom result.
  5. Volume > 20mL surfaces a warning; the input is hard-capped at 30mL; the old 50cc ceiling is gone.
  6. The geometry L is the erect/stretched length; the flaccid-length splint gain is display-only and never enters the girth calc.
  7. Headline shows the peak value with the durable value displayed alongside (not hidden) + a fade note.
  8. Full filler test suite green (old-value locks updated to the researched values, V21-style); new tests for the k-ladder, fade, erect-state, corrected condom ladder, glans saturation, volume cap, SE-Asian defaults, length-secondary; npm run build clean; obfuscated build still computes identically; Rule Q-vis L1 on the real rendered card.

Risks / Open Questions

  • Erect-efficiency is a model extrapolation — the k-ladder was fitted on flaccid-shaft geometry; Wang's erect serial data (n=38) is the main erect anchor. Documented as a caveat, not hidden.
  • Thai girth data gap — defaults use the Vietnamese n=14,597 regional proxy; no rigorous Thai girth cohort exists. Acceptable for a toy; disclosed.
  • Individual variance dwarfs the point estimate (SD ±0.95–2.2cm) — mitigated by the ±1cm band + "estimate not promise" copy.
  • HA-specific — calibration is for hyaluronic acid; PLA/PMA differ. The sim assumes HA (the clinic's product).
  • Honesty vs marketing — peak is the headline (user choice) but durable is shown co-equally; this is the agreed compromise between an impressive number and not over-promising.
  • Test churn: ~98–102 existing filler tests lock the old constants → expect a batch of V21-style updates alongside the new tests.
