Customer-link header + configurable LINE link keywords — Design Spec

Date: 2026-07-07 · Status: Approved by user (Q1=B, Q2=Customer Info, Q3=strip hn, Q4=global list) · Implementation plan: docs/superpowers/plans/2026-07-07-customer-link-header-and-line-keywords.html

Purpose / Problem

ส่วน 1: หน้า customer link (?patient=) ยังใช้ภาษา "ผู้ป่วย" + โชว์ avatar ตัวย่อชื่อ ("ไฉ") + HN badge ซึ่ง user ไม่ต้องการให้ลูกค้าเห็น → ตัด avatar + HN ออก, เปลี่ยนเป็น "ข้อมูลลูกค้า" / "Customer Info".

ส่วน 2: คำนำหน้าที่ลูกค้าพิมพ์ใน LINE OA เพื่อขอผูกบัญชี (ผูก / ผูกบัญชี / link + เลขบัตร/พาสปอร์ต) เป็น hardcoded regex ใน lineBotResponder.js — user ต้องการตั้งค่าชุดคำได้เอง (>1 คำ, เผื่อเปลี่ยนในอนาคต) จากเมนู "คำขอผูก LINE" ใน backend. หมายเหตุ: link <เลข> ใช้ได้อยู่แล้ววันนี้ — งานนี้คือทำให้ชุดคำ configurable.

Flow

ส่วน 1 — โหลดหน้า ?patient= (ตัด hn ตลอดสาย)

ลูกค้าเปิดลิงก์ ?patient=→ /api/patient-view resolve token→ payload: name · phone · courses · appointments (ไม่มี hn แล้ว)→ การ์ดกึ่งกลาง: ชื่อ + เบอร์ (ไม่มี avatar/HN)

ส่วน 2 — ตั้งค่าคำผูก → bot ใช้คำใหม่ (data-flow)

แอดมินแก้ chips ใน "คำขอผูก LINE"→ validate (validateIdLinkKeywords)→ เขียน clinic_settings/link_id_keywords (AMENDED — ดู Decisions)

ลูกค้าพิมพ์ "<คำ> 1234567890123" ใน LINE→ webhook อ่าน chat_config (อ่านอยู่แล้ว — 0 read เพิ่ม)→ interpretCustomerMessage(text, {idLinkKeywords})→ intent id-link-request→ rate-limit → คิวแอดมิน → same-reply ack (เดิมทุกอย่าง)

ไม่มี field → default ['ผูก','ผูกบัญชี','link'] = พฤติกรรมเดิมเป๊ะ (zero migration) · เลขเปล่าไม่มีคำนำหน้า → ยังทำงานเหมือนเดิม (คนละ branch, ไม่แตะ) · hint ตอบกลับ (เลขผิด format) render จากคำแรกในชุด

Architecture

ไฟล์การเปลี่ยนแปลง
src/pages/PatientDashboard.jsxTX.th.headerSub → "ข้อมูลลูกค้า" · TX.en.headerSub → "Customer Info" · การ์ด: ลบ avatar + HN badge → flex-col กึ่งกลาง (ชื่อ + PhoneLink) · ลบ getInitials (ไม่มีผู้ใช้แล้ว) + ตัวแปร hn/brokerProClinicHN mapping ที่ตายแล้ว
api/patient-view.jsตัด hn ออกจาก response + อัพเดท field-minimized comment (AV anon-safety แข็งขึ้น)
src/lib/lineBotResponder.jsNEW export DEFAULT_ID_LINK_KEYWORDS + validateIdLinkKeywords(list) (pure) · interpretCustomerMessage(text, opts) รับ opts.idLinkKeywords — regex สร้างจากชุดคำ (escape พิเศษ, เรียงยาว→สั้น, case-insensitive) · hint constant → ฟังก์ชันรับคำแรก
api/webhook/line.jsส่ง chatConfig.idLinkKeywords เข้า interpretCustomerMessage + hint call site (config ถูก fetch อยู่แล้วใน handler)
src/components/backend/LinkRequestsTab.jsxการ์ดตั้งค่า chips CRUD + save ลง chat_config (client SDK — pattern เดียวกับ LineSettingsTab)

Decisions (locked via Q&A with user)

  • Q1 = B: การ์ดชื่อจัดกึ่งกลาง (ชื่อ + เบอร์)
  • Q2 = Customer Info: หัวเพจ EN
  • Q3 = ตัด: hn ออกจาก API response ด้วย (data minimization)
  • Q4 = Global: ชุดคำเดียวทั้งคลินิก, แก้ได้ทุกคำ, seed defaults; ไม่ per-branch. AMENDED (implementation-forced): เก็บใน doc ใหม่ clinic_settings/link_id_keywords แทน chat_config — chat_config เป็น secret-store ที่ client SDK ถูก deny (WS1-C2-bis); doc ใหม่อยู่ใต้ wildcard rule เดิม (read public — คำไม่ใช่ secret, bot hint บอกลูกค้าอยู่แล้ว / write staff) → ไม่แตะ firestore.rules

Acceptance Criteria

  1. หน้า ?patient= (TH): หัวเพจ "ข้อมูลลูกค้า", การ์ดมีเฉพาะชื่อ + เบอร์ จัดกึ่งกลาง — ไม่มี avatar, ไม่มี HN; EN: "Customer Info"; สไตล์การ์ดอื่นเดิมทุกเส้น (pixel-level)
  2. /api/patient-view response ไม่มี field hn; courses/appointments/ชื่อ/เบอร์ เหมือนเดิม (L2 diag)
  3. ไม่ตั้งค่าอะไรเลย → bot ตอบ ผูก/ผูกบัญชี/link <เลข> + เลขเปล่า เหมือนเดิมทุกกรณี (default fallback)
  4. แอดมินเพิ่มคำ (เช่น "เชื่อม") ใน "คำขอผูก LINE" → ลูกค้าพิมพ์ เชื่อม 1234567890123 → เข้าคิวคำขอผูก; ลบคำ "ผูก" → ผูก <เลข> ไม่ trigger อีก
  5. Validation ครบ: ≥1 คำ, ≤10, 1-30 ตัว, ไม่มีช่องว่าง, ไม่ใช่ตัวเลขล้วน, ไม่ซ้ำ; regex-special chars ในคำ (เช่น ".", "*") ไม่ทำ regex พัง
  6. Rate-limit + anti-enumeration + bare-ID + คิวแอดมิน ไม่เปลี่ยนพฤติกรรม

Risks / Open Questions

  • คำชนกับ trigger อื่น: id-link check รันก่อน exact-match triggers และ fire เฉพาะเมื่อมีข้อความตามหลังคำ → คำอย่าง "คอร์ส" เป็นคำผูกได้โดยไม่ชน "คอร์ส" เดี่ยวๆ (ยังไปที่เมนูคอร์ส) — บันทึกไว้ ไม่บล็อค
  • Legacy session-mode: การ์ดเดียวกันใช้กับ session-mode เก่า → avatar/HN หายด้วย (intended — user สั่งเอาออก)
  • chat_config เป็น global doc: ทุกสาขา (ทุก LINE OA) ใช้ชุดคำเดียวกัน — ตรง Q4
  • Rule Q ฝั่ง bot: interpret layer เป็น pure (unit-เทสแน่น) + wiring source-grep; พฤติกรรมบน LINE จริง = user L1 หลัง deploy (พิมพ์คำ custom จริง)
