V121 — Card-flow Notifications Design Spec

Date: 2026-05-23 EOD+1 LATE+8 · Status: Approved by user ("approve") · Owner area: AdminDashboard tab bar + AppointmentHubView sub-pill bar · Implementation plan: writing-plans (next step) · V-tag: V121 (next available after V120)

Purpose / Problem

User directive (verbatim, 2026-05-23): "เมื่อลูกค้ากรอกข้อมูลเสร็จแล้ว แล้ว admin ยังไม่ได้อ่าน ให้ทำ bubble notification ใน tab นัดหมาย และ tab ย่อย และการแจ้งเตือน push notification ให้เหมือนกับที่เคยทำกับ คิวหน้า Clinic เป๊ะๆด้วยนะ คือพูดง่ายๆ ระบบการแจ้งเตือนเหมือนเดิม แค่ย้ายหน้า"

The gap: V120 hides V118 card-flow OPD sessions from the คิวหน้า Clinic queue (since the Card has its own affordances). This breaks the existing notification surface — admin no longer sees the bubble on คิวหน้า Clinic tab when a customer fills the form via card-flow QR. V121 restores notification visibility on the Frontend นัดหมาย tab + its sub-pills, using the EXACT same notification primitives (sound + browser push + FCM + bubble count) — just rendered on a different surface.

Class-of-bug avoided: notification blackout caused by a UI scope change. The notification primitives are stable; only the rendering surface moves.

Critical implementation details

Modal-open gate (skip isUnread:false for card-flow)

// AdminDashboard.jsx:3417 area — V121 modification
const isDepositKeepUnread = session.formType === 'deposit' && session.isUnread;
// V121 (2026-05-23): card-flow sessions (V118 + V120 hide-from-queue) use
// SAVE as the read action, NOT modal-open. Skip the isUnread:false write
// for these — bubble persists until handleOpdClick succeeds.
const isCardFlowSession = session.createdFromBackendBooking && session.isHiddenFromQueue;
if (session.isUnread && !isDepositKeepUnread && !isCardFlowSession) {
  lastViewedStrRef.current[session.id] = stableStr(session.patientData || {});
  lastAutoSyncedStrRef.current[session.id] = stableStr(session.patientData || {});
  try {
    await updateDoc(doc(db, 'artifacts', appId, 'public', 'data', 'opd_sessions', session.id), { isUnread: false });
  } catch(e) { console.error('updateDoc isUnread:', e); }
}

Bubble count memo (AdminDashboard)

// V121 (2026-05-23): card-flow pending session count across all session arrays.
// allNotifData = [...sessions, ...noDepositSessions] (existing). V121 spans
// ALL 5 session arrays to catch V118 sessions wherever they live.
const cardFlowUnreadCount = useMemo(() => {
  let count = 0;
  for (const arr of [sessions, archivedSessions, depositSessions, archivedDepositSessions, noDepositSessions]) {
    for (const s of arr || []) {
      if (isCardFlowUnread(s)) count++;
    }
  }
  return count;
}, [sessions, archivedSessions, depositSessions, archivedDepositSessions, noDepositSessions]);

Sub-pill counts (HubView)

// AppointmentHubView.jsx — derive per-sub-pill counts from the same
// per-row state derivation already used for OpdLifecycleRow rendering.
// State D sessions are the only card-flow unread + customer-filled state.
const cardFlowSubPillCounts = useMemo(() => {
  const counts = { today: 0, tomorrow: 0, future: 0, past: 0 };
  for (const a of appts) {
    const linkedSession = resolveLinkedSession?.(a) ?? null;
    if (!linkedSession || !linkedSession.isUnread) continue;
    if (isOpdSessionSaved(linkedSession)) continue;
    if (!linkedSession.createdFromBackendBooking || !linkedSession.isHiddenFromQueue) continue;
    const bucket = appointmentBucketForDate(a.date, new Date());
    if (counts[bucket] !== undefined) counts[bucket]++;
  }
  return counts;
}, [appts, resolveLinkedSession]);

Push notification listener extension

The existing detection at AdminDashboard.jsx:2362+ processes allNotifData = [...data, ...ndData]. V121 widens this to also include V118-hidden card-flow sessions. The detection logic + push template remain unchanged — only the input set grows.

Sub-tab scope (per V118 spec § Q2 locked)

  • ✅ วันนี้, กำลังจะถึง, ก่อนหน้า — bubble visible when count > 0
  • ❌ ยกเลิก — no bubble (matches V118: OPD lifecycle row hidden on cancelled sub-pill)

Class-of-bug protection (Rule P)

  • AV118 extended — sanctioned-callsite list grows from 4 to 5 (new isCardFlowUnread). All OPD-state derivations still funnel through opdSessionState.js. No inline duplicates allowed.
  • V12 multi-reader-sweep protection — bubble counts pulled from 3 surfaces (desktop tab, mobile dock, sub-pill bar) all consume the SAME helper. Single source = no drift.
  • V80/V119 hook-import drift — V121 uses existing useMemo (already imported post-V119); no new hook adds. Permanent AV60 gate from V119 protects.
  • R1 (listener inclusion verification) — Spec-time check: do the 5 session listeners load isHiddenFromQueue:true sessions? Per V116 design, the listener queries the whole collection + UI filters. If listener filters out hidden → would need an additional source. Implementer to verify pre-task-1.

Decisions (locked via Q&A with user)

  • Q1 = B: Bubble persists until 🔴 บันทึก OPD click. Pure review (ดูข้อมูล) does NOT mark read. Stronger UX nudge than the existing Clinic-queue semantic.
  • Push pipeline: EXTEND existing listener (no new infra). Same sound, same browser push, same FCM.
  • Bubble surfaces: Desktop sidebar tab + mobile bottom dock + each active sub-pill (วันนี้/กำลังจะถึง/ก่อนหน้า). NOT ยกเลิก sub-pill.
  • Bubble color: Purple #a855f7 to distinguish from existing red/orange/green bubbles.
  • Filter: createdFromBackendBooking && isHiddenFromQueue && isUnread && !isOpdSessionSaved — all 4 predicates required.
