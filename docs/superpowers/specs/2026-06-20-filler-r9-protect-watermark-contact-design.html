Purpose / Problem

Three additions to the public Penile Filler Size Simulator, all clinic-protection / branding:

  3. Contact buttons (📞 โทร · LINE OA · Facebook) in the header AND footer, with real brand icons, theme-aware in both dark and light.

Decisions (locked via Q&A with user)

  • Q1 = Client obfuscation: scoped vite-plugin-javascript-obfuscator on the 4 filler files (prod build only). Keeps the page pure-client/instant/offline. Deterrence, not encryption (disclosed).
  • Q2 = Centered-faint watermark: real PNG logo, theme-aware, same on 2D (in-SVG) + 3D (DOM overlay), ~8–10% opacity, non-interactive.
  • Q3 = Contact buttons: header (compact icons) + footer (full), real brand SVG icons, real values (097-525-1525 · lin.ee/mFFsDkG · facebook.com/loverclinickorat).
  • Theme-aware (user add-on): button colors + icon colors adapt — dark = filled brand + white icon; light = soft brand-tint + deepened brand-color icon/label + border (AA-safe). Watermark logo swaps white/black.

Acceptance Criteria

  1. Header shows 3 compact contact icon-buttons (real brand SVGs); footer shows 3 full buttons. Each has the correct href (tel:0975251525 / lin.ee/mFFsDkG / facebook.com/loverclinickorat); LINE+FB carry target="_blank" rel="noopener noreferrer".
  2. Buttons + icons are theme-aware: dark filled+white, light soft-tint+brand-color; verified visually in both themes (Rule Q L1).
  3. Centered-faint clinic logo watermark (theme-aware, ~8–10% opacity, pointer-events:none) renders on the 2D side-view + cross-section AND over the 3D canvas; visible in a screenshot of each view.
  4. npm run build clean; dist/ filler chunk obfuscated — grep the built JS for K_REALISTIC / 2.37 / CONDOM_LADDER → NOT present in clear. The rest of the bundle is byte-unaffected by the plugin scope.
  5. Full vitest stays 16837/0 (vitest bypasses the build plugin); new v5.5 + obfuscation-config tests pass.
  6. NOT deployed (await explicit "deploy"; pure-client → vercel-only, no Probe-Deploy-Probe).

Risks / Open Questions

  • Obfuscator bundle bloat / JSX interplay: the plugin runs on bundled JS post-transform; controlFlowFlattening:false + compact:true keep size sane. Verify the filler chunk size before/after + that the page still runs (Rule Q L1).
  • Watermark opacity tuning: 8–10% is a starting point; final value tuned in preview (must read as a faint watermark, not obscure the anatomy).
  • 3D overlay vs in-scene: DOM overlay is screenshot-covered (the stated threat). A right-click "save canvas image" would NOT include it — documented as an acceptable limitation; in-scene HUD sprite is a future upgrade if needed.
  • Contact values: hardcoded (page is pure-client; values are public marketing info, provided by the clinic operator). If the clinic changes a number/link, edit the CLINIC_CONTACT const.
