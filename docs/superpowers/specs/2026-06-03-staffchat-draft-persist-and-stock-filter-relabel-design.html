Staff-Chat Draft Persist on Minimize + Stock Filter Relabel/Reorder

Date: 2026-06-03 (EOD+2) · Status: Approved by user ("ok") · Implementation plan: docs/superpowers/plans/2026-06-03-staffchat-draft-persist-and-stock-filter-relabel.html (pending) · Type: 2 independent small UI changes (cosmetic-shell spirit — no flow/logic/wiring rewrite).

Purpose / Problem

Feature A — Staff-chat draft lost on minimize. In the staff chatbox, after typing a message / setting a reply / staging image+file uploads, pressing the – (minimize) button and re-opening loses everything. Re-typing is friction during real clinic use. The user wants the in-progress working state preserved across a minimize→reopen cycle on the same screen/tab. Losing it on a full Frontend↔Backend switch / page reload / tab-close is acceptable ("ยกเว้นปิดไปทั้ง tab หรือ เปลี่ยน tab หรือปิดไปทั้งหน้า อันนั้นไม่เป็นไร").

Root cause (verified in source): StaffChatWidget.jsx:66-95 renders chat.minimized ? <StaffChatBubble/> : <StaffChatPanel>…</StaffChatPanel>. Minimizing therefore unmounts the entire Panel including StaffChatComposer, whose local React state (text, pendingFiles — holding live File objects + URL.createObjectURL previews) is destroyed; the composer's unmount effect (StaffChatComposer.jsx:58-60) even revokes the object URLs. (replyingTo lives in the always-mounted useStaffChat hook, so it already survives — but the dominant text+file loss makes the whole thing feel wiped.)

Feature B — Stock balance filter labels too verbose + wrong order. On tab=stock → ยอดคงเหลือ, rename the filter หมด (คงเหลือ 0) → หมด and ติดลบ (ต้องเติมสต็อค) → ติดลบ, and swap them so หมด comes before ติดลบ. Source: StockBalancePanel.jsx:385-395.

Architecture

Feature A (2 files, ~10 lines)

  • StaffChatWidget.jsx — replace the minimized ? Bubble : Panel ternary with: always render <StaffChatPanel hidden={chat.minimized} …>{children}</StaffChatPanel>, and render <StaffChatBubble … /> additionally when chat.minimized. Children (MessageList + Composer) stay mounted across minimize.
  • StaffChatPanel.jsx — accept a hidden prop; apply Tailwind hidden (= display:none) to the fixed root when true (zero layout impact; it's an overlay). Required follow-on: the mobile body-scroll-lock effect (sets data-staff-chat-open, lines 29-34) must key off visible-state not mount — set the attribute when shown, remove when hidden — or minimizing on mobile would leave the page scroll-locked. (useStaffChatPanelResize + grip unaffected.)
  • No change to StaffChatComposer internals, the upload pipeline (taskRefs/cancelRef/progress/retry), or useStaffChat. Mid-upload minimize keeps the upload running in the still-mounted composer.

Feature B (1 file)

  • StockBalancePanel.jsx:385-395 — move the filter-out-of-stock (หมด) <label> block before filter-negative-stock (ติดลบ); change the two label texts to drop the parentheticals. Nothing else moves (row badges at ~454-462 stay).

Decisions (locked via Q&A with user)

  • Feature A approach = "ซ่อนไม่ unmount": keep the Panel mounted + display:none on minimize. Chosen over (B) lifting draft into the hook (invasive to the upload pipeline, mid-upload-unmount risk) and (C) sessionStorage (can't hold staged File/Blob uploads → fails the "preserve files" requirement).
  • Feature A scope = keep across backend sub-tabs: draft persists through Stock→Sales sub-tab nav (widget never unmounts there); clears only on Frontend↔Backend switch / reload / tab-close. No extra per-tab clear logic.
  • Feature B = relabel + reorder only: "หมด (คงเหลือ 0)"→"หมด", "ติดลบ (ต้องเติมสต็อค)"→"ติดลบ", หมด before ติดลบ. (Direct instruction, no alternatives needed.)

Acceptance Criteria

  1. A1: Type text + set a reply + stage ≥1 image and ≥1 non-image file in the staff chat, press –, then re-open → the typed text, the reply strip, and every staged chip (with working preview/icon) are all still there.
  2. A2: The staged files still upload + send correctly after a minimize→reopen cycle (object-URLs not revoked; File objects intact).
  3. A3: A full page reload OR a Frontend↔Backend switch clears the draft (real unmount/unload) — confirms the persistence is per-mount, not leaked to storage.
  4. A4: On mobile, minimizing un-locks page scroll (the data-staff-chat-open body lock follows visible-state); re-opening re-locks it.
  5. A5: V82 force-open (incoming message auto-expands while minimized) still works and shows the intact draft.
  6. B1: The two filters read exactly หมด and ติดลบ (no parentheticals), in the order … เกินสต็อก · หมด · ติดลบ.
  7. B2: Both filters still filter correctly (=== 0 / < 0); badges + colors unchanged.
