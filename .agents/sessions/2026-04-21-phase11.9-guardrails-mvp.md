## Decisions

1. **Phase 11.9 product-group schema is 2-type not 4** — re-Triangle scan of `/admin/product-group` on 2026-04-21 confirmed ProClinic has only `ยากลับบ้าน` and `สินค้าสิ้นเปลือง` as `product_type` options. Phase 11.2 had wrongly claimed 4 options (V10 violation). Legacy data auto-migrated via `normalizeProductType` helper on every read. `productType` radio disabled in edit mode per ProClinic behavior.

2. **Per-group-product qty via pivot table** — ProClinic's `/admin/api/product-group` returns `products[].pivot.qty` (e.g. `"12.00"` → shows as "Paracetamol (12 เม็ด)"). Our schema stored flat `productIds: string[]`, losing qty. Fix: `products: [{productId, qty}]` canonical; `productIds: productIds(derived)` kept for legacy grep.

3. **Full-field sync (JSON API not HTML scrape)** for products + courses. Triangle re-scan revealed HTML `extractProductList` grabbed cells by index, often mis-coding `product_type` (→ user's bug "ทุก product เป็น ยา หมด"). Switched to `/admin/api/v2/product` + `/admin/api/course` with 200 per_page pagination. Normalizers preserve every field mapMasterToProduct/Course reads, including nested `product_label` (flattened to snake_case in master_data per existing mapper contract).

4. **Removed master_data fallback** in TreatmentFormPage med/consumable modals. Previous `beGroups.length > 0 ? beGroups : getAllMasterDataItems('medication_groups')` silently fell back to legacy cache when be_product_groups was empty — user deleted all groups in Tab but modal still showed old data ("มันหลอน"). Rule H: be_ canonical, master_data = dev-only cache. Delete-in-tab MUST propagate.

5. **Adapter registry = single source of truth for "has be_*?"** — BE_BACKED_MASTER_TYPES expanded 4→13 (Phase 12.x + Phase 9 + Phase 11). Fixed MasterDataTab [A3] debug panel that falsely showed "ยังไม่มี be_*" for 9 types that actually had be_ collections. User caught the contradiction: "2 ตารางขัดแย้งมาก".

6. **claude-guardrails shipped for SELF-USE, not public release.** User: "ใช้เองไปเลย". Skipping v1.0 polish (npx CLI, stack examples, external testing). Template is git-cloneable into any future project via `bash F:/claude-guardrails/install.sh .`. Growth via manual Rule D — never AI-compressed.

7. **Session handoff as SKILLS not just docs.** Previous approach = user manually pastes Resume Prompt. New: `/session-start` skill reads 4 mandatory files + verifies iron-clad comprehension; `/session-end` updates state files + emits Resume Prompt; `/violation-log` enforces V-entry format. Discovered via Claude Code's skill-description trigger mechanism.
