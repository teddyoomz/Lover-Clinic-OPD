## Decisions (1-line each)

1. **2 pre-rollout shared extracts first**: UnitField + Pagination — pure
   refactors, ship before rollout commits. Tests catch import path issues
   via `npm run build`.
2. **Pagination key = serialized filter state per panel** — reset to page 1
   on filter change. Functional setter in clamp-on-shrink avoids race when
   key + items both change in same render.
3. **Items 3+4 architecture**: per-branch movement audit is correct; user
   wanted unified audit trail across both endpoints. Solution: writer adds
   `branchIds: [src, dst]` to 4 movement types (8/9/10/13); reader does
   dual-query (`branchId == X` UNION `branchIds array-contains X`). Old
   movements still match Q1 → no schema migration required.
4. **Item 2 root cause**: NOT a scope bug (e65d335 fixed that). The actual
   issue is pre-V20 batches with `branchId='main'` while BranchContext now
   returns `BR-XXX`. Solution: opt-in `includeLegacyMain: true` flag in
   `listStockBatches`. 3 stock create forms opt in. V21 anti-regression:
   flag is explicit knob, can be dropped after admin migration.
5. **Item 5 schema**: 4 NEW fields on `be_stock_transfers` —
   dispatchedByUser/At + receivedByUser/At. Non-cancel/reject path; cancel/reject
   actor not added per user's specific request (only 3 roles wanted).
6. **Item 6 schema**: 4 NEW fields on `be_stock_withdrawals` —
   approvedByUser/At + receivedByUser/At. "ผู้อนุมัติและส่งสินค้า" combined
   per user copy.
7. **Item 7 four-form pattern**: CentralStockOrderPanel gets smart UnitField
   dropdown (replaces raw `<input type="text">`). Adjust/Transfer/Withdrawal
   get a read-only "หน่วย" cell auto-populated from picked batch — they
   don't have user-editable unit (unit is determined by chosen batch).
8. **V14 lock everywhere**: `_normalizeAuditUser` returns
   `{userId, userName}` not undefined. branchIds always uses `.filter(Boolean)`.
   getUnitOptionsForProduct + getDefaultUnitForProduct never return undefined.
9. **Composite-index soft-fail**: dual-query in both `listStockMovements`
   (branchIds array-contains) and `listStockBatches` (legacy main fallback)
   wraps Q2 in `.catch()` returning empty docs. V31 no-silent-swallow:
   console.warn classifies the failure. First admin run logs the
   create-index URL; subsequent runs work.
