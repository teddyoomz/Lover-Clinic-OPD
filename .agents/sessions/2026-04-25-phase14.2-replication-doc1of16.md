## Decisions (non-obvious, worth preserving)

1. **Raw-HTML placeholder `{{{key}}}` (3 braces)** — user-reported bug
   "ใบยังมีมั่ว" showed table rows printing as escaped text `<tr><td>...`.
   Engine fix: 3-brace placeholder bypasses htmlEscape entirely. ONLY use
   for app-built strings (never user input). Templates use it for
   treatmentRecordRows / homeMedicationRows / deductionRows. Pass order:
   lang → unless → if → raw-HTML → escaped (so dropped blocks still strip
   raw HTML inside).

2. **Schema mapping verified via preview_eval, NOT guessed** — V13 lesson
   reinforcement. Real `be_treatments.detail` fields:
   - `treatmentItems` (NOT courseItems!) → ProClinic "Treatment record" table
   - `consumables` + `medications` → "Home medication" table
   - `vitals = { systolicBP, diastolicBP, pulseRate, respiratoryRate,
     temperature, oxygenSaturation, weight, height }` — combine systolic
     + diastolic for BP display
   - `doctorName` field gets concatenated 3x (primary+co-doctor+assistant
     stored as single string `"Name (X)Name (X)Name (X)เลือกแพทย์ประจำตัว"`)
     → strip via regex `/^[^)]+\)/` to get just first occurrence

3. **F12 test bank catches doc-replication drift** — 32 tests (16 docTypes ×
   2: full + empty context). Asserts: no `{{key}}` leak, no escaped
   `<tr/<td/<table` text, mustContain[] strings present, mustHaveValue[]
   prefilled values appear. Caught 3 failing docs (chart / consent / +1)
   that haven't been fully replicated yet — these are the explicit
   user-driven-verification mechanism per "เขียนเทสขึ้นมาแล้วทดสอบเองเลย".

4. **Per-cert toggle config (not COMMON_TOGGLES across all)** — verified
   via Chrome MCP DOM: medical-cert (5 โรค) + driver-license + fit-to-fly
   + patient-referral have NO toggles (TH/EN switch only). Opinion / PT /
   Thai-traditional / Chinese-traditional have showCertNumber +
   showPatientSignature toggles. Templates with empty toggles default
   universal toggles to TRUE so {{#if X}} blocks in shared sub-templates
   still render.

5. **Auto cert# via runTransaction** — clinic_settings/cert_counters keyed
   by `{prefix}:{YYYYMM}`. Format `MC-202604-0002` (verified live in
   preview). Race-safe per Rule C2 invoice-counter pattern. Best-effort —
   silent fallback to empty if rules block (UX: cert# shows blank, user
   can fill manually).

6. **Chrome MCP installed + connected** — Browser 1 (Windows), deviceId
   `8bdc85cc-b6e5-47d9-b3cd-56957264819d`. javascript_tool successfully
   extracted ProClinic DOM (clinic letterhead + section structure +
   computed styles for medical-certificate). Output gets blocked when it
   contains cookies/CSRF tokens — workaround: clone + strip hidden
   inputs + script tags before extracting innerHTML.
