## Decisions (non-obvious — preserve reasoning)

### D1 — V31 orphan recovery: auto-delete on email-already-exists, refuse for owners + cross-ref
Server `handleCreate` catches `auth/email-already-exists` → looks up
existing user by email → cross-references be_staff + be_doctors for that
uid → if no cross-ref AND email is not in OWNER_EMAILS / @loverclinic.com,
auto-deletes the orphan and retries create. Refusal cases throw with
Thai-translated specific reason ("อีเมลนี้ผูกกับ {role} {id} อยู่แล้ว").

### D2 — V31 self-delete: 3-layer defense
UX (button disabled + tooltip) + Client (handleDelete early-return with
setError before confirm dialog) + Server (uid === caller.uid check).
All three required because each catches a different bypass path
(normal click / programmatic / curl bypass).

### D3 — PDF offstage container at viewport origin (not negative left)
html2canvas (used by html2pdf.js) silently produced blank canvas for
elements positioned at `position: fixed; left: -99999px` in some Chrome
builds. New strategy: outer `<div>` at `top:0; left:0; width:0; height:0;
overflow:hidden; opacity:0; pointer-events:none; z-index:-9999` →
inner wrapper inside at proper paper dimensions with `position: relative`.
html2canvas sees real coords; user sees nothing.

### D4 — Single-page PDF enforcement
`pagebreak.mode='avoid-all'` + `height: 297mm` (fixed, not min-height) +
`overflow: hidden`. Constrains content to exactly 1 PDF page. Templates
shorter get whitespace fill; longer get visible crop (admin can fix the
template). User reported "เกินมา 1 หน้าเป็นหน้าเปล่า" — even with content
fitting, html2pdf's default mode produced an empty 2nd page.

### D5 — listAllSellers emits row per id alias
Not just the canonical staffId/doctorId but also `proClinicId` (numeric).
Old sales saved with "614" still resolve to "Test" without DB migration.
Dedupe via Map by id (later doctors override earlier staff for same id).

### D6 — PV.F.11 directory-walk anti-regression invariant
Test scans every .jsx in src/components/backend (except MasterDataTab)
for `getAllMasterDataItems('staff'|'doctors')`. Fails CI on re-introduction.
Permanent guard for user directive "ไม่ต้องการ mirror from master_data
อีกสักที่เลย".

### D7 — buildPrintContext extended with 8 new keys
patientAddress, birthdate, bloodGroup, emergencyName, emergencyPhone,
passport, visitCount, nationality, customerNameEn, ageMonth, hn, HN
aliases. Some templates use `{{patientAddress}}` (consent / fit-to-fly /
patient-referral) — those rendered blank because keys weren't populated.
