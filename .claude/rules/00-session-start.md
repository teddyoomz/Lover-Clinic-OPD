**O. üÜï productId is the ONLY identity for stock; productName MUST be live-resolved at write time** (added 2026-05-08 after user's 4th-round skip-stock-deduction class bug ‚Äî V43+V44+V45+V46 saga):

User-reported (verbatim, frustrated): **"ý∏´ýπâý∏≤ý∏°ýπÄý∏≠ý∏≤ý∏äý∏∑ýπàý∏≠ý∏°ý∏≤ý∏ïý∏±ý∏î ... ý∏Åý∏πýπÄý∏Ñý∏¢ý∏≠ý∏òý∏¥ý∏öý∏≤ý∏¢ýπÑý∏õýπÅý∏•ýπâý∏ßý∏ßýπàý∏≤ýπÄý∏≠ý∏≤ý∏äý∏∑ýπàý∏≠ý∏°ý∏≤ý∏ïý∏±ý∏îýπÑý∏°ýπàýπÑý∏îýπâ ýπÉý∏™ýπàýπÑý∏ßýπâýπÉý∏ôý∏Åý∏éýπÄý∏´ý∏µýπâý∏¢ýπÑý∏£ý∏ÅýπáýπÑý∏îýπâýπÉý∏´ýπâý∏°ý∏∂ý∏áýπÑý∏°ýπàý∏•ý∏∑ý∏°ý∏≠ý∏µý∏Å"**.

**Invariant**: For any stock-related Firestore write (be_stock_movements, be_stock_batches, be_stock_adjustments, be_stock_transfers, be_stock_withdrawals, be_stock_orders, etc.):
1. **productId is THE primary identity**. All lookups, batch resolution, FIFO allocation, negative-stock targeting use productId only ‚Äî NEVER productName fuzzy match in critical paths.

3. **batch.productName is denormalized DISPLAY CACHE only** ‚Äî never authoritative for new movement writes. Existing batches may carry POISONED names from older bug rounds (V44-era course-name leak, etc.); this is why live-resolve is mandatory.
4. **Fallback chain (in this exact order)**: `liveName || item.productName || batch.productName || ''`. Empty string is the FINAL fallback ‚Äî better than course-name leak. NEVER substitute course-name as a fallback.
5. **AUTO-NEG batch creation MUST also live-resolve productName** at insert ‚Äî prevents poisoning at source. (V46 fix surface 3.)

**Why this rule exists**: V43 + V44 + V45 + V46 are all the **same class-of-bug**: "name leaked into stock layer at some denormalization point". V43 fixed customer.courses[] frozen flag. V44 fixed canonical mapper bypass. V45 fixed dedup-shadow OR-merge. V46 fixes batch denormalization poisoning. Each round revealed a new layer that had silently dropped or misrepresented productName. The architectural fix is: **productId is the ONLY identity; productName comes from be_products live OR from canonical TFP post-V44 chain ‚Äî NEVER from frozen denormalization caches**.

**Audit invariant**: AV24 (`audit-anti-vibe-code` SKILL.md) ‚Äî every `tx.set(stockMovementDoc(...)...)` call must have a `_resolveProductNameLive(productId)` call within the same transaction body OR the productName field must come from item.productName (canonical TFP source). **Anti-pattern (forbidden by Rule O + AV24)**:
```js

  productId: b.productId,
  productName: b.productName,  // ‚Üê DENORMALIZATION POISON RISK
  ...
});
```

```js
// ‚úÖ GOOD ‚Äî live-resolve at write time
const liveName = await _resolveProductNameLive(b.productId);
tx.set(stockMovementDoc(id), {
  productId: b.productId,
  productName: liveName || item.productName || b.productName || '',
  ...
});
```
