Q3 Animation: เคลื่อนไหวยังไง?

ทุก demo card ใช้ B intensity + appointments cyan สี (ตัวอย่างเดียวกัน) — ดูพฤติกรรม animation 4 แบบ.

A — STATIC

นิ่งสนิท ไม่มี animation

เปล่งแสงคงที่ตลอดเวลา. GPU 0%. Reduced-motion 100% safe. รู้สึก "ติดป้าย" ไม่ใช่ "ทำงานอยู่".

Q4 Scope: ที่ไหนได้แสง?

User บอก "backend และ frontend เรา" → ทั้งสองฝั่ง. แต่ "frontend" มี 2 ส่วน — admin (queue/chat) vs customer-facing (PatientForm/QR-schedule). เลือก scope.

A — NARROW

Backend tabs only

  • Backend cards (customers, sales, stock, marketing, finance, reports, master)
  • Backend modals (form modals, detail views, audits)
  • Frontend admin (queue cards, QR panel)
  • Frontend customer (PatientForm, ClinicSchedule)

ปลอดภัยสุด. V85 scope = นี่. Lowest blast radius.

B — BACKEND + ADMIN FRONTEND

Backend + Admin-facing Frontend

  • Backend ทั้งหมด (เหมือน A)
  • + AdminDashboard queue cards
  • + QR/queue panel (รายการคิวผู้ป่วย)
  • + Chat panel
  • + Calendar grid (ปฏิทินนัดหมาย, ตารางหมอ)
  • PatientForm (customer fills) — เก็บเรียบไว้
  • ClinicSchedule public link — เก็บเรียบไว้

⭐ Recommended. Glow ทุกที่ที่ admin ใช้งาน. Customer-facing เก็บ minimal (ไม่ทำให้ patients รู้สึก "ตื่นเต้น" ตอนกรอกฟอร์ม).

C — EVERYTHING

Backend + Frontend ทั้งหมด

  • Backend (เหมือน A)
  • + AdminDashboard ทั้งหมด
  • + PatientForm (customer)
  • + ClinicSchedule public link
  • Menu system (AV81 lock — ห้ามแตะ)
  • Print views (AV81 lock — PDF breaks)

ทั่วถึง. ⚠ patient-facing pages อาจดู "เกินไป" สำหรับ first impression.
